# torch.nn实现单机多卡模型并行的tensor parallel

**作者**: NULL让智能更廉价: AI芯片软件栈与异构计算系统

**原文链接**: https://zhuanlan.zhihu.com/p/632456305

---

我们知道torch.nn的并行最开始只有单机多卡数据并行，也就是训练时候会常用到的torch.nn.DataParallel；

支持tensor parallel的模型并行是torch2.0刚加入的，而且这个模型是基于多机多卡的distributed包，这个包比较重，并且对单机模型并不容易改造；

如果想在单机多卡上，推理tensor parallel的模型，一时间找不到趁手的工具，于是基于torch.nnDataParallel中单机多卡的通信机制，改造一个支持tensor parallel的transformer；

基于torch源语封装的tensor parallel是针对transformer中的 liner算子进行改造，效率会低于faster transformer这种对整个attention子图进行tp

下面是一个liner的例子：

import torch
import torch.nn as nn
import torch.nn.functional as F
import torch.nn.parallel

# 创建模型
class MyModel(nn.Module):
    def __init__(self):
        super(MyModel, self).__init__()
        #MyModel作为一个warp没有固定权重，运行时从kv cache里面load
        self.weight = None
        self.bias   = None

    def forward(self, input):
        return F.linear(input, self.weight, self.bias)

# 复制模型副本到多个设备
model = MyModel().cuda()
device_ids = [0, 1, 2, 3]  # 假设有4个设备
replicas = nn.parallel.replicate(model, device_ids)

#每个模型分别设置自己的权重
weight = torch.randn(4, 5, 10)
bias   = torch.randn(4, 5)
weight_chunks = nn.parallel.scatter(weight, device_ids, dim=0)#模型并行,将模型切到4个GPU上
bias_chunks   = nn.parallel.scatter(bias, device_ids, dim=0)

for i in device_ids:
    replicas[i].weight = torch.flatten(weight_chunks[i], start_dim=0, end_dim=1)#降1维
    replicas[i].bias = torch.flatten(bias_chunks[i], start_dim=0, end_dim=1)

# 准备输入数据
input = torch.randn(4, 10)

# 使用 nn.parallel.comm.broadcast 将输入数据广播到多个设备上
inputs = nn.parallel.comm.broadcast(input, device_ids)
#print(inputs)

# 在每个设备上执行线性变换并汇总输出
outputs = nn.parallel.parallel_apply(replicas, inputs)

# 汇总输出结果reduce sum
# transform中mlp和attention最后一层fc都是行切之后sum
output = nn.parallel.comm.reduce_add(outputs, 0)

# 打印输出结果
print(output.size())
