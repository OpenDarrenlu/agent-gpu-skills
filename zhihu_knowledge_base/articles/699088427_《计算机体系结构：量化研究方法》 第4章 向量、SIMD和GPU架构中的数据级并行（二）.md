4.3 用于多媒体的SIMD指令集扩展/SIMD Instruction Set Extensions for Multimedia

许多媒体应用程序操作比32位更窄的数据类型，如图形、音频。图形系统使用8位来表示三个主要颜色和额外8位表示透明度；音频样本通常以8位或16位表示。

通过拆分256位加法器的进位链，处理器可以对32个8位操作数、16个16位操作数、8个32位操作数或4个64位的短向量同时进行操作。这种加法器拆分的额外成本很小。如图4.8所示，列举了典型的多媒体SIMD指令。

图4.8 典型SIMD多媒体支持的256位宽操作总结。IEEE 754-2008浮点数标准添加了半精度（16位）和四倍精度（128位）的浮点操作。

就像向量指令，SIMD指令给向量数据指定了同一操作。但是不像向量指令需要大的register file，可以在32个向量寄存器中存32个64位的元素。SIMD指令倾向指定较少的操作数，只要较小的register file。

与向量架构相比，SIMD扩展有主要缺少三个要素：（1）没有向量长度寄存器，（2）没有带步长或聚集/散布数据传输指令，（3）没有mask寄存器。使得编译器更难生成SIMD代码，并增加了在SIMD汇编语言中编程的难度。

同时因为多媒体SIMD扩展固定了指令中的数据操作数数量，这导致x86架构的MMX、SSE和AVX扩展增加了数百条指令。

1996年引入的MMX指令重新利用了64位浮点寄存器，因此基本指令可以同时执行8个8位操作或4个16位操作。这些指令还包括并行的MAX和MIN操作，各种各样的masking和条件指令（通常在数字信号处理器DSP中），以及在重要的媒体库中有用的临时指令。此外，重新利用了浮点数据传输指令来访问内存。

在1999年，流式SIMD扩展（SSE）增加了16个独立的寄存器（XMM寄存器），宽度为128位，此时指令可以同时执行16个8位操作，8个16位操作或4个32位操作，还可以执行并行的单精度浮点运算。由于SSE有独立的寄存器，需要独立的数据传输指令。在2001年的SSE2、2004年的SSE3和2007年的SSE4指令集中增加了双精度SIMD浮点数据类型。

在2010年增加先进向量扩展（AVX）将寄存器的宽度增加到256位（YMM寄存器），从而提供加倍较窄数据类型操作数数量的指令。图4.9展示了用于双精度浮点计算的AVX指令。2013年的AVX2新增了30条新指令，例如聚集（VGATHER）和向量移位（VPSLL，VPSRL，VPSRA）。2017年的AVX-512再次将寄存器宽度扩大到512位（ZMM寄存器），将寄存器数量增加到32个，并新增了约250条新指令，包括散布指令（VPSCATTER）和mask寄存器（OPMASK）。AVX还为提供了未来将寄存器扩展到1024位的可能性。

多媒体SIMD架构编程

考虑到SIMD多媒体扩展的临时性，使用这些指令的最简单方法是通过库或使用汇编语言编写。

近期扩展变得更加有规则，使得编译器有一个更合理的目标。通过借鉴向量化编译器的技术，编译器开始自动生成SIMD指令。然而，必须确保将所有数据在内存中对齐到代码运行的SIMD单元的宽度，以防止编译器将本来可以向量化的代码生成为标量指令。

Roofline可视化性能模型

Roofline模型（Williams等人，2009）是一种可视化比较不同SIMD架构潜在浮点性能的方法，将浮点性能、内存性能和算术强度联系到二维图中，如图4.11所示。

算术强度是每个访问的内存byte的浮点操作比例，可以通过程序的总浮点操作数除以程序执行期间传输到主存的总数据byte数来计算。如图4.10所示，展示了几个示例的相对算术强度。

图4.10 算术强度示例

通常，峰值浮点运算性能可以在硬件规格中找到。处理器可达到的峰值内存带宽，通常需要通过Stream基准测试找到峰值内存性能。

图4.11 NEC SX-9向量处理器（左侧）和Intel Core i7 920多核计算机（右侧）的Roofline模型。这个Roofline模型是针对单位步长内存访问和双精度浮点性能的。

如图4.11所示，Y轴表示可达到的浮点性能，范围从2到256 GFLOPS/s。X轴表示算术强度，算术强度从1/8 FLOP/DRAM byte访问到16 FLOP/DRAM byte访问。

如何绘制性能峰值?

X轴是FLOP/byte，Y轴是FLOP/s，所以byte/s在图中形成了一个45度的对角线。绘制第三条线用于表示计算机的内存系统在给定算术强度下所能支持的最大浮点性能。

可以表示为一个公式，绘制如图4.11中的线:

Attainable GFLOPs/s = Min( Peak Memory BW × Arithmetic Intensity, Peak Floating Point Perf. ） 可达到的GFLOPs/s = Min( 峰值内存带宽 × 算术强度, 峰值浮点数性能 ）

“Roofline”根据内核的算术强度设置了其性能的上限，依据算术强度可以将Roofline模型划分为2个区域，一个是带宽受限，一个是计算受限。如图4.11，右侧的虚线（算术强度为4）是计算受限的例子，左侧的虚线（算术强度为1/4）是带宽受限的例子。
