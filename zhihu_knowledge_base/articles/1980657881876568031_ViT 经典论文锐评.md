（2020）ViT, AN IMAGE IS WORTH 16X16 WORDS: TRANSFORMERS FOR IMAGE RECOGNITION AT SCALE

https://arxiv.org/pdf/2010.11929

本文是一个早期作品，阐述了图像分patch、patch embedding作为token的思想。

图像 patch 分割与嵌入：

将输入图像 x∈R&(H×W×C) 分割为 N=HW/P^2 个 P×P 大小的 patch（如 16×16）；
每个 patch 展平为向量后，通过可训练线性投影映射到维度为 D 的patch embedding（与 Transformer latent vector 尺寸一致）。
关键嵌入补充：[class] token：在 patch embedding 序列前 prepend 一个可学习的嵌入向量，其在 Transformer 编码器输出端的状态作为图像全局表征，用于分类（类比 BERT 的 [CLS] token）；

vision

小数据集时，CNN架构优势明显；大数据时，ViT反超。优势主要体现在计算效率、预训练成本上。
锐评：CNN的平移不变、近距离局部计算等空间假设，带来了更低的学习成本，但也限制了表达空间。

集合预测

目标检测任务，是典型的集合预测问题，需要全局推理机制，对所有预测元素之间的交互进行建模，以避免出现重复的结果。
本文直接输出目标集合（类别 + 边界框），无需中间步骤。输出的顺序无所谓，和真值匹配时使用匈牙利匹配，解决标签分配问题。

模型

变种的encoder-decoder结构
encoder，每层的attention都会在Q、K上，加上位置编码。和vanilla transformer不同，后者是只在输入层加，只加一次。
decoder，本文使用并行解码，提高推理速度。非GPT-like decoder, 不使用causal attention mask。
整体看起来，在位置编码上加了很多模型结构上的trick，尤其是decoder的learnable pos queries在每一层都加，感觉不一定会很有效，反而限制了学习能力。

loss

分类损失：类别不平衡处理：“无目标” 类别损失权重降低 10 倍（类比 Faster R-CNN 的正负样本采样）。
边界框损失：采用L1 损失 + 广义 IoU（GIoU）损失的线性组合。

（2020）Deformable DETR: Deformable Transformers for End-to-End Object Detection
arXiv reCAPTCHA
探索Deformable Attention：打破传统注意力机制的边界

文中提出了Deformable Attention. 由query feature直接预测离散的、有限的注意力点、注意力值，使用注意力点从feature map直接index获取value，和注意力值相乘，得到结果。该方法的计算复杂度与序列长度无关，不是传统的全局注意力。

锐评：与其说是transformer attention，不如说是变种deformable convolution。没有query、key，直接预测postion和aggregation weight，难评。

（2021）CLIP, Contrastive Language-Image Pre-training
https://arxiv.org/pdf/2103.00020
神器CLIP：连接文本和图像，打造可迁移的视觉模型

锐评：图文对齐的真正的有影响力的开山之作

dataset size: 400M image-text pairs

# image_encoder - ResNet or Vision Transformer
# text_encoder - CBOW or Text Transformer
# I[n, h, w, c] - minibatch of aligned images
# T[n, l] - minibatch of aligned texts
# W_i[d_i, d_e] - learned proj of image to embed
# W_t[d_t, d_e] - learned proj of text to embed
# t - learned temperature parameter

# 分别提取图像特征和文本特征
# 有两种架构，CNN和ViT
# CNN使用attention pooling 而非average pooling. the query is conditioned on the global average-pooled feature.
# ViT使用最前面的[Class] token
I_f = image_encoder(I) #[n, d_i]
# EOS token feature被视为整段文本的特征代表，经过一层线性层投影到多模态特征空间。
T_f = text_encoder(T) #[n, d_t] 

# 对两个特征进行线性投射，得到相同维度的特征，并进行l2归一化
I_e = l2_normalize(np.dot(I_f, W_i), axis=1)
T_e = l2_normalize(np.dot(T_f, W_t), axis=1)

# 计算缩放的余弦相似度：[n, n]
logits = torch.matmul(I_e, T_e.T) / t

# 对称的InfoNCE Loss：等价于N个类别的cross_entropy_loss
labels = np.arange(n) # 对角线元素的labels
loss_i = cross_entropy_loss(logits, labels, axis=0)
loss_t = cross_entropy_loss(logits, labels, axis=1)
loss = (loss_i + loss_t)/2

(2021) DPT: Vision Transformer for Dense Prediction

https://arxiv.org/pdf/2103.13413

本文章提出了一种方法，可以以transformer encoder + convolutional decoder的方式来生成密集预测，用于解决深度预测、全景分割等任务。

convolutional decoder的设计：patch（as 16x16）、embedding（>=768）。embedding的通道数大于等于patch的像素数量*3，意味着可以完全保留信息。
经transformer encoder
从不同深度的transformer里取特征
read: transformer encoder的输入序列里，设计了借鉴 NLP 的 [CLS] 令牌，用于全局图像表示。read操作是将此全局token，加到各个位置的token上
concatenate：每层特征经拼接转换后，从1维的序列变成一张二维的特征图
resample：二维特征图，先经1x1卷积调整通道数，再经上采样、或下采样的卷积运算后，改变分辨率。越深来源的transformer特征，resample后的分辨率越低。
原文实现是形成1/32、1/16、1/8、1/4原图分辨率的特征图。
fusion: 不同深度的不同分辨率的特征图，使用RefineNet方法（像U-Net一样层层相加）合并。并上采样一倍。
最终输出原始分辨率一半的prediction
接task-specific head

锐评：transformer版的UNet，保守但有用。有没有更创新的方法？

(2021) Swin Transformer: Hierarchical Vision Transformer using Shifted Windows
https://arxiv.org/pdf/2103.14030
图解Swin Transformer
开源

试图解决 Transformer 从自然语言处理（NLP）适配到计算机视觉的关键问题 —— 视觉实体尺度变化大、图像像素分辨率远高于文本单词。

锐评：微观transformer + 宏观cnn，但总感觉有点不太平滑。全局注意力的缺乏，还是差点意思。

Swin Transformer 的解决方案：

抛弃了transformer经典的全局注意力，改成小窗口视野，通过patch merging、shift window attention来逐层扩大感受野。

分层特征图：从细小图像补丁（patch）开始，深层逐步合并相邻补丁、降低分辨率，生成多分辨率特征图（与 CNN 特征图分辨率一致），可直接适配 FPN、U-Net 等密集预测技术。
局部窗口自注意力：将图像划分为非重叠局部窗口，每个窗口包含mxm个patch，自注意力仅在窗口内计算（窗口大小固定，复杂度与图像尺寸线性相关）。
移位窗口连接：连续自注意力层交替使用 “正则窗口” 和 “移位窗口”，实现跨窗口信息交互（图 2），弥补局部窗口的建模局限性。移位窗口，移动m/2 x m/2。
相对位置偏差: 自注意力计算中加入相对位置偏差，捕捉补丁间相对位置信息

Details

参考点生成：在输入特征图上生成均匀网格参考点（下采样因子为 r，记为HG​×WG​=HW/r2），并归一化到 [-1, +1] 范围（对应特征图四角）。
偏移网络生成偏移量：
输入：query 特征（q=xWq​）；
结构：5×5 深度可分离卷积（捕捉局部特征）→ GELU 激活 → 1×1 卷积（无偏置，避免强制偏移）；
稳定训练：偏移量经Δp←s⋅tanh(Δp)缩放（s 为预设因子，防止偏移过大）。
可变形键值采样：通过双线性插值（可微）在 “参考点 + 偏移量” 位置采样特征，生成变形后键（kˉ）和值（vˉ），公式为：ϕ(z;(px​,py​))=∑(rx​,ry​)​g(px​,rx​)g(py​,ry​)z[ry​,rx​,:]
其中g(a,b)=max(0,1−∣a−b∣)，仅对 4 个邻近整数位置加权。
可变形相对位置偏差：针对变形后键的连续位置，通过 “参数化偏差表插值” 计算偏差（而非 Swin 的离散查表），覆盖所有可能偏移值，增强空间信息建模。
偏移分组（Offset Groups）：将特征通道分为 G 组，每组独立生成偏移量，提升采样多样性；注意力头数 M 设为 G 的整数倍，确保多头部复用一组变形键值。

(2022) DN-DETR
https://arxiv.org/pdf/2203.01305
DN-DETR: 把去噪训练引入到 DETR 中打辅助，从而加速收敛

DETR具有收敛慢的缺点，本文认为是匈牙利匹配在前期的多义性造成的问题，并尝试解决这个问题。

新增一个去噪辅助任务，和原本的DETR任务并行。该辅助任务：

输入：
在gt框的基础上加噪，形成新的query
控制噪声的幅度，形成多个group。不同group之间相对独立。
decoder embedding改成class label embedding（label -> embedding查表） + indicator(哪个任务）
输出：gt框
模型：
输入和输出一一对应，避免匈牙利匹配的多对多、多义性匹配。
特殊attention mask，避免不同group、不同任务之间出现信息泄露。

模型

一个模型架构，两种参数，凑成两个实际模型，分别为student model，teacher model。
backbone+head。backbone是训练目标，head输出n分类、只在训练里使用。

训练过程

一张图片，crop为96x96（local）、224x224（global）两种规格。
global进teacher，经centering + 带温度的softmax，输出n分类的logits
centering，防止坍缩到固定单波峰输出。
温度sharpen，防止坍缩到均匀输出。
local/global进student，经带温度的softmax，输出n分类的logits
teacher/student的logits经cross entropy loss，得到梯度。
梯度只训student，不训teacher
teacher以momentum形式从student里更新，训练步数越多，惯量常数越大，后期几乎不更新。

2023 DINOv2

论文：https://arxiv.org/pdf/2304.0719

核心之一，数据清洗




自监督为啥也要洗数据？

低质量、带噪声的数据，和正常数据分布不同，有害
冗余、重复的数据降低了训练效率，缺乏多样性
任务调优
根据任务对齐洗数据，能提高任务性能
防止测试集泄漏
scaling场景下，大模型更容易拟合低质量数据噪声

训练目标：

image-level objective. 和DINOv1类似，蒸馏prototype scores
patch-level objective. 类似于Bert的MIM loss
Sinkhorn-Knopp centering，替代DINOv1的batch moving average centering. 后者仍无法避免prototype scores坍缩到少数prototype的问题，SK避免了这个问题。
SK算法：对于一个矩阵，先逐行做归一化，就是将第一行的每个元素除以第一行每个元素的和，得到新的"第一行";对于每行都做相同的操作； 再逐列做归一化。 重复以上的两步，最终可以收敛到一个行和为1，列和也为1的Doubly stochastic matrix。
涨点收益没有，但是对训练稳定性的提升很多，减少NaN概率
KoLeo regularizer. 最大一个batch内部的向量之间的最小欧氏距离。强迫结果的多样化。

3、4决定了，数据清洗的去重很重要。

锐评：数据、算法、效果都很棒，但是没有突破，local-global蒸馏、MIM终归不是视觉自监督的正道。

GramLoss是在1M个iteration后作为第二训练阶段加入的。
