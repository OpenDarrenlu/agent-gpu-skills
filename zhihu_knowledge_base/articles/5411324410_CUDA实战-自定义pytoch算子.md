首先定义一个square_martix函数，通过这个函数返回kernel的计算结果。因为kernel必须定义成void类型，所以需要多一层封装将结果返回给 前端python调用

#include <torch/extension.h>
torch::Tensor square_martix(torch::Tensor matrix){
 
    .........
    return result;
}


这个函数具体的作用，可以总结为设计好kernel要使用的硬件资源并调用cuda kernel，如grid_size,block_size等。但是不需要像前文一样对数据进行设备端的内存开辟和复制了，这些我们在python端都会完成处理。具体代码为

torch::Tensor square_martix(torch::Tensor matrix){
    // 定义主机端 和 host端数据
    // 矩阵的 高宽 
    const auto height = matrix.size(0);
    const auto width = matrix.size(1);

    // 数据传输的问题 ---> python层面直接传输给cuda然后直接调用定义在cuda上的算子实现就行
    auto result = torch::empty_like(matrix);

    dim3 threads_per_block(16, 16);
    dim3 number_of_blocks((width + threads_per_block.x - 1) / threads_per_block.x,
                          (height + threads_per_block.y - 1) / threads_per_block.y);

    square_kernel<<<number_of_blocks, threads_per_block>>>(
        matrix.data_ptr<float>(), result.data_ptr<float>(), width, height);
    return result;
}


拿到了我们能 使用的硬件资源，再就轮到具体kernel的设计了。首先我们需要获得输入数据的内存地址 ，以及输出数据的内存地址，直接对其进行操作。并且传入width 和 height保证线程不会处于越界的位置。

__global__ void square_kernel(const float* matrix, float* result, int width, int height){
   ...... 
}


定义好这些，就可以去定义这个算子具体的实现以及使用一些优化手段,比如使用shared memory，拆分循环 等手法。这里采用的最简单的实现，具体代码如下

// 定义核函数
__global__ void square_kernel(const float* matrix, float* result, int width, int height){

    //  得到矩阵的x纬度坐标 ，y纬度坐标
    int row =  blockIdx.y * blockDim.y + threadIdx.y;
    int col = blockIdx.x  * blockDim.x + threadIdx.x;
    if(row<height && col<width){
        int thread_id = row*width+col;
        result[thread_id]=matrix[thread_id]*matrix[thread_id];
    }
}

封装 kernel以及 绑定调用

做完了上述cuda kernel以及调用的定义，就应该将这些 函数进行封装 编译确保python前端能正确导入这些函数。这里需要使用pybind11来进行封装，

PYBIND11_MODULE(TORCH_EXTENSION_NAME, m) {
m.def("square_martix", torch::wrap_pybind_function(square_martix), "square_martix");
}


如果只进行封装的话，确实也可以。但是这样导致每次都需要进行cuda编译会比较耗时，所以需要将其编译成可调用的.so文件

python导入和使用自定义算子

做完前面的所有工作，我们就可以在python端导入这个我们编译好的文件 ，就可以开始 使用了。具体代码如下：

import torch
from torch.utils.cpp_extension import load
square_module = load(name='square_lib', 
        sources=['./cuda_kernel/square.cu'], 
        extra_cflags=['-std=c++17']
    )

在导入之后，我们就可以开始调用 这个模块里面的一些方法 ：

def square(input_martix):
    output_martix = torch.empty_like(input_martix)
    output_martix=square_module.square_martix(input_martix)
    return output_martix

device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
input_tensor = torch.tensor([[1., 2., 3.], [4., 5., 6.]], device='cuda') #将数据定义到cuda上
output_tensor = square(input_tensor)

我们 也可以对这个导入 的模块做一些性能和正确性上的分析 ，比如和正常的矩阵乘法的运行时间对比，kernel的 cpu耗时和gpu耗时等，具体代码如下：

CPU和 GPU耗时测量代码如下 ：

def time_pytorch_function(func, input):
    # CUDA IS ASYNC so can't use python time module
    start = torch.cuda.Event(enable_timing=True)
    end = torch.cuda.Event(enable_timing=True)

    # Warmup
    for _ in range(5):
        func(input)

    start.record()
    func(input)
    end.record()
    torch.cuda.synchronize()
    return start.elapsed_time(end)

time_pytorch_function(square,input_tensor)
