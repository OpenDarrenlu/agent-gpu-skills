论文链接：HTTPS://arxiv.org/pdf/2602.14234 （HTTPS://redsearchagent.GitHub.io/）

研究团队：REDSearcher Team 小红书团队

核心贡献：提出 REDSearcher 统一搜索代理优化框架，通过双约束复杂任务合成、工具增强查询设计与能力导向的中期训练，并结合本地模拟环境下的强化学习优化，解决长时程搜索任务中高质量轨迹与有效奖励信号稀缺问题，显著提升大语言模型在文本与多模态深度搜索任务中的规划能力、工具使用能力与整体搜索性能，并实现多项基准测试的最优结果（SOTA）。

一、研究背景

现有 Deep Search 瓶颈 由于有效监督信号的极端稀疏，优化 LLM 的 Deep Search 受到了阻碍。包括两个方面： 1.合成高质量数据困难：大规模合成复杂、高质量推理任务的难度极大； 2.合成数据成本高：收集涉及广泛外部工具使用的交互密集型轨迹所需的计算和时间成本极高。

二、核心方法：REDSearcher

REDSearcher 是一个用于在纯文本和多模态（图像-文本）设置下训练工具增强型深度搜索智能体的框架。该框架联合优化了任务合成、中期训练（Mid-training）和后期训练，从而实现了对长程搜索行为的可扩展、可控且具有成本效益的优化。 2.1 符号定义与 agent 交互过程示例

2.2 数据合成分思路：

2.2.1 复杂任务合成的两个指标介绍 包括两个指标：1.Topological Logical Complexity；2.Information Dispersion。

指标 1:Topological Logical Complexity： 核心思路：

G = (V, E)

代表查询结构，是一个巨大的图，我们可以尝试从通过“分组”的方式，伪装成一棵树，进而利用树的性质解决问题。 公式建模：

G=(T,\{X_i\}_{i \in I})

中，T 代表树，

X_i

表示第 i 个包。需要满足：




所有包的并集等于 V。
对于每一条边
(u, v) \in E

，存在一个包
X_i

同时包含 u 和 v。
对于任意顶点 v，包含 v 的节点集合
\{i \mid v \in X_i\}

在 T 中形成一个连通子树。 该分解的宽度定义为
max_{i\in I}|X_i|-1

，图 G 的树宽是最小宽度：
tw(G)=min_{T,\{X_i\}}(max_{i\in I}|X_i|-1)

指标推导:Complexity Scaling 推导过程：直观上，树宽可以作为满足耦合约束所需工作记忆（Working Memory）的替代指标：类树结构（低树宽）允许分而治之，而高树宽则表示变量间存在更强的纠缠。我们参照树分解上的动态规划算法，将推理成本近似为：
C_{reasoning}=O(N \cdot d^{k+1})

，其中 N 是推理步数，d 是每步的分支因子（top-d 搜索），k=tw(G)。这突出了 k 的增加会带来指数级负担，迫使智能体必须维护多个纠缠的假设，而非执行简单的顺序推导。 举例子说明：（如下图）

第一类：线性推理 (k = 1)：结构为树或简单链。例如：“A 是 B 的父亲，B 是 C 的父亲……A 是谁？”认知负荷：智能体只需跟踪前驱节点。复杂度为多项式级
O(N \cdot d^2)

。这代表了目前大多数多跳问答数据集的现状。 第二类：环形/菱形约束 (k = 2)：构包含环路或重新汇聚的并行路径。例如：“在哪部 1990 年的黑帮电影中，导演选用了自己的女儿扮演主角的女儿？”认知负荷：智能体必须同时满足电影、导演和女演员之间的约束。这需要内存中维护更大的变量“包”（三元组）以验证一致性，一个分支的失败必然导致回溯。 第三类：高维耦合 (k ≥ 3)：结构为团状结构（如四面体）。例如：“识别人物 A：一位在 80 年代中期权力斗争中被赶出公司的技术联合创始人，他创办了一家新公司，其核心技术后来被原公司收购。”认知负荷：此处变量 A、B、C 和 D 完全耦合，问题无法分解为独立的子问题，若无有效剪枝，将导致组合爆炸。

指标 2:Information Dispersion 核心思路：互联网信息很多，可能存在检索捷径：单个详尽的文档可能包含多个逻辑关联的事实（如节点 A、B、C、D），使得理论上复杂的实例通过“单次检索”即可解决。 最小源分散度 (Minimum Source Dispersion, MSD)，用于衡量所需证据在来源中的碎片化程度。MSD 定义为覆盖推理图 G 所需信息的最少独立文档数量：

D_{\text{task}} = \min_{S \subseteq W} |S| \quad \text{s.t. } \text{Cover}(S, G) = \text{True}




常见问题 Q&A： 问题 1:为什么要提树宽？ 很多 Agent 论文只提“多跳（Multi-hop）”，但 10 跳的链式推理可能比 3 个变量互相锁死的环状推理（k=2）要简单得多。 问题 2: 如果维基百科里有一个词条直接写了“乔布斯在被苹果赶走后创办了 NeXT 然后又被苹果收购”，那么即使逻辑结构再复杂，Agent 只要搜到这一页就“作弊”成功了。引入 MSD 就是为了剔除这些简单的“直达题”，强迫模型去做复杂的跨页整合。

2.2.2 合成数据流水线： 本文构建了合成数据流水线，包括问答生成，验证流水线，具体如下：

2.2.2.1 问答生成流水线： 种子收集与过滤：利用维基百科实体初始化种子库，并进行四层过滤：(i) 长度阈值：剔除过短（信息贫乏）或过长（过于热门）的页面；(ii) 结构过滤：去除列表、索引和词汇表；(iii) 元页面删除；(iv) 概念过滤：利用 LLM 分类器区分具体实体与抽象理论。 图构建与拓扑增强：采用 有向无环图 (DAG) 作为基础数据结构。为了超越简单的多跳检索并达到 k > 2 的结构复杂度，引入了拓扑增强的跨源图构建阶段： 1.不仅依赖 Wikidata 的显式关系，还部署了一个 Graph Agent 来增加拓扑密度。 2.这种“增密”操作在依赖图中引入了环路，打破了搜索路径的线性。这迫使求解器从“顺序检索”转向“联合约束满足”——即有效答案不再是通过单一线索找到的，而是通过验证多个分布式的证据片段是否互相同步一致。 高效子图与答案采样:构建完整的密态图成本极高。本文采用“一图多任务”策略：从一个主图中提取多个截然不同的连通子图作为独立的推理上下文。根据节点的拓扑角色（如深层叶节点 vs 高度数中心节点）选择答案。复用底层图使训练实例增加了数量级，有效分摊了构图开销。 工具强制的查询演化:为了强制智能体使用工具，实现了工具注入策略：专门的 Editor Agent 将查询中的静态实体重写为工具可解的函数依赖。 示例：不再直接命名地点，而是通过地图 API 的路由约束来描述（例如：“位于 [实体 A] 以西约两小时车程的城市”）。 这些重写制造了无法仅靠文本检索填补的信息鸿沟，使工具执行成为推理轨迹的内在前提。

2.2.2.2 验证流水线： 为了确保数据集既有挑战性又可靠，本文采用了由易到难的多级验证：

无工具 LLM 预过滤：如果 LLM 在不使用工具的情况下就能答对，说明题目太简单，剔除。
可检索性检查：通过搜索引擎 API 查询，如果前 50 条结果的摘要中没出现答案，说明该题在开放网络上缺乏支持，剔除。
幻觉/一致性检查：提供构题时的原始证据给 LLM 验证器，剔除存在矛盾的实例。
智能体运行验证 (Agent Rollout)：让强力 Agent 运行 n 次，至少有一次预测对才保留，并记录通过率作为置信度信号。
答案唯一性检查：剔除那些 Agent 发现了其他合理替代答案或歧义解的实例。

2.2.2.3 质量研究：

人类校验：500 个样本中超过 85% 通过了逻辑一致性和归因充分性检查。
难度量化：强力开源模型 DeepSeek-V3 在该设置下准确率仅为 ~40%。
人类对比：即使给人类 30 分钟的搜索时间，解决率也仅为 47%。
结论：这证明了数据不仅质量高，且对模型和人类都极具挑战性。

2.3 多模态任务合成流水线： 亮点：复用相同的端到端骨架，仅修改少量步骤以合并视觉证据。这种设计保持了依赖结构的显式性和可验证性，同时使多模态合成在大规模扩展时具备与纯文本合成几乎相同的效率。 核心思路：引入了模态注入（Modality Injection），将纯文本推理 DAG 转变为跨模态推理 DAG，使某些约束锚定在图像中。

2.3.1 模态注入 (Modality Injection)：通过两种互补机制实现： 视觉属性锚定：选择 DAG 中的中间节点 u，为其附加一张图像，并生成详细的视觉内容描述（如显著物体、场景类型、独特标志或图表模式）。这被视为节点属性，作为后续构建的约束。 跨模态依赖：强制执行“视觉不可替代性”约束。如果模型不从图像中提取关键视觉线索（例如背景物体、衣服上的徽章或图表趋势线），就无法获取推导下游节点 v 所需的信息。这确保了图像不是装饰性的，而是解题的必要前提。 多模态问题模糊化 (Multimodal Question Fuzzing)：

视觉语义抽象：避免在问题中直接命名图像内容，而是使用代词或相对描述，迫使模型先识别视觉实体再进行搜索。
模态转换：允许在推理轨迹的任意位置注入视觉证据，而非仅在开头。这可以在文本推理后设置“视觉瓶颈”，增加有效推理深度。 多模态验证流水线 (Multimodal Verifier Pipeline)： 在纯文本验证器的基础上增加了额外检查：
纯文本可解性剔除：如果仅靠文本推理或搜索就能答对，说明图像非必需，剔除。
纯视觉可解性剔除：如果只看图不搜索就能猜到答案，说明题目太直接，剔除。
视觉-搜索对齐验证：确保图像内容与检索到的网页形成互补的推理环路，过滤掉图像与任务无关或仅起装饰作用的实例。 2.3.2 多模态轨迹生成 (Multimodal Trajectory Generation) 本文使用实例化了标准化工具模式的 ReAct 智能体来合成高质量的 SFT（监督微调）轨迹。本文采用 Qwen3VL-235B 模型，在生成意图感知的推理和发布结构化工具调用之间交替进行。
轮次限制：每条轨迹上限为 20 轮交互。
质量筛选：仅保留最终答案与标准答案（Ground-truth）一致的轨迹用于模型微调。

问题 Q&A： 视觉怎么保证有效性：通过图论中的“断路”设计，确保如果不去解析图像中的特定细节（如：勋章上的文字），推理链条就会断掉，模型必须调用视觉能力。 怎么在推理轨迹中间注入视觉证据： 常规做法：看到图 A -> 搜索 -> 得到答案。 本文做法：搜索关键词 1 -> 找到信息 B -> 根据信息 B 去看图 A 里的特定位置 -> 得到线索 C -> 继续搜索。

2.3 mid-train 思路： 包括两个阶段：第一阶段加强原子能力，包括知识对齐（Grounding）和规划（Planning）；第二阶段在此基础上培养多轮环境交互和长程决策能力。 第一阶段：意图锚定的对齐与层级化规划 (32K 上下文) 搜索智能体任务要求模型在长程交互中规划多步搜索策略，并从海量网页中过滤和整合信息。这一过程根本上依赖于两项核心原子能力：

对齐（Grounding）能力：根据当前意图从冗余的观测结果中提取关键信息。
层级化规划（Hierarchical Planning）能力：将复杂任务分解为层级化的子目标，以支持多步规划并保持与全局目标的一致。 “对齐”的实现思路：采用一种带有干扰项的逆向问答合成方法：给定中心实体 E 及其对应文档 D，提取相关事实片段 F，并据此合成查询意图 Q。为了模拟真实网络环境的噪声，输入中不仅包含相关文档，还加入不相关的干扰文档（Distractors）。直接利用维基百科和网页爬虫快照作为种子，无需额外数据采集。 “层级化规划”的实现思路：通过层级化规划将复杂问题的切入点分为两类：明确目标（已有清晰意图并期望获取特定信息）和模糊目标（需要通过查询缩小不确定性以确定具体目标），把知识库实体和网页的拓扑结构沿信息流方向展平，利用 LLM 基于前文生成相应的规划。 第二阶段：智能体工具使用与长程交互 (128K 上下文) 预训练模型缺乏对环境反馈（智能体系统的关键组件）的接触。为此，在中期训练中加入了涉及外部环境交互的工具调用数据，并引入长程交互轨迹。

2.4 post-train 思路： 采用两阶段后期训练过程：首先是对合成的智能体轨迹进行监督微调（SFT），随后进行智能体强化学习（Agentic RL）。

2.4.1 真实环境中的高质量轨迹合成： REDSearcher 采用了五个真实环境接口，包括：网页搜索、网页访问、Python 代码执行、Google Scholar 和 Google Maps。 采用 ReAct 工作流进行轨迹合成。该范式通过“思考-行动-观测”的迭代循环解决复杂问题。合成时，我们将最大上下文长度设置为 128K，剔除超长样本。

2.4.2 监督微调 (SFT)： 采用标准的下一个 Token 预测损失（Next-token Prediction Loss），同时遮蔽（Masking）环境观测部分，使其不参与梯度更新。SFT 期间的最大上下文长度设为 128K。 2.5.2 强化学习 (RL)： 模型通过 ReAct 范式与环境交互。交互结束后，由 LLM 裁判通过评估智能体预测值与标准答案的一致性来提供一个可验证的奖励（0 或 1）。 RL 算法：我们使用 GRPO 作为训练算法。具体而言，针对每个问题采样一组轨迹，计算最终奖励，并在组内进行归一化处理以获得相对优势（Relative Advantages）。我们使用剪切策略梯度目标（Clipped Policy-gradient Objective）来更新模型参数。

2.5 实现细节优化： 上下文管理：由于基于搜索的智能体轨迹仍然很容易由于以下原因超过模型的最大输入长度：

多次重复的工具调用
冗长的网页内容
以及累积的中间推理笔记 本文采用的方法是：当当前上下文长度超过预设的窗口预算阈值时，会重置上下文中的工具调用历史，只保留问题和以及最小任务说明，再进行 rollout。 本质上，这是通过牺牲长期的上下文记忆能力，来换取更大的剩余 token 预算，从而继续进行探索和工具调用。

sft 数据：(1) 仅保留最终答案正确的样本；(2) 过滤掉包含大量失败动作或工具响应的样本，防止模型学习错误行为；(3) 每个问题仅保留一条轨迹以增加多样性。

格式奖励：RL 没有采用，因为 sft 已经学会了。

三.实验设计与结果分析 3.1.agent 测试方法效果总实验

任务：agent 测试（Humanity’s Last Exam (人类最后的考试)、BrowseComp、BrowseComp-ZH (中文版)、GAIA）
指标：准确率，输出长度（部分）
实验结果： 1.准确率领先：模型取得了 51.3 的总分，大幅领先同规模的领先竞争对手，如 Tongyi DeepResearch-30B (48.5) 和 WebSailorV2-30B (46.0)，也超过了闭源模型比如 Claude-4.5-sonnet (41.1) 和 OpenAI-o3 (49.6)。

3.2 中训练消融的有效性 实验结果：

实验结果： 1.第一阶段（对齐与规划） 侧重于构建基础原子能力。Intent-anchored Grounding 通过增强噪声环境下的信息提取能力，使 BrowseComp 的得分提升了 1.87 分。而层级化规划（Hierarchical Planning）带来了 GAIA 得分的显著飞跃（+4.13），证实了将目标划分为具体和模糊子任务对于复杂推理至关重要。 2.第二阶段 agentic tool 促进了模型从“理解”向“执行”的转变，通过引入环境反馈和长程交互轨迹，我们在 BrowseComp-ZH 上看到了最实质性的增长（+8.91）。这一突破表明，接触真实的“动作-反馈”循环以及 128K 上下文，对于在深度搜索场景中保持目标一致性和稳健执行至关重要。

3.3 强化学习持续提升模型能力

实验结果： 1.性能持续增长：即便是在一个已经非常强劲的 SFT checkpoint 基础上，agentic RL 仍能带来持续的改进。 2.搜索效率的演进：随着 RL 训练的推进，模型的轨迹长度（rollout length）逐渐缩短，而奖励值却保持稳定甚至持续提升。这一现象表明，模型通过 RL 学会了更高效的探索和搜索策略。从定量角度看，平均工具调用次数从 100.6 次下降到 90.1 次，减幅达 10.4%。在轨迹缩短的同时性能并未下降，这说明模型已经学会了识别更精简的任务完成路径，在不牺牲有效性的前提下减少了冗余的工具调用。

3.4 多模态测试方法效果总实验：

任务：多模态测试（ MM-BrowseComp、BrowseComp-VL、MMSearch-Plus、MMSearch 以及 LiveVQA）
指标：准确率
结果： 1.高难度基准表现：在极具挑战性的基准测试（如 MM-BrowseComp）上，方法与最先进的系统（如 Gemini-3-Pro 和 Seed1.8）相比具有极强的竞争力，同时大幅超越了基于 Qwen3-VL-235B 构建的强力智能体基线。 2.多难度覆盖：在相对简单的多模态搜索基准（如 MMSearch 和 LiveVQA）上，本方法保持了卓越的结果，表明其在不同难度级别下都具备稳健的多模态检索与推理能力。 3.通用性验证：在纯文本基准测试上评估了该多模态搜索模型，它依然取得了强劲的表现。这表明模型学到的搜索和推理能力具有良好的迁移性，即便在没有视觉输入的情况下依然有效。

3.5.多模态深度搜索分析 自变量：轮次 因变量：正确和错误的比例 实验结果：

分布差异显著：简单基准通常只需少量轮次即可检索到充分证据并自信作答；而挑战性基准则往往需要更多的搜索轮次。
过度搜索 (Over-searching)：由于缺乏结束推理的信心，模型有时在已经接触到正确证据后仍会继续搜索。
截断效应：这种“过度搜索”在挑战性基准上尤为明显，大量实例集中在 30 轮的截断点附近，表明模型经常会一直搜索直到被强制要求回答。 此外，我们观察到 RL 训练后工具调用轮次有所减少，在简单基准上尤为突出。这归功于 RL 阶段设置的严格轮次限制（20 轮），它促使模型在保持准确性的同时最小化搜索步骤。

现象观察：思维模式 (Thinking Patterns)： 将模型在使用工具时的核心思维模式归纳为三类：

分解 (Decomposition)：将复杂查询拆解为更小的、可操作的子问题，并通过有针对性的工具调用顺序解决。
反思 (Reflection)：重新审视中间结论，识别缺失的证据或不确定性，并据此调整搜索计划。
验证 (Verification)：在最终作答前，通过额外来源或多重证据对候选答案进行交叉核对。
实验结果：思维模式随难度而异：在简单基准上，反思和验证的比例极低。此外，在多模态基准上，模型在推理过程中更有可能将视觉信息纳入考量。

四、核心结论

数据合成创新：针对高质量训练数据稀缺的问题，提出了双约束任务合成方法。该方法能生成结构复杂度高且证据分布分散的推理任务，从而确保了迭代规划和跨文档综合能力的必要性。
训练效率优化：为了降低采集交互轨迹的计算与时间成本，引入了高性价比的中期训练（Mid-training）方案，将原子子技能的获取与交互式执行解耦；并配合功能等效的模拟环境，在不依赖昂贵的实时 API 调用的情况下，实现了高通量的轨迹生成。
能力激活与进化：在此基础上，通过轨迹合成、监督微调（SFT）和智能体强化学习（Agentic RL），进一步提升了模型的搜索智能。
