GPTQ比OBQ核心加速地方在于不用每次量化一个点的weight时候去更新整行的H逆，计算复杂度和内存使用至少低一个量级；

选择同一行的下一个量化weight点的时候是一样的顺序，如果这时候所有行的hession阵是一样的话，这就给并行加速带来了可能；

好巧不巧，论文说了：

待量化二维权重矩阵的所有行对应的Hessian是一样的，因为H=2XX^T( 公式建议看OBQ或者AdaRound论文一部分)，Hessian只与X(本层input)有关和其它weight没关系，所以，量化二维权重矩阵所有行对应的Hessian是一样；

“待量化二维权重矩阵”是我编的词，因为论文名词太缩略不好懂，看代码就知道为什么这样叫；

GPTQ和OBQ代码出自同一team，关于Hessian求解代码是一样的，add_batch这个函数中，注释我拿resnet的第一层卷积举例；

    #根据输入构建weight矩阵的Hessian矩阵
    #什么是权重的二维矩阵呢？这里举例resnet第一层卷积权重(weight)是四维的(64,3,7,7),
    #因为output channel独立量化,把4维变2维时候Oc不变，其它维度融合，于是变成了(64, 3*7*7)
    #根据input要计算这个权重矩阵(64, 147)的Hession阵(64, 147, 147)
    #下面主要逻辑就是要把inp变成(147, X)的形状，然后用inp*inp.t()求出Hessian矩阵
    def add_batch(self, inp, out):
        #假设是resnet第一层卷积:inp.shape(128,3,224,224) out.shape(128,64,112,112)
        if DEBUG:
            self.inp1 = inp
            self.out1 = out
        tmp = inp.shape[0]#input数据的batch=128
        if isinstance(self.layer, nn.Linear):
            if len(inp.shape) == 3:
                inp = inp.reshape((-1, inp.shape[-1]))
            inp = inp.t()
        if isinstance(self.layer, nn.Conv2d):
            unfold = nn.Unfold(
                self.layer.kernel_size,#(7,7)
                dilation=self.layer.dilation,#(1,1)
                padding=self.layer.padding,#(3,3)
                stride=self.layer.stride#(2,2)
            )
            #以下几行逻辑是把layer的输入变形到权重二维阵的输入
            inp = unfold(inp)
            #inp.shape=(128, 147, 12544)=(128, 3*7*7, 112*112)=(batch, 权重二维阵的列, 多少步的输入)
            inp = inp.permute([1, 0, 2])
            inp = inp.flatten(1)#(147, 1605632)
        self.H *= self.nsamples / (self.nsamples + tmp)
        self.nsamples += tmp
        self.H += 2 / self.nsamples * (inp.matmul(inp.t())).double()#参考论文公式
        #self.H的shape是(64,147,147)

首先把weight变形成Oc为行，Oi*K*K为列的二维矩阵，因为Oi与输入不可切割，Oc与输入独立(类比CNN中的perchannel量化)，每个channel可以有独立的量化参数，因为每个channel的输入都是一样完整的；

GPTQ论文中量化步骤对应的代码在函数fasterquant,

首先是用cholesky形式代替表达H逆

        #下面三行是求得H逆的cholesky表达形式
        #cholesky分解前提是H是正定阵，而H=2*XX^T,X向量线性无关(或正交)时候H是正定
        H = torch.linalg.cholesky(H)
        H = torch.cholesky_inverse(H)
        H = torch.linalg.cholesky(H, upper=True)
        Hinv = H#H逆的cholesky表达形式可以确保H^-1是对称正定的

接着就是根据H逆来做量化：

        for i1 in range(0, self.columns, blocksize):#blocksize是128,一次处理128列权重矩阵
            i2 = min(i1 + blocksize, self.columns)

            #拿LLaMa第一层model.layers.0.self_attn.k_proj.weight举例,这层W矩阵是(4096, 4096)
            #因此第一块,W1.shape是(4096，128)
            W1 = W[:, i1:i2].clone()#
            Q1 = torch.zeros_like(W1)
            Err1 = torch.zeros_like(W1)
            Losses1 = torch.zeros_like(W1)
            Hinv1 = Hinv[i1:i2, i1:i2] #对应的Hinv1.shape是(128，128)

            for i in range(count):#cout是128,一列列按顺序执行
                w = W1[:, i]#w.shape是(4096),一次处理4096行
                d = Hinv1[i, i]#diag取对角线值,公式与OBQ一致
                ......
                q = quant.quantize(
                    w.unsqueeze(1), self.quantizer.scale, self.quantizer.zero, self.quantizer.maxq
                ).flatten()
                Q1[:, i] = q
                Losses1[:, i] = (w - q) ** 2 / d ** 2

                err1 = (w - q) / d
                #更新Block里面没有量化的权重(如论文中的图Hinv1[i, i:]多次迭代后呈上三角形)
                W1[:, i:] -= err1.unsqueeze(1).matmul(Hinv1[i, i:].unsqueeze(0))
                Err1[:, i] = err1

            Q[:, i1:i2] = Q1
            Losses[:, i1:i2] = Losses1 / 2
            #更新非Block里面的权重(论文中所谓的lazily batch)
            #只有在一个块被完全处理后，才对整个H^-1和W进行全局更新
            W[:, i2:] -= Err1.matmul(Hinv[i1:i2, i2:])
