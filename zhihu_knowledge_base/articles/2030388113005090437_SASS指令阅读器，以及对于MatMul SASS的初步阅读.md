SASS 阅读器

在进行 cuTile MatMul 实验的时候，发现将 tile 的 size 变大时（变成 tm=256 tn=256 tk=256 的时候），出现了大量的寄存器溢出，经过对 SASS 的检查发现这个时候并没有编译出 UTCHMMA 指令，而是编译出了大量的 HMMA 指令，也就是说开始出现兜底的编译，由于使用 HMMA 需要大量使用寄存器因此出现了寄存器溢出的问题。（具体 tile 是如何定义在这篇文章中：Thor 上 cuTile MatMul 实践与 Cache 异常现象分析）

但是在阅读 SASS 的时候发现代码被分割为大量的 BB 块（在最后会有一个初步的解释），实在是没有办法阅读，于是在 codex 的帮助下写了一个 SASS 阅读器（代码在 HTTPS://GitHub.com/zhao008/cubin_cfg_html），借助.dot 文件生成了一个 HTML 文件，其主要的界面如下图所示：


SASS阅读器界面

它把 nvdisasm -bbcfg -poff 的输出整理成单文件网页，让 basic block 控制流和块内 SASS 可以放在同一个界面里交互式查看。它支持把 cubin 或已有的 DOT 直接转换成可视化页面，左侧是可以拖拽、缩放和自动聚焦的 CFG，右侧则用于查看当前 block 的详细信息和完整代码，并且支持在同一个 cubin 的多个函数之间切换。

使用者可以直接搜索 block 名称，也可以搜索任意指令文本，例如 LDTM.x8、STTM.x8、UTCHMMA、HMMA、STS.128 或 SYNCS，命中的结果会同时在图中和代码区高亮，并且右侧结果列表支持点击跳转。搜索还支持 Regex 和 Opcode Only 两种模式，适合更严格地筛选指令。为了便于长期观察，它还支持把搜索结果固定成不同颜色的持久高亮，也支持对单个 basic block 打勾常亮，这些状态会保存在浏览器本地，刷新页面后仍然可以恢复。

在交互层面，它不只是一个静态查看器。点击任意 basic block 后，可以选择只高亮当前块，或者只看它的上游、下游，或者同时看上下游；如果只想在自己标记过的候选块范围内分析，还可以启用 Pinned Only Path 限制高亮传播范围。右侧还会显示当前块的前驱和后继，方便直接跳转到相邻 block。针对控制流阅读，它对 BRA、CALL、RET、EXIT、BSSY、BSYNC 等分支和同步相关指令做了额外强调。除此之外，它还支持在两个搜索集合之间做 From/To 路径分析，并提供 Swap From/To 按钮方便快速切换方向，更适合拿来分析不同指令集合所在 basic block 之间是否存在执行关系。

对于 MatMul SASS 的阅读

目前尝试使用这个阅读器对于 MatMul 的 SASS 进行阅读，目前主要探索几个重要的指令，分别是 UTCHMMA（第五代 MMA 指令）、STTM 和 LDTM（TMEM 数据搬移指令）、UTMASTG.2D 和 UTMALDG.2D（TMA 数据搬移指令），在 SASS 阅读器中 pin 住这些指令，最终形成的高亮形式如下图所示：

MatMul SASS阅读器使用用例

随后尝试对这些指令进行 cfg 分析，发现只有 STTM 和 LDTM 是在一个执行流中的，并且可以到达 exit BB 块，其余的指令都被分配到了单独的指令流中（每一个指令一条指令流），并且这些指令流都可以到达 exit BB 块，也就是说编译器（tileiras）为了保证计算的高效性在编译阶段会将整个计算过程编译成多个异步计算的过程，然后由同步指令（SYNCS.ARRIVE.TRANS64 和 SYNCS.PHASECHK.TRANS64.TRYWAIT）保证多个异步执行过程的同步。这其实也解释了前面的遗留问题，为什么这里会出现这么多的 BB 块，这也是编译器为了异步执行对于计算过程进行的划分。
