本文设计一个MIPS五级流水线处理器，支持访存指令，算术逻辑指令，转移指令。采用旁路的方式解决了流水线的一般指令的数据冲突，采用阻塞的方式解决了流水线的装载指令造成的冲突。对于控制冒险，我们采用假设分支不发生的方式处理，当发生时再进行阻塞。

所设计的MIPS处理器经过经过流水线冲突验证，再进行AES程序的运行验证。

跳转指令J，Jal，JR的区别
J直接跳转
Jal跳转时还应把PC+8地址存入寄存器31
JR取寄存器的值作为地址

由于Jal跳转时需要写寄存器$31，所以为了避免和写回阶段进行冲突，将Jal的写回也放入WB级（此时寄存器堆写回信号也需要改动），也就是说跳转的判断也需要在WB级给出。同时由于Jal信号的给出在WB级，此时在IF、ID、EX、MEM级都填充了顺序的指令。故当Jal发生时，需要给出信号JALFlush，用于给到指令寄存器进行多选器跳转判断和冲刷IF/ID，ID/EX，EX/MEM，MEM/WB寄存器的信号。

J和JR也是无条件跳转指令。ID级时，J就可以计算出跳转地址，而JR也可以直接得到寄存器的值作为地址。因此在ID级时就可以产生跳转，在产生跳转时给出 JFlush和 JRFlush跳转信号，其作用在于给到IF级的多路选择器判断下一个跳转地址和冲刷ID/IF寄存器的值。

另外，对于条件判断指令bne和beq指令，在MEM级得到结果，因此需要跳转信号PCSrc选择地址和冲刷IF/ID，ID/EX，EX/MEM级的流水线寄存器。

MIPS处理器冲突

流水线的冲突分为三类：结构冲突、数据冲突和控制冲突。结构冲突（structural hazard）主要是硬件资源不够导致的，只需要增加硬件资源即可。结构冲突比较简单比较简单，这里讨论数据冲突（data hazard）和控制冲突（control hazard）。

数据冲突

数据冲突也称流水线数据冒险（pipeline data hazard），即因无法提供指令执行所需数据而导致指令不能在预定的时钟周期内执行的情况。一般采用的解决方式位前推（forwarding）或者旁路（bypassing）的方法解决。

数据冲突主要分为几种，一是当EX级需要Rs或者Rd寄存器数据时，所需要的最新结果还在MEM级或者WB级；二是可能发生对寄存器堆同时读写的情况；三是对于取数-使用型数据冒险（load-use data hazard），由于需要等待数据从dcache中取出，下一条指令不得不阻塞一个步骤。

对于第一种，即当EX级需要Rs或者Rd寄存器数据时，所需要的最新结果还在MEM级或者WB级。在EX级采用“旁路单元”来进行检测，当检测到需要写寄存器，且EX级进去的寄存器地址等于EX/MEM出来的寄存器地址，或者EX级进去的寄存器地址等于MEM/WB出来的寄存器地址时，就发生了这种冲突。并且对于不同的情况，输出不同的值，以控制ALU输入的多选器，使得ALU输入正确的值。

多选器控制	ALU输入来源	解释
Forward A = 00	ID/EX	第一个ALU操作数来自寄存器堆
Forward A = 10	EX/MEM	第一个ALU操作数由上一个ALU运算结果旁路获得
Forward A = 01	MEM/WB	第一个ALU操作数从数据存储器或者前面的ALU结果旁路中获得
Forward B = 00	ID/EX	第二个ALU操作数来自寄存器堆
Forward B = 10	EX/MEM	第二个ALU操作数由上一个ALU运算结果旁路获得
Forward B = 01	MEM/WB	第二个ALU操作数从数据存储器或者前面的ALU结果旁路中获得

控制条件如下：

wire EXDectA,MEMDectA,EXDectB,MEMDectB;
// forward A
assign EXDectA = EX_MEM_RegWrite                               // EX冒险条件
                & (rd_from_EX_MEM!=5'd0)
                & (rd_from_EX_MEM == rs_from_ID_EX);
assign MEMDectA = MEM_WB_RegWrite                              // MEM冒险条件
                & (rd_from_MEM_WB != 5'd0)
                & (~EXDectA) 
                &(rd_from_MEM_WB==rs_from_ID_EX);
// forward B
assign EXDectB = EX_MEM_RegWrite                               // EX冒险条件
                & (rd_from_EX_MEM!=5'd0)
                & (rd_from_EX_MEM == rt_from_ID_EX);
assign MEMDectB = MEM_WB_RegWrite                              // MEM冒险条件
                & (rd_from_MEM_WB != 5'd0)
                & (~EXDectB) 
                &(rd_from_MEM_WB==rt_from_ID_EX);

always@(*)begin
    if(EXDectA)begin
           ForwardA <= 2'b10;
    end
    else if(MEMDectA)begin
        ForwardA <= 2'b01;
    end 
    else ForwardA <= 2'b00;
end

always@(*)begin
    if(EXDectB)begin
           ForwardB <= 2'b10;
    end
    else if(MEMDectB)begin
        ForwardB <= 2'b01;
    end 
    else ForwardB <= 2'b00;
end


对于第二种，发生对寄存器堆同时读写的情况。在寄存器堆中进行判断，当读地址等于写地址时，除了写入相应寄存器之外，还直接把输入数据给到输出。

assign ReadData_rs = ((RegWrite==1)&(WriteAddr_rd==ReadAddr_rs))? WriteData:RegRam[ReadAddr_rs];
assign ReadData_rt = ((RegWrite==1)&(WriteAddr_rd==ReadAddr_rt))? WriteData:RegRam[ReadAddr_rt];


对于第三种，取数-使用型数据冒险。在ID级加入了“冒险检测单元”，如果发生了取数-使用型数据冒险，冒险检测单元就会阻塞PC和IF/ID寄存器并清除所有的控制字段。判断条件如下，当ID/EX有读dcache并且写入寄存器的地址和之后一条指令的地址（此时该地址在IF/ID）相等时，就会发生：

assign Stall = MeMRead_from_ID_EX
             &((rt_from_ID_EX==rs_from_IF_ID)|(rt_from_ID_EX==rt_from_IF_ID));

assign IFIDWait = (Stall)? 1'b1:1'b0;
assign PCwait = (Stall)?   1'b1:1'b0;     
assign Detect_to_Control = (Stall)? 1'b1:1'b0;

控制冲突

控制冲突也称控制冒险，分支冒险（branch hazard）。因为取到的指令并不是所需要的（或者说指令地址的变化并不是流水线所预期的）而导致指令不能在预定的时钟周期内执行。

控制冲突采用假设跳转不发生（针对条件判断指令），当发生时再进行流水线冲刷。如前面在指令介绍中提到的，对于不同的地址其判断出结果所在的流水线级数不同。对于J和Jr，可以在ID级就得到跳转地址；对于Jal，为了能写$31寄存器，需要在WB级才能跳转。对于条件跳转指令bne和beq，在EX级才能给出跳转结果。

在不同级判断出的跳转发生，还需要冲刷掉之前流水线已加载的指令和控制信号，并给到取值阶段的多选器以选择跳转地址。

整体框架

下图是本MIPS处理器的结构图，其中红色线代表控制信号，其他线代表数据。

数据通路
IF模块

​IF级包括指令寄存器，PC寄存器，加法器和多选器。指令寄存器存储指令，根据PC值输出指令。加法器计算PC+4，使得下一时钟可以执行下一指令。多选器用于不同跳转指令的地址跳转，从左到右依次为Jr，J，bne和beq，Jal。它们的顺序是根据判断结果在寄存器中的产生级数确定的。比如若在EX级和WB级同时发生跳转信号，因为WB级的指令”更老“，因此它的优先级也应该更高。

ID模块

​
ID指令包括冒险检测，控制单元，地址计算部分，寄存器堆和符号扩展。冒险检测前面已经介绍，这里不再赘述。控制单元是根据指令输出所需要的控制信号。地址计算部分是为跳转指令J和Jal计算地址，将指令低26位左移两位，再将PC+4的高四位拼接到地址高位上，得到32位地址。寄存器堆Regs是32个32位寄存器它的写入来自于WB级，同时它的Rs读数据也指向IF级作为可能跳转的地址。符号扩展是将16位立即数扩展到32位。对于计算型的立即数（如addi），是将最高位进行扩展，对于逻辑类立即数（如andi），是使用0扩展。

EX模块

​
EX级包括条件跳转的地址计算，PC+8地址计算，ALU单元，写回地址的多选器以及旁路单元。条件跳转的地址是将扩展的立即数ijnx左移2位如何和PC+4进行相加得到。PC+8的值是为了当Jal发生时可以存入寄存器$31中。Mux根据控制信号选定之后写回寄存器的地址值时哪一个字段。ALU输入的多选器用来选择输入ALU的值是立即数还是来自于寄存器堆，又或者来自于发生数据冲突时的值。ALU在后面详细介绍。

MEM模块

​
MEM级包括dcache和Pcsrc生成部分。数据存储器即用来存储数据。Pcsrc用来产生条件跳转信号。它根据ALU结果是否为0和条件类型进行判断，如下：

assign Pcscr = (bne_EX_MEM &(~zero_EX_MEM))|(beq_EX_MEM & zero_EX_MEM);


当发生bne且非零，或者beq且为零是信号有效。

WB模块

​WB级即写回。包括写寄存器信号的产生，写回数据和写回地址的选择。写信号的逻辑还包括的add，sub等的特殊判断（这里overflow只有相关指令时溢出时才为1，其他时候或者无关指令都为0），溢出发生时则写入信号失效。当Jal发生时，还控制多选器将写回地址设为31。

ALU

EX级中的ALU（arithmetic logic unit），算术逻辑单元，包括ALU控制和ALU计算部分。ALU控制根据输入的指令funct字段和来自控制信号的ALUop控制决定ALU执行何种操作，包括与，或，或非，异或，左移，右移，加，减，小于则置位以及lui置高位。

列混淆涉及到域里的乘法，会麻烦一些。在直接给出列混淆的汇编之前，我们先了解一下有限域GF(28)的有限域乘法。

有限域乘法

有限域说白了就是一个拥有有限个元素的集合，并且其满足一些性质。在有限域内的计算，所得到的结果都在有限域内，如果超出了有限域则需要模上有限域的不可约多项式。对于有限域内的加法而言，它的操作就是异或操作，并且不会超出有限域的范围。但是对于乘法而言就可能超出范围，AES加密算法采用以m(x)=x8+x4+x3+x+1，在域中表示位(100011011)。而对于域中元素a=(a7a6a5a4a3a2a1a0)，a∈{0,1}来说，它的多项式表示为a(x)=a7x7+a6x6+a5x5+a4x4+a3x3+a2x2+a1x1+a0x0。它和x（也可以说成是{00000010}的多项式表示）的乘法公式如下：

x⋅a(x)=(a7x8+a6x7+a5x6+a4x5+a3x4+a2x3+a1x2+a0x) mod m(x)={a6x7+a5x6+a4x5+a3x4+a2x3+a1x2+a0xa6x7+a5x6+a4x5+a3x4+a2x3+a1x2+a0x+x4+x3+x+1a7=0a7=1

简单的说，就是若8bit元素a的最高位为1，那它和x相乘就是左移一位再和(00011011)异或，否则，只需要移位即可。而a与其他多项式相乘则可以分解为多次和x相乘再求和。

在列混淆的矩阵乘中，乘的元素只有三个，{1,2,3}。对于1相乘没有变化；2（即x)相乘则是上面的公式；而对于3，可以拆分成2和1相乘后的求和（实际上，对于3的系数表示是（011），也就是x+1)




列混淆处理
void mixcom(unsigned char a[4][4]){
    unsigned char i, j;
    unsigned char s[4][4];    
    for(i=0; i<4; i++){
        for(j=0; j<4; j++){
            s[i][j] = a[i][j];
        }
    }
    for(j=0; j<4; j++){
        a[0][j] = gf_mul(0x02, s[0][j])^gf_mul(0x03, s[1][j])^s[2][j]^s[3][j];
        a[1][j] = s[0][j]^gf_mul(0x02, s[1][j])^gf_mul(0x03, s[2][j])^s[3][j];
        a[2][j] = s[0][j]^s[1][j]^gf_mul(0x02, s[2][j])^gf_mul(0x03, s[3][j]);
        a[3][j] = gf_mul(0x03, s[0][j])^s[1][j]^s[2][j]^gf_mul(0x02, s[3][j]);
    }
}
