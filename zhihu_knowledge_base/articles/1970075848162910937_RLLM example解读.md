最近在看agent相关的领域，因为我实习的时候大量接触了verl而且做了很多相关项目开发，所以我希望找到一个verl 作为基础的agent训练框架，于是我找到了RLLMhttps://github.com/rllm-org/rllm。因为对agent领域还不太熟悉，所以希望能通过快速查看RLLM example，了解现在agent的大致类型，训练集和一些benchmark。

只评测类：

Appworld：https://github.com/rllm-org/rllm/blob/main/examples/appworld

评测体系，appworld benchmark的。




demo类别：

SmolAgents：展示了如何使用 SmolAgents 的 AsyncCodeAgent，让语言模型（如 Qwen3-0.6B）自动 生成并运行 Python 代码 来完成任务




带工具训练（加评测）类：

countdown ：借助verl实现countdown数据集的训练。（一个 数字列表（例如 [15, 68, 5, 34]）， 一个 目标值（例如 22）， 模型需要输出一个 表达式，只用这些数字（每个最多一次）以及 +/−/×/÷ 来达到目标。）

deepcoder：下载deepcoder的数据集+从16k-64k deepcoder的训练。

deepscaler：DeepScaleR-Math训练+aime24评测； （qwen2.5）

math-tool：DeepScaleR-Math训练+aime24评测；（qwen3，应该就是格式不同）

eval_protocol：frozenlake训练+测试

Frozen Lake：在Frozen Lake（传统RL任务）训练+导航

MiniWoB：BrowserGym-MiniWoB++训练+测试

VimGolf ：VimGolf Public Challenges 训练




agent 调用/工具提供类：

deepresearch：实现了：1.通义的agent调用方式；2.实现agent 的工作流；3.tool：搜索，执行器，文件解读，日历；

MCP：实现了：使用 Tavily MCP 服务器运行 HotpotQA 问答评估，以实现 Web 搜索功能

search：Wikipedia corpus作为数据集，E5做embedding，Dense index做向量索引，采用rllm搜索

strands：传统 Strands SDK Strands 自身不保存强化学习需要的全部轨迹数据（如模型调用内容、工具调用细节等）。这里可以做完整保存。

terminal：终端工具




paper类：

swe：使用R2E-Gym进行训练，在swebenchVerified测试。
