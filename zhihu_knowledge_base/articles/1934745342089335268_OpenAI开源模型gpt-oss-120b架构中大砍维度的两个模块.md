其二，为什么 gpt-oss-120b 的 intermediate size = hidden size = 2880？

主流的模型，intermediate size 通常都是hidden size的2.5-4倍，MoE模型的 hidden size 通常也在1.5-2.5倍。这么设置的原因在这篇知乎回答讲得很好，使用像ReLU的函数时，0值可能会使矩阵降秩，为了使得 intermediate hidden state 的“秩”低于 hidden size 的概率小，就需要将 intermediate size 放大到 hidden size 的数倍。（主流模型选择的激活函数是SwiGLU，也即 intermediate hidden state 由 `y = SiLU(W_gate(x)) * W_up(x)` 算出，其中SiLU函数与ReLU函数类似，这会导致过完SwiGLU后 hidden state 值接近0，降低有效秩，道理同ReLU类似）。

而在gpt-oss-120b中，intermediate size 却只与 hidden size 相等。单看一个专家是找不出原因的，但是一旦考虑到它的激活专家数为4，一切便豁然开朗。4个专家其实可以完全等价地拼成一个大FFN，推导如下：

经过等价变换，模型实质上的 intermediate size 便是 hidden size 的4倍。这个分析大致给出了MoE模型中 intermediate size 可以被压缩的一个上限（不牺牲秩的前提下），至于为什么现在很多MoE模型的 intermediate size 远未逼近这一上限，是效果不佳，还是该方向探索较少，还有待各位大佬的解答。
