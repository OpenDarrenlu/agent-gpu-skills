4. 头文件
4.1 头文件的作用
把需要使用的函数声明在头文件中
其他代码在使用这个函数的时候，只需要引用头文件即可，不需要再次声明这文件告诉编译器我需要使用这个文件
4.2 头文件的搜索路径
#include <cstdio> // 表示不再当前目录进行搜索

#include "hello.h" // 优先搜索当前目录

5. 子模块

add_subdirectory 是 CMake 中的一个命令，用于向当前项目添加子目录。这个命令非常有用，特别是在处理大型项目时，因为它允许您将项目组织成较小、更易于管理的部分

5.1 目录结构
# /project
#    |--- CMakeLists.txt
#    |--- main.cpp
#    |--- /hellolib
#         |--- /hello.h
#         |--- /hello.cpp
#         |--- CMakeLists.txt

5.2 hellolib/CMakeLists.txt
add_library(hellolib STATIC hello.cpp)

#可以直接在子模块中
target_include_directories(hellolib PUBLIC .)


target_include_directories(hellolib PUBLIC .)：

target_include_directories 用于为hellolib指定 include 目录。include 目录是编译器查找头文件的地方。
PUBLIC 关键字指定了这些包含目录应该被该目标及其任何链接到该目标的其他目标使用。这意味着任何链接到hellolib的目标也将自动获得对这些包含目录的访问权限。
如果不希望让引用 hellolib 的可执行文件自动添加这个路径，把 PUBLIC 改成 PRIVATE 即可。这就是他们的用途：决定一个属性要不要在被 link 的时候传播。
5.3 project/CMakeLists.txt
# 在主项目的 CMakeLists.txt 中
cmake_minimum_required(VERSION 3.10)
project(MyProject)

# 添加子目录
add_subdirectory(hellolib)

add_executable(a.out main.cpp)

# 链接子模块的动态库
target_link_libraries(a.out PUBLIC hellolib)
# 处理子模块的头文件(如果子模块中作了处理，project中则不需要处理）
target_include_directies(a.out PUBLIC hellolib) // hellolib 是目录

6. Cmake 其他命令解析
target_include_directories(myapp PUBLIC /usr/include/eigen3)
这个命令用于向 myapp 目标添加一个包含目录。包含目录是编译器在查找头文件时会检查的目录。
PUBLIC 关键字表示这个包含目录不仅对 myapp 可见，也对任何链接到 myapp 的其他目标可见。
/usr/include/eigen3 是添加的包含目录的路径，此处是 Eigen3 数学库的标准安装路径。
target_link_libraries(myapp PUBLIC hellolib)
用于指定 myapp 在链接时需要链接的库。这里指定了 hellolib 库。
PUBLIC 表示这个链接依赖关系也会传递给链接到 myapp 的任何其他目标。
target_compile_options(myapp PUBLIC -fopenmp)
为 myapp 添加特定的编译选项。这里的 -fopenmp 是告诉编译器开启对 OpenMP 的支持，允许并行编程。
PUBLIC 指这个编译选项也会应用到任何链接到 myapp 的其他目标。
target_sources(myapp PUBLIC hello.cpp other.cpp)
用于为 myapp 目标添加额外的源文件。这里添加了 hello.cpp 和 other.cpp 两个文件。
7. 第三方库
7.1 纯头文件第三方库
优点：
只需要把他们的 include 目录或头文件下载下来，然后 include_directories(spdlog/include) 即可。
缺点：
函数直接实现在头文件里，没有提前编译，从而需要重复编译同样内容，编译时间长。

https://github.com/nothings/stb

stb_image 系列，涵盖图像，声音，字体等，只需单头文件！

https://github.com/Neargye/magic_enum

枚举类型的反射，如枚举转字符串等（实现方式很巧妙）

https://github.com/g-truc/glm

模仿 GLSL 语法的数学矢量/矩阵库（附带一些常用函数，随机数生成等）

https://github.com/Tencent/rapidjson

单纯的 JSON 库，甚至没依赖 STL（可定制性高，工程美学经典）

https://github.com/ericniebler/range-v3

C++20 ranges 库就是受到他启发（完全是头文件组成）

https://github.com/fmtlib/fmt

格式化库，提供 std::format 的替代品（需要 -DFMT_HEADER_ONLY）
7.2 作为子模块引入
使用方法
把第三方库的代码放到工程的根目录，作为子模块取引入
# 在主项目的 CMakeLists.txt 中
cmake_minimum_required(VERSION 3.10)
project(MyProject)

# 添加子目录
add_subdirectory(hellolib)

add_executable(a.out main.cpp)
target_link_libraies(a.out PUBLIC fmt)

7.3 引用系统中预安装的第三方库
find_package(fmt REQUIRED)
target_link_libraries(myexec PUBLIC fmt::fmt) // 指定组件

find_package(TBB REQUIRED COMPONENTS tbb tbbmalloc REQUIRED)
target_link_libraries(myexec PUBLIC TBB::tbb TBB::tbbmalloc)
