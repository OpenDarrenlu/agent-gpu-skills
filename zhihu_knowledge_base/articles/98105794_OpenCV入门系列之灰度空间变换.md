本篇博文主要介绍灰度空间变换，该技术是图像处理中最基础，最简单的方法，但是也是应用最广泛的方法。灰度空间变换属于一种点处理技术，与之对应的滤波则是一种邻域处理技术。所以灰度空间处理的抽象化函数很简单，如下：

s = T(r)


其中，r是原始图像的像素，s是经过T处理后的像素。

以上是直方图均衡化，直方图均衡化是一种自动化的不需要更多的参数可以完成的图像增强技术，需要自动增强的时候，这是一种好的方法；但是其缺点也恰恰在此，直方图均衡化是自动增强整幅图像的对比度,具体增强效果不易控制。因此在现实使用中也存在这样的需求，我们希望处理后的图像具有规定的直方图形状，这个很好理解，就是我们给出一种直方图的形状，希望按照某种变化后，图像的直方图就是这样的。所以接下来我们介绍另外一种直方图处理技术，直方图规定化。

A均衡化以后的灰度级分布：
s0 = 1 s2 = 5 s4 = 6 s6 = 7
s1 = 3 s3 = 6 s5 = 7 s7 = 7

B均衡化后的灰度级分布:

G(z0) = 0 G(z2) = 0 G(z4) = 2 G(z6) = 6
G(z1) = 0 G(z3) = 1 G(z5) = 5 G(z7) = 7


对照以上的灰度级分布，G(zq)的数据分布并不是严格单调，给定一个sk可以找到对应的G(zq);可以得到：

G(z3) = s0; G(z5) = s2; G(z7) = s5;
G(z4) = s1; G(z6) = s3;


找到了映射关系，但是我们知道这是z与s的映射关系；s是由r映射过来的；换句话说，A的真实灰度级是r，r经过均衡化映射为s，s经过上述关系G映射为z；而我们真正需要的是r与z的映射关系，这样就相当于我们找到了A到B的映射关系T，使B能够满足给定的直方图分布。

s与r的映射关系如下：

s0 = r1; s2 = r5; s4 = r6; s6 = r7;

s1 = r3; s3 = r6; s5 = r7; s7 = r7;


所以递推出z与r的映射关系:

z3 -> r1; z5 -> r5; z7 -> r7;
z4 -> r3; z6 -> r6;

只需要将A中rk像素替换为zq像素的值，即可得到B，而B的直方图便是按照我们规定的直方图排布的。如果有兴趣的读者仔细回推，B的直方图分布其实不是完全按照我们给定的pz(zq)分布的；但是是近似的，很难完全吻合，只是一种匹配；所以直方图规定化，也叫作直方图匹配。最后再总结一次，计算过程是我们根据已知的A的直方图分布求出s，根据规定的直方图分布，求出B的直方图分布z；然后列出r->s的映射关系和s->z的映射关系，直接对应出r->z的映射关系。由A求出B。

以上介绍的两种方法都是根据整幅图像进行的直方图处理，是对整幅图进行直方图统计和处理，虽然这种全局的方法适用于整幅图像的增强，但存在增强图像中小区域细节的需要，在这样的区域中，一些像素的影响在全局变换的计算中可能被忽略，因为全局变换没有必要保证期望的局部增强，解决办法便是以图像中每个像素的邻域中的灰度分布作为基础，设计变换函数。接下来介绍局部直方图处理。

局部直方图处理

局部直方图处理可以理解为直方图处理的特殊形式，变换函数设计与之前介绍的方法相同；区别在与之前图像是以整幅图像作为基础，而现在以每个像素的邻域做为基础，对邻域范围进行处理。需要关注的是每个像素以邻域作为处理，后一个像素需要以前一个像素更新以后的值作为邻域处理的基础。另外也可以将处理的步长设置为与邻域半径相等，这样各个像素的邻域不重叠，但是很容易出现棋盘效应。

直方图API
关于直方图的API OpenCV提供有多个，这里介绍两个最常用的；一个是直方图均衡化，一个是计算直方图。如下：
void cv::calcHist(const Mat* images,
                  int nimages,
                  const int* channels,
                  InputArray mask,
                  OutputArray hist,
                  int dims,
                  const int* histSize,
                  const float** ranges,
                  bool 	uniform = true,
                  bool 	accumulate = false)	


-- images: 要计算直方图的图像；支持多个图同时输入；
-- nimages: 输入图像的数量
-- channels: 图像的通道数，这里是一个数组，如果是gray图像，单通道的，就是channles[1] = {0};如果是是rgb图像需要计算三个通道的，则是channels[3] = {0, 1, 2};如果只求第三通道，则是channels[1] = 2；
-- mask: 适用于遮挡图像，确定哪些点参与计算的mask矩阵，实际应用中作用比较大；实验中使用空矩阵Mat();
-- hist: 直方图的输出图像。试验中给出了如何确认数据类型及数量的方法。
-- dims: 得到直方图的维数，gray图像是1维，rgb图像是3维；
-- histSize: 直方图坐标的区间数，如果是16，则会横坐标分为16份，然后统计每个区间的像素点总和；
-- ranges: 二维数组，用于指定每个区间的范围。
-- 后两个参数一般使用默认值，uniform参数表明直方图是否等距，最后一个参数与多图下直方图的显示和存储有关。

#include <iostream>
#include "opencv2/imgproc.hpp"
#include "opencv2/imgcodecs.hpp"
#include "opencv2/highgui.hpp"

int main(int argc, char* argv[])
{
    std::cout << "\nGray space transform demo \n"
                "----------------------------- \n"
                "./run param [picture]\n"
                "[0]: black and white reverse\n"
                "[1]: histogram equalize \n" << std::endl;
    bool bwr = argc >= 2 ? (atoi(argv[1]) == 0 ? true : false) : false;
    bool eqh = argc >= 2 ? (atoi(argv[1]) == 1 ? true : false) : false;
    bool is_has_input = argc == 3 ? true : false;

    //
    std::string filename;
    cv::Mat src = cv::imread(is_has_input ? cv::samples::findFile(argv[2]) : "gray_transform.png", cv::IMREAD_GRAYSCALE);
    if(src.empty())
    {
        fprintf(stderr, "Error can't open input image\n");
        fprintf(stderr, "Program Arguments:[histogram 0 picture.png] \n");
        return EXIT_FAILURE;
    }

    if(bwr == false && eqh == false)
    {
        fprintf(stderr, "Error cant't set param, don't know run [black and white reverse] or [histogram equalize]");
        fprintf(stderr, "Program Arguments:[histogram 0 picture.png] \n");
        return EXIT_FAILURE;
    }

    const char* window_src = "The original image";
    cv::imshow( window_src, src );

    if(bwr)
    {
        cv::Mat dst(src.size(), src.type());
        for(int row = 0; row < src.rows; row++)
        {
            for(int col = 0; col < src.cols; col++)
            {
                dst.at<uchar>(row, col) = 255 - src.at<uchar>(row, col);
            }
        }

        const char* windows_bwr = "black and white reverse";
        cv::imshow( windows_bwr, dst );
    }

    if(eqh){
        cv::Mat dst;
        cv::equalizeHist(src, dst);
        const char* windows_eqh = "histogram equalize";
        cv::imshow(windows_eqh, dst);

        // calculates this histogram
        cv::Mat src_hist;
        int histSize = 256;
        float sranges[] = {0, 256};
        const float* range= {sranges};
        cv::calcHist(&src, 1, 0, cv::Mat(), src_hist, 1, &histSize, &range, true, false);
        const char* windows_hist = "histogram src";
        fprintf(stderr, "src histogram: \n");
        
        int elem_type = src_hist.type();
        int elem_size = src_hist.elemSize();
        float total = 0;
        for(int row = 0; row < src_hist.rows; row++)
        {
            for(int col = 0; col < src_hist.cols; col++)
            {
                float val = src_hist.at<float>(row, col);
                // printf( "%d ", (int)val);
                // if((int)(row * src_hist.cols + 1) % 16 == 0) printf("\n");
                total += val;
            }
        }
        fprintf(stderr, "\n");
        if(total == src.cols * src.rows) printf("Success\n");
        else printf("total: %f\n", total);
    }
    cv::waitKey(0);
    return 0;
}
