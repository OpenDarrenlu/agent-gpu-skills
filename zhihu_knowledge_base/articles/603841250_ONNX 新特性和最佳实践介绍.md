最近在 OpenMMLab 社区开放麦做了一次分享，介绍了 ONNX 最近几个版本中的一些新特性，如

ONNX Parser，用文本格式构造 ONNX 图，和helper.make_node 说再见
Function OP，也就是 ONNX 中的多级表示，推广开之后可大大降低后端适配 ONNX 的工作量
Reference Runtime，ONNX 自带的后端，只依赖 Python 和 NumPy，可用于简单的调试和验证

和 ONNX 生态中的一些周边工具，如

onnxsim，自动简化 ONNX 模型，应该是大家都很熟悉的工具了
onnx-graphsurgeon，手动修改 ONNX 模型
onnx-modifier，可视化地修改 ONNX 模型
PPQ，支持 ONNX 模型的量化工具
convertmodel.com，基于 WASM 在浏览器内将 ONNX 模型转为 NCNN/TNN/... 等
