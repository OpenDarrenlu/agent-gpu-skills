众所周知，在强化学习训练中的关键环节就是奖励信号的获取，准确的奖励信号对于训练的效果至关重要。在经典RL 中，奖励信号可以看作环境的一部分 —— 即行动后环境的真实反馈，而在 RL 训练 LLM 中，奖励值的来源主要有两种方式：

批判式：即 RLHF 中的 RM，该方式给出的是一个连续的标量值，但是由于 OOD 的问题，在一些样本上可能结果并不准确；
验证式：即 RLVR 中的 Verifier, 通过与预设的答案或规则相比较，给出一个二元值，这种方式仅适用于有标准答案的场景，而在开放问题中则不太适用

而无论是以上的哪种情况，其奖励值都是旁证式的、外在的，那么在 LLM 与上下文构成的环境中，是否存在内在的奖励呢？即谜底是否存在在谜面上呢？本文将围绕该问题展开讨论，具体问题包括：

为什么会有内在奖励，其底层逻辑和原理是什么？
内在奖励的形式有哪些，受哪些因素影响？
内在奖励的准确度如何，与外在奖励相比怎样？
一、隐式奖励角度

在 LLM 后训练领域主要存在两种方法论途径：

第一种途径是SFT，通过专家演示学习，或模仿学习， 使用正样本来训练模型。
第二种途径侧重于从环境信号中学习，主要通过强化学习方法 (PPO, GRPO 等)

在两种路径之间，还有一种特殊的方法 DPO，一方面相较于 SFT，DPO 增加了负样本和隐式奖励学习；另一方面相较于 RL，DPO 的训练数据又是完全 off-policy 的， 且省略了显式的奖励信号的训练过程, 下面我们尝试一探究竟。

1.1 DPO 及其隐式奖励

事实上，最早提出隐式奖励的工作就是 DPO 了，接下来我们还是不厌其烦地梳理一下其中的核心过程及要点。

RLHF 的损失可表示为： \max _{\pi_\theta}\left\{\mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta(y \mid x)}\left[r_\phi(x, y)-\beta \log \left(\frac{\pi_\theta(y \mid x)}{\pi_{\text {ref }}(y \mid x)}\right)\right]\right\}\\ 提取常数​ -\frac{1}{\beta} 并应用恒等变换，有： \begin{align} & \min _{\pi_\theta}\left\{\mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta(y \mid x)}\left[\log \frac{\pi_\theta(y \mid x)}{\pi_{\mathrm{ref}}(y \mid x)}-\frac{1}{\beta} r_\phi(x, y)\right]\right\} \\ &=\min _{\pi_\theta}\left\{\mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta(y \mid x)}\left[\log \frac{\pi_\theta(y \mid x)}{\pi_{\text {ref }}(y \mid x)}-\log e^{r_\phi(x, y) / \beta}\right]\right\} \\ &=\min _{\pi_\theta}\left\{\mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta(y \mid x)}\left[\log \frac{\pi_\theta(y \mid x)}{\pi_{\mathrm{ref}}(y \mid x) \exp \left(r_\phi(x, y) / \beta\right)}\right]\right\} \end{align} \\ 这样的话实际上构造出来了一个新的分布，即： \pi_r(y \mid x)=\frac{\pi_{\mathrm{ref}}(y \mid x) e^{r_\phi(x, y) / \beta}}{Z(x)}\\ 其中 Z(x) 是一个归一化常数，确保 \pi_r 是一个合适的概率分布（即概率之和为 1）: Z(x)=\sum_y \pi_{\mathrm{ref}}(y \mid x) \exp \left(\frac{r_\phi(x, y)}{\beta}\right)\\虽然这个针对 \pi_r 的解决方案本身是不可解的，但它可以用来将奖励表示为最优策略的函数，即得到最优策略后就可以反解出奖励模型，该解只需要对上述公式稍加变换即可： r_\phi(x, y)=\beta \log \frac{\pi^*(y \mid x)}{\pi_{\mathrm{ref}}(y \mid x)}+\beta \log Z(x)\\ 这样，从定义整体对齐目标，到推导最优策略，最终将奖励模型与对齐模型连接起来，而不需要直接训练奖励模型。

1.2 理解隐式奖励

上文中，我们得到了奖励与最优模型的基本关系，在此基础上有很多工作进行了更多细化，在此不加证明地介绍引文 [8] 中的结论，即 r(x, y)=\beta \log \frac{\pi^*(y \mid x)}{\pi_{ref}(y \mid x)}+V^*\left(s_0\right)-V^*\left(s_t\right)\\ 其中 r⁢(x,y) 代表 LLM 的输入输出对 (x,y) 的奖励， \pi^∗⁢(a∣s) 是最优策略， V^∗ 是最优策略的价值函数。

初始奖励 r 可以被视为 V_{\pi}⁢(s_0) ，对所有 V_{\pi}⁢(s_t)=0 适用，在此假设下，初始策略 \pi 是相对于初始奖励的最优策略。稍微对上式进行变化可得到 SFT 形式的目标，即： \max _\pi \mathbb{E}_{\mu_E}[\beta \underbrace{\log \pi(y \mid x)}_{\text {MLE }}-\underbrace{\log \pi_{\text {ref }}(y \mid x)-V_\pi\left(s_t\right)}_{\text {Serve as Constants }}]\\ 由此可知，SFT 过程沿着最优策略-奖励子空间进行搜索，试图对专家演示中隐含的奖励进行建模。在优化过程中，模型迭代的过程，也即策略-奖励的最优子空间的搜索过程。

采用该方案进行 RL 训练的典型案例即 PRIME，即通过以下方式获得过程奖励： r_\phi^t:=q_\phi^t-q_\phi^{t-1}=\beta \log \frac{\pi_\phi\left(y_t \mid \mathbf{y}_{<t}\right)}{\pi_{\mathrm{ref}}\left(y_t \mid \mathbf{y}_{<t}\right)}\\这样的好处是显而易见的，即可以通过收集response-level 数据并训练一个 ORM 来简单地获得 PRMs，而无需高成本的数据标注。

值得一提的是 ORM 的损失函数，与正常的 ORM 训练一样，唯一的不同是将 r_\phi(\mathbf{y}) 替换为 \beta \log \frac{\pi_\phi\left( \mathbf{y}\right)}{\pi_{\mathrm{ref}}\left( \mathbf{y}\right)} ,即 \mathcal{L}_{C E}=l \cdot \log \sigma\left(\beta \log \frac{\pi_\phi(\mathbf{y})}{\pi_{\mathrm{ref}}(\mathbf{y})}\right)+(1-l) \cdot \log \left[1-\sigma\left(\beta \log \frac{\pi_\phi(\mathbf{y})}{\pi_{\mathrm{ref}}(\mathbf{y})}\right)\right]\\

在训练过程中，策略模型和 PRM 都使用 SFT 模型进行初始化。在每个 RL 迭代中，策略模型首先生成 rollouts。然后，隐式 PRM 和结果验证器对 rollouts 进行评分，隐式 PRM 在 rollouts 上使用结果奖励进行更新。最后，结果奖励 r_o 和过程奖励 r_p ​ 被结合并用于更新策略模型。

总结一下，此类隐式奖励是通过 策略模型或者 impolicity PRM 与 SFT 相比较产生的，其奖励的是相比 SFT 获得更高分数的 token，换句话说，该方式会加强 SFT 中的行为，使得在最终模型中的概率分布更加集中。

二、逆强化学习角度

在经典的使用 RL 训练 LLM 的流程中，其过程是：SFT -> RM (verifier) -> PPO (GRPO) , 当缺少 rule-based verifier 时，该流程关键依赖于在人类偏好上训练的奖励模型（RM）来评分模型输出，最终对齐的 LLM 的质量基本上由该 RM 的质量决定。而训练 RM 需要构建大规模、高质量的人类偏好数据集，这是缓慢、昂贵且难以扩展的。

那么如果不去构造偏好数据集是否还可以得到准确的奖励信号呢？在上节中，我们提供了一种基于DPO 方式的隐式奖励，本节中我们将尝试通过另一个角度，即逆强化学习的方式来审视这一过程。

2.1 逆强化学习原理与 LLM 奖励信号建模

RL 是给定交互环境与奖励函数 r 后求解一个最优策略 \pi^* 使得期望累积奖励最大化。而逆强化学习(Inverse Reinforcement Learning, IRL) 则相反：假设我们有一批采集于专家策略 \pi_E (可看作最佳策略) 的离线轨迹数据 \mathcal{D}_E, 希望找到一个奖励函数 r_E 来解释数据中的行为，即认为专家策略 \pi_E 的生成是通过一个内在奖励函数 r_E 得到的。

求解IRL的核心在于建立从奖励函数r_E到专家策略\pi_E的映射，从而通过专家数据求解反问题来恢复r_E。典型的求解方法有两种：

一种是直接概率建模，即通过线性规划来直接建模\pi_E在r_E下的MDP的在最优性， 典型方法即最大熵逆强化学习；
一种则是最大边际算法, 即最大化\pi_E的值函数与其他策略的值函数的差距来建立最优性， 典型方法即最大边际逆强化学习。

1.1.1 最大熵逆强化学习

通过寻求一个既能够合理解释专家演示结果，同时对数据中未明确体现的行为保持最大程度中立态度的奖励函数，从而解决了多种奖励函数均可解释同一行为所导致的模糊性问题。这导致了一个极小极大优化问题： \begin{equation} \operatorname{max}_r \operatorname{min}_{\pi}\left(\mathbb{E}_{\tau \sim \pi_{\mathrm{E}}}\left[\sum_{h=1}^H r\left(s_h, a_h\right)\right]-\mathbb{E}_{\tau \sim \pi}\left[\sum_{h=1}^H r\left(s_h, a_h\right)+\alpha H\left(\pi\left(\cdot \mid s_h\right)\right)\right]\right) \tag{1} \end{equation}\\其中，期望 \mathbb{E}_{\tau \sim \pi_{\mathrm{E}}} 是基于专家策略 \pi_{\mathrm{E}} （由数据集 \mathcal{D}_E 近似）的轨迹分布计算的，而期望 \mathbb{E}_{\tau \sim \pi} 是基于学习到的策略 \pi 生成的轨迹分布计算的。此外 H\left(\pi\left(\cdot \mid s_h\right)\right)=\mathbb{E}_{a_h \sim \pi\left(\cdot \mid s_h\right)}\left[-\log \pi\left(a_h \mid s_h\right)\right] 表示 \pi\left(\cdot \mid s_h\right) 的熵， \alpha>0 是正则化系数。目标是在相同的奖励 r 下，寻找一个奖励函数 r ，使得专家的预期回报与最优策略 \pi 的熵正则化回报之间的差距最大化。

接下来尝试求解以上问题。一种方法是inverse soft Q-learning，即寻找一个能够最好地解释静态数据集 \mathcal{D}_E={(s_1^i,a_{1:H}^i)}_{i=1}^n 中专家数据的 Q 函数。目标是解决以下优化问题： \max _Q \frac{1}{n} \sum_{i=1}^n \sum_{h=1}^H\left[Q\left(s_h^i, a_h^i\right)-\alpha \log \left(\sum_{a_h \in \mathcal{V}} \exp \left(Q\left(s_h^i, a_h\right)\right) / \alpha\right)\right] \tag{2}\\ 一旦我们找到最大化该目标的最佳 Q 函数 Q^* ，则可以通过逆软贝尔曼算子恢复相应的理想奖励函数 r^* ： r^{*}\left(s_h, a_h\right):=Q^{*}\left(s_h, a_h\right)-\alpha \log \left(\sum_{a_{h+1} \in v} \exp \left(Q^{*}\left(s_{h+1}, a_{h+1}\right)\right) / \alpha\right) \tag{3}\\ 那么 Q^* 该如何求解呢？事实上，它已经体现在任何使用标准下一个词预测目标训练的语言模型的 logits 中。

公式(2)中的目标函数可以通过变换，表示为一个最大似然问题： \begin{align}& \max_Q \frac{1}{n} \sum_{i=1}^n \sum_{h=1}^H\left[Q\left(s_h^i, a_h^i\right)-\alpha \log \left(\sum_{a_h \in \mathcal{V}} \exp \left(Q\left(s_h^i, a_h\right)\right) / \alpha\right)\right] \\ &=\alpha \max_Q \frac{1}{n} \sum_{i=1}^n \sum_{h=1}^H\left[Q\left(s_h^i, a_h^i\right)/\alpha- \log \left(\sum_{a_h \in \mathcal{V}} \exp \left(Q\left(s_h^i, a_h\right)\right) / \alpha\right)\right] \\ &= \max_Q \frac{1}{n} \sum_{i=1}^n \sum_{h=1}^H\left[\log(\exp( Q\left(s_h^i, a_h^i\right)/\alpha)) - \log \left(\sum_{a_h \in \mathcal{V}} \exp \left(Q\left(s_h^i, a_h\right)\right) / \alpha\right)\right] \\ &= \max _Q \frac{1}{n} \sum_{i=1}^n \sum_{h=1}^H \log \left(\frac{\exp \left(Q\left(s_h^i, a_h^i\right) / \alpha\right)}{\sum_{a_h^{\prime} \in v} \exp \left(Q\left(s_h^i, a_h^{\prime}\right) / \alpha\right)}\right) \end{align} \tag{4}\\其中策略 \pi_Q\left(\cdot \mid s_h\right)=\operatorname{softmax}\left(Q\left(s_h \cdot\right) ; \alpha\right) 这正是通过 NTP 来训练语言模型的目标。在训练过程中，语言模型 \hat \pi 会被优化以最大化数据的对数似然，即 \max _\pi \sum_{i=1}^n \sum_{h=1}^H \log \left(\pi\left(a_h^i \mid s_h^i\right)\right) 。由此产生的策略 \hat \pi 按定义是数据生成分布的最大似然估计，即被参数化为模型 logits 上的 softmax 。

通过这种方式，LLM 的输出 logits \hat f 直接是式(3)中 irl 优化问题的 Q 函数的有效解。即语言模型的 logits 不仅仅是随机的分数，还是一种原则性的 Q 函数，隐式地表示模型训练所用数据的最佳奖励函数，这一观点在引文[3,8] 中都有所证明。

现在令 \hat Q = \hat f , V_{\hat{Q}}\left(s_h\right):=\alpha \log \left(\sum_{a_h \in v} \exp \left(\hat{Q}\left(s_h, a_h\right) / \alpha\right)\right) , 代入公式（3）则有： \hat{r}\left(s_h, a_h\right)=\alpha \log \left(\hat{\pi}\left(a_h \mid s_h\right)\right)+V_{\hat{Q}}\left(s_h\right)-V_{\hat{Q}}\left(s_{h+1}\right) \tag{5} \\

可以看到这个结果与第一节中推导的结果殊途同归，所不同的是该结果中都不需要参考模型了。

总结一下：

任意一个 LLM 的token 生成策略 \pi _{\theta}\left( \cdot |s \right) =\mathrm{softmax} \left( f_{\theta}\left( s,\cdot \right) \right) , 都可以对logit 函数 f_\theta 通过逆强化学习诱导出一个内在奖励 r_\theta

r_{\theta}\left( s,a \right) =\log \pi _{\theta}\left( a|s \right) +V_{\theta}\left( s \right) -V_{\theta}\left( s\circ a \right) \\使得 (r_\theta,f_\theta) 满足贝尔曼最优方程。

2. 给定任意一个LLM生成策略 \pi_\theta, 在其诱导出来的内在奖励 r_\theta 上进行 RL 训练, 本质上优化的是

\underset{\pi}{\mathrm{arg}\max}\,\,\mathbb{E} _{x\sim \mathcal{D}}\mathbb{E} _{y\sim \pi \left( \cdot |x \right)}\left[ \log \pi _{\theta}\left( y|x \right) \right] \\会将策略 \pi 与 \pi_\theta 的高概率分布对齐。因此 r_\theta 本质上提供了一个模仿 \pi_\theta 生成的 token-level 奖励。

3. 内在奖励r_\theta 的好坏，依赖于 \pi_\theta 的生成质量的好坏。因为 r_\theta 的高奖励区域是 \pi_\theta 的高概率区域，这就与第一节中的隐式奖励产生了完美的呼应。

2.2 RLPR：无参考策略

理解了上述论证的过程，我们就理解了内生奖励的核心，细心的读者可能也会意识到所谓的“内生”却仍然依赖于一个参考策略，那么这个参考模型是否可以去除呢？RLPR 做了这方面的重要尝试，下面我们来一探究竟。

RLPR的思想源于这样一个认知：模型给 正确答案 的 token 概率高，给 错误答案 的概率低， —— 这个 内在置信度 本身就是对“推理好坏”的自然评价。因此，RLPR直接用 参考答案 token 的平均概率 当奖励，无需外部验证器。

尽管这个过程没有参考模型，但是却有参考答案，对于输入问题 Q ，输出答案前的推理内容 z ，生成答案 y , 参考答案 y^* , 其奖励计算的过程如下：

用参考答案 y^* 替换 y → 得到序列 o'
前向一次模型，取 y^* 对应 token 的概率 p_1,\dots,p_k
计算奖励值 r=\frac{1}{\left|y^*\right|} \sum\left\{p_i \mid o_i^{\prime} \in y^*\right\} ，其中 长度归一化 避免长答案天然吃亏，均值而非连乘 减少长尾低概率 token 的方差
去偏，直接用 r 会受 prompt 或答案风格干扰。额外计算一次 无推理 时直接给出 y^* 的概率 r′，用差值 \hat{r}=\operatorname{clip}\left(0,1, r-r^{\prime}\right) 只奖励因为推理 z 而带来的概率提升。

通过这种方式，RLPR 让 RLVR 走出“数学孤岛”，用 模型照镜子的方式 告诉自己“答得好不好”，从而在任何能写 prompt 的地方都能做强化学习。
