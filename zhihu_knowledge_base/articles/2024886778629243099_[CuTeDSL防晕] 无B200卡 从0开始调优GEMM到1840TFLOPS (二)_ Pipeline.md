CuTeDSL Pipeline概念
2. Pipeline API 概述
2.1 核心概念

Pipeline 是建立在 mbarriers上的sychronization classes。它抽象了软流水所需的协调模式：

模式：

producer: acquire → 写入 → commit
consumer: wait → 读取/使用 → release
3. mbarriers
3.1 mbarrier table

每个 pipeline 阶段两个mbarrier — 一个从prodcuer视角跟踪，一个从consumer视角跟踪：

Q1: 这张表是什么意思？

左列: mbarrier怎么创建的
Full (p) = 使用 producer_group 参数创建的 full mbarrier
Empty (c) = 使用 consumer_group 参数创建的 empty mbarrier
中间单元格: 每个函数触发什么
例如，producer_acquire 在 empty mbarrier上触发 wait
例如，producer_commit 在 full mbarrier上触发 arrive
	producer_commit	consumer_wait	consumer_release	producer_acquire
Full (producer -> consumer)	arrive	wait	-	-
Empty (consumer -> producer)	-	-	arrive	wait

这些信息阅读 custom pipeline（e.g. FA4 ）时会比较重要。读源码的时候也很有用。

3.2 states

state是一个有四个值的简单的类：

stage：buffer有几个stage
count：现在的循环数，通常用于gmem上的k index
index：现在在第几个stage上了,其实就是count%stage。通常用于smem上的index
phase：就是现在是在wait哪一个phase。

下面介绍的两种使用方式里第一种方式可以不用自己创建state比较方便一些。

3. 使用模式
3.1 Implicit State（更简单的方式）

使用 PipelineProducer 和 PipelineConsumer 对象与 handle 进行自动状态管理：

# 创建 5 stages的 pipeline
pipeline = PipelineAsync.create(
    num_stages=5,                   # pipeline 阶段数量
    producer_group=producer_warp,
    consumer_group=consumer_warp,
    barrier_storage=smem_ptr,       # mbarrier 数组的 smem 指针
)

# result types：PipelineProducer, PipelineConsumer
producer, consumer = pipeline.make_participants()

# ─── 生产者端 ───
for i in range(num_iterations):
    handle = producer.acquire_and_advance()  # 等待empty → 前进
    # handle.count  → 当前迭代计数
    # handle.index  → 当前缓冲区索引
    # handle.barrier → mbarrier
    cute.copy(tma_atom_A, gA_slice[(None, handle.count)], sA[(None, handle.index)])
    handle.commit()   # 通知缓冲区已满

# ─── 消费者端 ───
for i in range(num_iterations):
    handle = consumer.wait_and_advance()     # 等待full → 前进
    # 从 pipeline 缓冲区读取数据
    handle.release()  # 通知缓冲区已空
Q1: 这个 handle 是什么？
它是一个ImmutableResourceHandle — 同时充当producer/consumer对象+state：

可以调用 handle.acquire_and_advance() / handle.wait_and_advance() 方法
可以调用 handle.commit() / handle.release() 方法
可以访问 handle.index / handle.count / handle.barrier 成员
但它的状态是不可变的 — 您只能通过 xxx_and_advance() 方法推进它




3.2 Explicit State（进阶，推荐）

手动状态管理以获得更多控制：

# 创建 pipeline
pipeline = PipelineAsync.create(
    num_stages=self.num_ab_stage,  # 例如 2 或 3
    producer_group=producer_warp,
    consumer_group=consumer_warp,
)

# 初始化state
producer_state = pipeline.make_pipeline_state(
    PipelineUserType.Producer, self.num_ab_stage
)
consumer_state = pipeline.make_pipeline_state(
    PipelineUserType.Consumer, self.num_ab_stage
)

# ─── 生产者循环（例如 TMA 加载 Warp） ───
for k_tile in range(k_tile_cnt):
    # 1. 等待缓冲区变空
    pipeline.producer_acquire(producer_state)
    
    # 2. 获取缓冲区索引
    # state.index  → 当前缓冲区槽
    # state.count  → 迭代计数
    # 注意：没有 state.barrier！使用：
    #   pipeline.producer_get_barrier(producer_state)
    #   pipeline.sync_object_full.get_barrier(custom_index)
    buffer_idx = producer_state.index
    
    # 3. 写入数据到共享内存
    cute.copy(tma_atom_A, gA_slice, sA[(None, buffer_idx)])
    cute.copy(tma_atom_B, gB_slice, sB[(None, buffer_idx)])
    
    # 4. 通知缓冲区已满
    pipeline.producer_commit(producer_state)
    
    # 5. 前进到下一个阶段
    producer_state.advance()

# ─── 消费者循环（例如 MMA Warps） ───
for k_tile in range(k_tile_cnt):
    # 1. 等待缓冲区变满
    pipeline.consumer_wait(consumer_state)
    
    # 2. 获取缓冲区索引
    buffer_idx = consumer_state.index
    
    # 3. 计算 MMA
    # ... 使用 sA[(None, buffer_idx)], sB[(None, buffer_idx)] ...
    
    # 4. 通知缓冲区已空
    pipeline.consumer_release(consumer_state)
    
    # 5. 前进到下一个阶段
    consumer_state.advance()
Q1: 有什么需要注意的

手动状态跟踪: 必须自己维护state，每次调用 acquire/commit/wait/release 时传递它，并手动推进它
mbarrier访问很麻烦: 没有 handle.barrier 快捷方式 — 必须使用：
pipeline.producer_get_barrier(producer_state)，或
pipeline.sync_object_full.get_barrier(custom_producer_index) 如果使用自定义状态索引







3.3 隐式 vs 显式状态比较
方面	隐式状态	显式状态
易用性	更简单 — 自动state管理	更多手动工作
灵活性	有限 — 固定state advance	完全控制状态
mbarrier访问	通过 handle.barrier	通过 pipeline.producer_get_barrier()
适用场景	简单 pipeline	高级模式，自定义索引
推荐	适合初学者	适合生产 kernel
