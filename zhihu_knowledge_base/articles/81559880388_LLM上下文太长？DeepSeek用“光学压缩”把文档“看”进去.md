论文提出的想法是：如果我们不让LLM阅读文本，而是让它看文本呢？，也就是论文的核心思想“光学压缩”（Optical Compression）。简单来说，一篇1000个单词的文档可能包含数千个文本 token，但如果我们将这篇文档渲染成一张图片，一个高效的视觉编码器可能只需要几百个视觉 token 就能完整地“看到”所有内容。

什么是“光学压缩”？

“光学压缩”的本质非常直观：利用视觉2D映射来压缩1D的文本信息。

压缩：将长篇的数字文本文档（如PDF）渲染成一张高分辨率图片。一个专门设计的视觉编码器（Encoder）将这张图片“压缩”成一组数量极少的视觉 token。
解压：一个语言模型（Decoder）接收这些视觉 token，并将其“解压”还原为原始的文本内容。

DeepSeek-OCR 就是这样一个“概念验证”（proof-of-concept）模型。它要回答的核心问题是：这个压缩过程的极限在哪里？我们能以多高的压缩率，同时保持多高的文本还原精度？

剧透一下结果：DeepSeek-OCR 成功地在 10 倍压缩率下（即文本 token 数量是视觉 token 的 10 倍）实现了高达 97% 的解码精度。

模型设计：DeepEncoder 架构

为了实现高压缩率，DeepSeek-OCR 需要一个能处理高分辨率输入（比如一张A4纸的截图），同时输出极少视觉 token 的编码器。现有的视觉编码器架构（如 ViTDet、InternVL、Qwen2.5-VL 等）要么需要复杂的预处理，要么会产生过多的视觉 token，要么在处理高分辨率时内存开销巨大。

因此，团队设计了一个全新的 DeepEncoder 架构，这是整个系统的核心。

DeepSeek-OCR 架构图


DeepEncoder 的设计非常巧妙，它串联了三个关键组件：

SAM-base 局部注意力：首先，一个基于 SAM (Segment Anything Model) 的轻量级编码器（80M参数）处理高分辨率图片。它使用“窗口注意力”（Window Attention），只在局部区域内计算，高效地完成视觉感知任务，将图片切分成大量的初始 patch (例如 4096 个)。
16x 卷积压缩器 (Compressor)：这是关键的“瓶颈”。一个简单的2层卷积模块，对来自 SAM 的 4096 个 token 进行 16 倍的下采样，将其压缩到仅剩 256 个 token。
CLIP-large 全局注意力：最后，一个强大的 CLIP 编码器（300M参数）接收这 256 个被压缩的 token。由于 token 数量已经变得极少，模型现在可以“奢侈”地使用计算量大的“全局注意力”（Global Attention）来深度理解这些 token 蕴含的“知识”和上下文。

这个“先局部后全局、中间强力压缩”的设计，使得 DeepEncoder 既能处理超高分辨率的输入，又保持了极低的计算激活内存，完美地实现了“高压缩率”和“低开销”的双重目标。

模型的“大脑”（Decoder）则是一个 DeepSeek-3B-MoE 模型，它拥有 30 亿总参数，但在推理时只激活 5.7 亿（570M）参数，实现了小模型的推理效率和 大模型的表达能力。

实验结果

DeepSeek-OCR 的性能不仅验证了光学压缩的可行性，更在实际OCR基准上展现了惊人的效率。

1. 压缩率极限测试 (Fox Benchmark)

在 Fox 基准测试中，测试了不同压缩比下的模型表现：

< 10x 压缩 (100 视觉 token): 当文本 token 在 1000 个以内时（例如 600-700 个文本 token 被压缩为 100 个视觉 token），模型精度高达 **98.5%**。
~20x 压缩 (64 视觉 token): 即使在接近 20 倍的极限压缩下（例如 1200-1300 个文本 token 被压缩为 64 个视觉 token），模型依然能保持约 60% 的准确率。

这证明了，视觉 token 作为文本信息的压缩载体是完全可行的。

2. 实际 OCR 性能 (OmniDocBench)

在权威的 OCR 基准 OmniDocBench 上，DeepSeek-OCR 展示了其作为SOTA（State-of-the-Art）模型的强大实力和极致的效率：

DeepSeek-OCR (100 视觉 token) 战胜了 **GOT-OCR2.0 (256 视觉 token)**。
DeepSeek-OCR (约 800 视觉 token) 战胜了 **MinerU2.0 (约 7000 视觉 token)**。
对比了不同模型在 OmniDocBench 上的性能（y轴，编辑距离，越低越好）和平均视觉 token 数（x轴，越左越好）。DeepSeek-OCR 系列模型（红色和绿色点）位于左上角，展示了较好的性能-效率平衡。


结论是：DeepSeek-OCR 在端到端模型中，使用最少的视觉 token 达到了SOTA性能。

不只是OCR：通往“遗忘机制”的未来

DeepSeek-OCR 的意义远不止于一个更高效的OCR工具。论文作者向我们展示了“光学压缩”的真正潜力：模拟人类的记忆和遗忘机制。

1. 深度解析 (Deep Parsing)

模型不仅能识别文本，还能对文档中的复杂元素进行结构化解析，例如：

图表解析：将图表转换为 HTML 表格。
公式解析：将化学分子式图片转换为 SMILES 文本格式。
几何解析：结构化地理解平面几何图形。
2. 模拟 LLM 的长期记忆

这里挺有意思的。想象一个 LLM 正在进行一场非常长的对话：

近期记忆-高保真：当前和最近几轮的对话，以原始的文本 token 形式保留，确保“晶莹剔通”(Crystal Clear) 的理解。
中期记忆-压缩：几小时前或几天前的对话历史，可以被渲染成一张高分辨率图片，用 Large 或 Gundam 模式（例如 400 个视觉 token）进行压缩。
远期记忆-高度压缩/遗忘：几周或几个月前的历史，被渲染成更小的图片，用 Base 或 Small 模式（例如 100 个视觉 token）压缩。
模糊记忆-极限压缩：一年前的历史，用 Tiny 模式（例如 64 个视觉 token）压缩，文本变得模糊，只保留最核心的要点。
将人类的记忆衰退（随时间）与视觉衰退（随距离）和文本光学压缩（随分辨率降低）进行了类比。


通过这种方式，LLM 可以用极低的 token 成本维护一个“理论上无限”的上下文历史。这种渐进式的压缩和信息丢失，完美地模拟了生物的“遗忘曲线”，让模型能够抓住重点，同时“忘记”不重要的细节。

总结

DeepSeek-OCR 更像是一次“范式转移”的探索。它向我们证明了：

光学压缩是可行的：视觉模态可以作为一种极其高效的文本信息压缩介质，实现 10x 甚至 20x 的压缩。
效率即是SOTA：DeepSeek-OCR 用最少的视觉 token 实现了顶尖的OCR性能。
未来的LLM记忆：该技术为解决 LLM 长上下文瓶颈提供了一个全新的、受生物启发的思路——通过“看见”和“遗忘”来实现无限上下文。
