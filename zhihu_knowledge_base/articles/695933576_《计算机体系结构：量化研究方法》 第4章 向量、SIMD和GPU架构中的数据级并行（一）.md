Chapter 4 Data-Level Parallelism in Vector, SIMD, and GPU Architectures
4.1 介绍/Introduction

本章节覆盖了SIMD的三种变体：向量架构、多媒体SIMD指令集扩展和图形处理单元（GPU）。

向量架构。扩展许多数据操作的流水线执行。
多媒体SIMD指令集扩展。基本上意味着同时进行并行数据操作，存在于在大多数支持多媒体应用的指令集架构中。
图形处理单元。来自图形加速器社区，提供了比传统多核计算机更高的潜在性能。
4.2 向量架构/Vector Architecture

向量架构抓取分散在内存中的数据元素组，将它们放置到一个大的register file中，然后对register file中的数据进行操作，最后将结果分散回内存。一条工作在数据向量上的指令，会产生对独立数据元素数多次寄存器-寄存器操作。

这些大的register file充当编译器控制的buffer，既可以隐藏内存延迟，又可以充分利用内存带宽。由于向量加载和存储是流水线的，程序只需为每个向量加载或存储花费一次较长的内存延迟，延迟会被分摊到多个元素上，例如，32个数据元素。

RV64V指令集体系结构的主要组成部分如下:

向量寄存器（Vector registers）：每个向量寄存器保存一个单独的向量，RV64V有32个寄存器，每个寄存器宽度为64位。向量register file需要提供足够的端口来为所有向量功能单元提供数据，并且这些端口需要允许不同向量寄存器之间的高度重叠向量操作。总共至少有16个读端口和8个写端口，通过一对corssbar连接到功能单元的输入或输出。增加register file带宽的一种方法是由多个bank组成，这种方式对于相对较长的向量效果好。
向量功能单元（Vector functional units）：每个功能单元都是完全流水线化的，可以在每个时钟周期开始一个新的操作。控制单元用来检测冒险情况，包括功能单元的结构冒险和寄存器访问的数据冒险。
向量加载/存储单元（Vector load/store unit）：向量内存单元将从内存加载向量或存储向量到内存。此单元通常也会处理标量加载和存储操作。
标量寄存器（scalar registers）：标量寄存器可以为向量功能单元提供输入数据，也可以计算地址并传递给向量加载/存储单元。向量功能单元的一个输入将标量值锁存为从标量寄存器文件读出的值。

使用动态寄存器有几个好处：

传统向量体系结构需要许多指令来支持不同的数据类型，动态寄存器可以减少指令数量。

动态类型还允许程序禁用未使用的向量寄存器。启用的向量寄存器分配全部向量内存为长向量。

假设有1024个字节的向量内存，如果启用了4个向量寄存器，并且是64位浮点数类型，则处理器会给每个向量寄存器分配256字节或256/8=32个元素。这个值被称为最大向量长度（maximum vector length ，mvl），由处理器设置，不能被软件更改。

2. 传统向量架构如果具有更大的状态意味着更慢的上下文切换时间。动态寄存器类型，程序可以在不使用向量寄存器时将其配置为禁用状态，因此在上下文切换时无需保存和恢复它们。

3. 不同尺寸操作数之间的转换可以根据寄存器的配置隐式进行，不需要额外的显式转换指令。

vld和vst是向量加载和向量存储指令，用于加载或存储整个向量数据。

PS：例如，如果a0的值为1024，并且v0的类型为X32，那么vld v0, 0(a0)将生成地址1024，1028，1032，1036，...直到达到vl所设置的限制。

向量执行时间

向量操作的执行时间主要取决于三个因素：（1）操作向量的长度，（2）操作之间的结构冒险，（3）数据依赖关系。给定向量长度和启动速率（即向量单元消耗新操作数和生成新结果的速率），可以计算单个向量指令的时间。

所有现代的向量计算机都有具有多个并行流水线（或通道）的向量功能单元，每个时钟周期可以产生两个或更多的结果，但也可能有一些功能单元不是完全流水线化。简单起见，假设只有一个通道，每个时钟周期的启动速率为一个元素。因此，单个向量指令的时钟周期执行时间大约等于向量长度。

为了简化对向量执行和向量性能的讨论，书中使用convoy，表示可能一起执行的一组向量指令。一个convoy中的指令不能包含任何结构冒险；如果存在这样的冒险，指令需要被串行化，并在不同的convoy中启动。

在前面的例子中，vld指令和随后的vmul指令可以在同一个convoy中。

除了结构冒险外，RAW依赖冒险的序列，通过链接可以放在同一个convoy中。链接允许向量操作在其向量源操作数的各个元素可用时立即开始，向量操作在其向量源操作数的单个元素可用时立即开始（链中第一个功能单元的结果被“传递到第二个功能单元）。

通过允许处理器同时读取和写入到特定的向量寄存器（针对不同元素）来实现链接。早期的链接实现就像标量流水线中的forwarding一样，但这限制了链中源指令和目的指令的时序。目前采用使用灵活链接（flexible chaining），允许一个向量指令链接到几乎任何其他活动的向量指令，假设不产生结构冒险。

本书中使用chime，表示执行一个convoy所需的时间单位，作为向量执行时间的度量标准。m个convoy的向量序列需要执行m个chime，对于向量长度为n的简单RV64V实现来说，大约是m×n个时钟周期。

假设有下面代码序列，看下是如何构建convoy的，以及每个周期需要多少FLOP（floating-point operation）。

第1个convoy包含vld指令和vmul指令。vld指令和vmul指令之间的数据依赖关系通过链接解决。 第2个convoy包含vld指令和vadd指令。由于第1条vld和第2条vld之间存在结构冒险，放在第2个convoy中，vadd指令与第2个vld指令之间的数据依赖关系通过链接解决。 第3个convoy包含vst指令。由于vst指令与之前vld之间存在结构冒险，放在第3个convoy中。

除了发射开销外，还有个重要的开销影响向量执行时间计算的是是向量启动时间（流水线填满所需的时钟周期）。启动时间主要由向量功能单元的流水线延迟确定。例如，一些处理器中，浮点加法的流水线深度为6个时钟周期，浮点乘法为7个时钟周期，浮点除法为20个时钟周期，向量加载为12个时钟周期。

多通道：每个周期大于一个单元

向量指令集的一个关键优点是，允许软件只使用一条短指令将大量并行工作传递给硬件。一个向量指令可以包含多个独立操作，但编码的比特数与传统标量指令相同。

向量指令的并行语义允许实现使用深度流水线的功能单元来执行这些基本操作；或者并行的功能单元数组；或者使用并行和流水线功能单元的组合。

图4.4展示了如何通过使用并行流水线来执行向量加法指令来提高向量性能。左侧向量处理器（A）有单个加法流水线，每个时钟周期可以完成一次加法。右侧的向量处理器（B）有四个加法流水线，每个时钟周期可以完成四次加法。

图4.4 使用多个功能单元来提高单个向量加法指令C=A+B的性能

RV64V指令集中所有的向量算术指令只允许一个向量寄存器的第N个元素与其他向量寄存器的第N个元素参与运算，极大简化了高度并行向量单元的设计，可以将其构建为多个并行通道。就像高速公路一样，可以通过增加更多通道来增加向量单元的峰值吞吐量。

如图4.5所示，展示了一个四通道向量单元的结构。因此，从单通道增加到四通道使得向量计算时钟数变为1/4。为了发挥多通道具有优势，应用程序和架构都必须支持长向量；否则，将执行过快导致指令带宽不足，需要使用ILP技术提供足够的向量指令。

图4.5 一个包含四通道的向量单元结构

每个通道包含部分向量register file和流水线向量功能单元。每个向量功能单元每个周期执行一条向量指令。第一个通道为所有向量寄存器保存第一个元素（element 0），所以每个向量指令的第一个元素的源操作数和目的操作数位都位于第一个通道中。这种分配方式使得通道本地的算术流水线能够在不与其他通道通信的情况下完成操作。避免通道间的通信可以降低高度并行执行单元所需的布线成本和register file端口。

增加多个通道是提高向量性能的一种常用技术，仅增加少量控制的复杂性，并且不需要更改现有的机器代码；还允许在不牺牲峰值性能的情况下，权衡芯片面积、时钟频率、电压和能耗。如果向量处理器的时钟频率减半，将通道数量加倍仍可以保持相同的峰值性能。

通过一个向量长度寄存器（vl）可以解决这些问题。vl用于控制向量操作的长度，包括向量加载或存储。并且，vl中的值不能大于最大向量长度（mvl）。

谓语寄存器：处理向量循环中的IF语句

循环内部的条件语句（IF语句）和稀疏矩阵的在程序中较难实现向量化。

考虑如下C语言写的循环：

for(i=0;i<64; i=i+1) 
  if (X[i] != 0) 
    X[i] = X[i] – Y[i]; 

因为有条件执行的部分，这个循环不能够被向量化。但是如果内部循环可以在X[i] != 0的条件下运行，则减法操作就可以进行向量化。

常用通过vector mask控制。

在RV64V中，谓语寄存器保存mask，并为向量指令中的每个元素操作提供条件执行。这些寄存器使用布尔向量来控制向量指令的执行。当谓词寄存器p0被set时，所有后续的向量指令只对谓词寄存器中对应项为1的向量元素进行操作。目的向量寄存器中与mask寄存器中0对应的项，不受向量操作的影响。像向量寄存器，谓词寄存器可以被配置和禁用。开启谓词寄存器会全被初始化为1，意味着后续向量指令将对所有向量元素进行操作。

内存banks：为向量加载/存储单元提供带宽

加载的启动时间是从内存中获取第一个word到寄存器的时间。如果剩余向量可以在没有停顿的情况下提供，那么向量启动速率等于获取或存储word的速率。

为了维持每个时钟周期获取或存储一个word的启动速率，内存系统必须能够产生或接受这么多数据。通常采用分散内存访问多个独立的bank可以实现所需的速率。

大多数向量处理器使用内存banks，允许几个独立内存访问有三个原因:

为了支持每个时钟周期多个加载或存储操作。
为了支持加载或存储非连续的数据。
为了支持多个处理器共享同一个内存系统。

向量加载/存储单元在提供数据带宽方面起到类似于标量处理器中预取单元的作用，两者都通过提供数据流来向处理器提供数据带宽。

步长：在向量架构中处理多维数组

向量中相邻元素在内存中的位置可能不是连续的。

考虑下面，用c语言实现的矩阵乘法，可以将B的每一行与D的每一列的乘法进行向量化，并以k为索引变量对内部循环进行拆分。

for (i = 0; i < 100; i=i+1) 
  for(j=0;j<100; j=j+1){ 
    A[i][j] = 0.0;   for (k = 0; k < 100; k=k+1) 
      A[i][j] = A[i][j] + B[i][k] * D[k][j]; 
  } 

为了实现向量化，需要考虑怎么处理B和D中邻近的元素。

当一个数组被分配内存时，被线性化并且必须行优先（C语言）或列优先（Fortran语言），意味着行中或列中的元素在内存中不是相邻的。

例如，前面的C语言代码以行优先顺序分配内存，因此由内部循环中D中的任意列相邻元素相隔行的尺寸乘以数据为宽（8个byte），共800个byte。

PS：简单画了个示意图，矩阵B由于是以行进行访问，所以在内存空间中是相邻的。矩阵B由于是以列进行访问，但是内存存储是以列进行的，所以在内存空间中相邻的列元素是需要间隔一整个行的数据空间。


这个将要被采集到向量寄存器中的元素之间的距离称为步长（stride）。在这个例子中，矩阵D的步长是100个double word（800 bytes），而矩阵B的步长是1个double word（8 bytes）。

一旦向量被加载到向量寄存器中，就会像相邻的元素一样工作。向量处理器可以使用有步长能力的向量加载和向量存储操作来处理步长大于1（即非单位步长，nonunit stride）的情况。

访问非连续的内存空间，然后重塑（reshape）到一个稠密的结构中是向量架构的主要优势之一。

缓存固有地处理单位步长（unit stride）数据。增加block尺寸可以降低单位步长的大型科学数据集的未命中率，但是对于以非单位步长访问的数据来说，增加块尺寸可能产生负面效果。

在RV64V上，可寻址的单位是byte，上面例子D阵列需要的步长将为800。这个值必须在运行时计算，因为矩阵大小可能在编译时是未知的，在同一语句的不同执行中可能会发生变化。向量步长和向量起始地址可以放在通用寄存器中。通过VLDS（带步长的向量加载）指令将向量加载到向量寄存器中；通过VSTS（带有步长的向量存储）指令，存储一个非单位步长的向量。

步长大于1会使内存系统变得复杂，可能会导致对于同一个bank的频繁访问。如果出现多个访问对于一个bank的竞争，会内存bank的冲突，导致停滞部分访问。

聚集-散布：在向量架构中处理稀疏矩阵

在稀疏矩阵中，向量元素通常以某种紧凑的形式存储，然后通过间接方式访问。

假设有一个简化的稀疏结构，可能会看到类似的代码，实现了数组A和C的稀疏向量求和，使用索引向量K和M来标识A和C的非零元素。

for (i=0;i<n; i=i+1) 
  A[K[i]] = A[K[i]] + C[M[i]]; 

支持稀疏矩阵的主要机制是使用索引向量进行聚集-散布（gather-scatter）操作，在稀疏矩阵的压缩表示（不包括零元素）和正常表示（包括零元素）之间进行转换。

聚集（gather）操作。获得索引向量，抓取向量（其向量元素位于基地址与索引向量的偏置量相加的目标地址）。在向量寄存器中会得到一个稠密向量。
散布（scatter）操作。在以稠密形式对向量寄存器中的元素进行操作后，可以使用相同的索引向量将稀疏向量以散布形式存储。

RV64V指令采用vldx（向量索引加载或聚集）和vstx（向量索引存储或散布）。
