# MLIR入门理解1

**作者**: i4oolishAI框架，编译器，体系结构

**原文链接**: https://zhuanlan.zhihu.com/p/450022851

---

这篇文章是个人作为新入门者，对自己这段时间学习MLIR的一个总结，很浅显，作为一个学习过程中间阶段思路的总结，供以后去回顾思考。

说起MLIR，之前在知乎也看过很多问答和文章，对个人影响比较大的是如何看待Google的MLIR项目？和法斯特豪斯写的系列文章和视频教程。

但学完之后，一直存在两个问题：

MLIR解决什么问题，整体架构是什么；
MLIR如何像TVM那样，做计算调度优化？




以下是个人的思考：

核心目的：弥补框架和底层硬件之间的gap，连通上层框架和下层硬件；
解决方案：Dialect，DialectConversion；
实现方案：a. 实现Dialect -> C++定义Dialect/ODS定义（包含Operation, Attribute, Type, Constraint, Interface, Trait）；b. Dialect内部转换 -> Transformation；c. Dialect之间转换 -> DialectConversion；d. Pass优化。其中bcd均可以采用C++代码或者DRR方式去实现。

以上是MLIR的架构和背景，基于此，MLIR最大的贡献是1. 提供了一套编程接口，2. 以及一套Dialect。借鉴网友的说法，即：C++语法和STL库。以toy example来说明这两点：

toy example可以帮助我们理解MLIR的整体架构，即语法；
toy example无法解答的是Dialect的分工和交互（只是描述了ToyDialect, Affline, ..., LLVM，没有对Dialect整体架构进行描述）。

基于以上观点，可以认为不是toy不重要，而是单独toy还不够。

作为新入门选手，个人感觉去理解MLIR的open for extension, high performance codegen有点好高骛远。对我们来说，MLIR最重要的是1. 它提供的基础设施（Dialect, ODS, DRR, Transformation, DialectConversion；标准Dialects）；2. 以及胶水能力（即不同Dialect的分工和合作）。

这两点缺一不可，其中第一点是基础，第二点是升华。关于第二点，MLIR中Dialects分类及关联这篇文章讲的很好，一方面对MLIR现有Dialect进行了讲解和分类，另一方面，最重要的是唤起我们去思考Dialect之间的交互和分工。

接下来是关于应用的问题，个人觉得不存在应用到legacy项目的问题。在MLIR眼里，所有现有的框架，库，底层硬件都是新的Dialect。归根到底不是如何与legacy项目结合，而是如何设计不同Dialect之间的分工，以及交互，即核心是Dialect的架构和设计，即前面描述的2.胶水能力。




以上便是个人的粗浅认识，尚缺乏对MLIR最基础的架构和模块分析，以及对具体Dialect的分析。

接下来会从两方面去整理和总结：

MLIR代码架构和模块；
Dialect分类和关联。
