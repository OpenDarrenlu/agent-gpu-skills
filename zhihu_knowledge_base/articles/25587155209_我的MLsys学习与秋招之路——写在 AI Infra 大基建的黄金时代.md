秋招已经结束快两个月有余，我想要用这篇文章把这一年多以来学习和准备秋招的漫长旅途好好画上句号。

方向的选择
踏上 MLsys 之路

我与 System 方向结缘很早，本科的时候 OS 与编译器两门课是我个人认为我学的最好的本科课程。读研之后，因当时组里有跟阿里云合作的TensorFlow框架优化的项目，便在师兄和刘童璇前辈的指导下做了一些训练优化（Gradient Checkpoint）相关的研究工作。而后在23 年春项目结束之际正赶上ChatGPT的超大热度，便受命带着学弟一起去做了一些 RAG/Agent 相关 LLM 应用层的研究。这是我第一次正式接触大模型相关的课题，也被 LLM 所展现出的强大能力深深吸引。在课题结束后，我开始思考日后就业或进一步深造的研究方向，因我对传统后端兴趣不大，非常想要继续做大模型相关的事情，于是摆在我面前的就剩下了三个方向，应用层（RAG/Agent），算法层，以及 MLsys。在经过思考后，我个人认为，RAG/Agent 更像是产品经理需要去掌握的技能，因为它的门槛不在技术上，而在于如何去设计一个好的产品，用大模型作为组件把它搭起来。而算法我则是只懂皮毛，组里也没有相关方向。而 MLsys 我自身有一定基础，并且看着 Nvidia 的股价蹭蹭往上涨，我认为算力以后会成为水电气一样重要的基础资源，便下决心选择了这个方向继续深耕。

训练or推理？

选定大方向之后，我首先开始了比较全面的基础课程学习，在这里感谢 @ZOMI酱 以及五道口纳什的 B 站课程，在 23 年的秋天，我把二位当时的教学视频看了个遍，狠狠打了一波大模型算法和训推框架的基础，还要感谢 @方佳瑞博士的很多技术文章，我受益匪浅。同时，在跟导师进行了沟通之后，我还争取到了出去实习的机会（这里要狠狠吐槽一句中科大的地理位置劣势导致实习真的很不方便，感谢导师放我实习）。当时，得益于两位师兄的内推帮助，我的第一次找实习非常顺利，当时要么去阿里云做训练框架（类似于 Megatron），要么去腾讯云做推理框架（类似于 vLLM）。就此问题，我当时的看法是：大规模的基座模型训练应该只是少数玩家的事情，大模型真正铺开的 Bottleneck 应该在推理侧。于是我坚定了自己的想法，放弃了 PAI 团队的非常好的机会，去了鹅厂开始大模型推理加速的实习。

实习之旅
第一段实习：腾讯

当时我去到了腾讯 CSIG 的一个算法团队进行研究型实习，我的 mentor Hande 是一位很年轻的高 T 科大校友，在算法方向有很资深的经验。组里的前辈人非常好，给了我很宽松的学习环境，当时我和 mentor 想做一个类似于 NanoFlow 这样的Intra-device parallelism engine，那时还没有Cuda Green Context 这种黑科技，于是我去找了相关的论文自己按照思路手搓了一个可以将计算任务做具体 SM 映射的并行框架。而后我又开始学习爆火到如今的大模型推理引擎 vLLM 的源代码。把 vLLM 整个架构设计，调度策略，显存管理以及 Paged Attn算子进行了系统性的学习。之后又在此之上进行了大量的评测工作，给 nsys/ncu 玩的冒烟，也切身地体会到了LLM 推理不同的计算特性带来的优化上的复杂性。可惜的是最终努力并没有得到一个非常好的结果，在实习之前给导师保证去腾讯拿一篇像样的成果出来，论文一直没有做出来搞得我压力非常大，Mentor 一直宽慰我让我少些急躁。最后好在发现在做这份科研中的一些中间成果可以去对 Chunked Prefill 做一下小小的优化，能让自己的实习总算有所产出来丰富简历，也算是有所收获。

这段实习虽然没有做出来System顶会论文，但是系统性地建立了大模型推理的知识框架，也是我第一次跟工业界接触，了解了互联网大厂究竟是个什么样子。也让我第一次开始认真思考之后到底是继续读博士，还是去互联网，亦或是回家找个国企。并且，我也初步认识到了 ai infra 在工业界的地位，同样是做 mlsys，在不同的组做确实产生的效果是不同的，如果真的要在相关方向正式工作，最好还是去专门做 mlsys 的组里得到的资源和成长可能会更好。

第二段实习：美团

由于要考虑到秋招的问题，我又去找了一段暑期实习，去到了美团机器学习引擎组。在专门做 mlsys 相关组内我体会到的东西更加不一样，我看到了 MLsys 在搜广推领域的一些应用，也看到了 MLsys 相关工作也存在很多对业务很重要但对个人发展价值却有限的“Dirty Work”，这些信息才让我知道了自己在 MLsys 领域对哪些工作更感兴趣。

我非常感谢我当时的两位 mentor @Yineng Zhang 和 Ke Bao， 如今他们都是 SGLang 推理引擎的核心贡献者，特别是 Yineng 让我看到了参与开源项目的意义，同时也在我表示对大模型推理优化更感兴趣之后，教了我很多他们亲身参与过的东西，比如 Prefix Cache 和 Spec Decode等。在美团实习期间也是秋招的提前批，两位 mentor 给了我非常多的建议和鼓励，特别是 Ke Bao 前辈在我本身水平欠佳的情况下也依旧给了我很大的支持和肯定。也请读者朋友多多关注他们的工作～

总结来讲，两段实习让我搭建了：模型架构->框架->底层算子 这样一个比较系统化的知识体系，也给了我足够多的项目去应付秋招。

秋招？求招！
信息收集

在整个实习期间，除了学习知识之外，我也在收集业界的信息。首先是我之前在鹅厂实习的时候就关注到了腾讯 CSIG 叶老师的异构计算组，他们组做的 TACO-LLM是腾讯内部自用的很出色的推理引擎。而后我逐渐关注到了字节的 AML-Engine，阿里云的 PAI 团队等非常优秀的团队，在五月份，我关注到了字节跳动的 Seed 计划，当时还在感叹自己不是算法，没办法去跟着一起搞搞大模型，没想到后面竟有机会结缘Seed。总的来讲，我当时把整个业界的 MLsys 优秀的组尽可能的进行了信息收集，并且分别找好友咨询了一下相关情况，按照个人的意向进行了排序，做成了一份秋招计划表。

秋招经历

整个秋招历史半年，从七月初开始到 12 月底结束，纵观各家的考核，其实都在围绕模型架构，推理框架，底层算子和调优分析四部分展开，难度各异，BAT 的难度总体会高一档。我前前后后面了十余家大中厂以及初创，从一开始的紧张到后面的侃侃而谈，感觉面试更像是一种知识的交流而非单方面的考察，很多业界出名的前辈给了我无价的帮助和建议。比如在面阿里控股 @刘侃 前辈组的时候，跟面试官的交流让我想明白了 P-D分离体系中 PD 比例这一始终困扰我的问题，在面试阿里云 @孙敏敏 老师组的时候，我向九丰老师请教了很多关于我职业生涯规划的问题，得到了很宝贵的批评指教。面试 CSIG 异构计算组（感谢 @手抓饼熊 老师帮忙内推）的时候，发现了自己对于 Paged Attention Kernel 在细节上很多没有意识到的地方。在不断的面试过程中感觉自己的知识体系被进一步完善真的是非常好的体验。同时，在八月份的时候，我在公众号上刷到了字节 Seed-Foundation 团队的校招信息，发现它们正好有我对口的岗位，于是立即找朋友联系上了 HR @悦悦 ，在九月参加了推理优化组的面试。

秋招笔面试安排的冰山一角

在这里也给一些小小的建议：

尽可能围绕自己的技术栈讲故事，尽可能投递对口的岗位，不要把火烧到自己不熟悉的地方（比如我之前曾在简历上写过自己做过 TF 训练优化，但是我对训练不熟悉结果一问三不知，后面直接删掉了这个东西）
真诚至上，不会的就说不会（比如：我对 A 技术不了解，但是对 B 技术有所涉猎，不知您是否感兴趣），不是一定要去的坑也不要夸下海口说一定要去，组里想要你会让 HR 争取你的。
表达尽量精简，概要，遵循“为什么要做->怎么做->最终效果“的逻辑进行阐述，可以在熟悉范围内尽量讲一个大一点的故事来给面试官阐述你的技术栈，不过前提是你得包的住。
投递之前做好信息收集，不要去投一定不去的岗位，可以将一些“保底”性的岗位先投递先面试，经验丰富之后再去挑战大厂。同时请多多搞一下 connection，现在简历很多，请尽量找到组内的人，以便打听情况和简历直推。

最终，在经过漫长的泡池子等待之后，我拿到了阿里字节百度等多个大厂的SSP Offer，以及中兴蓝剑等少量人才计划 Offer，初创由于我暂不想去提前实习所以基本都面完无音讯了。最终权衡了职级、业务、发展和薪酬的情况下，选择了字节跳动 Seed 作为我职业生涯的第一站，同时也打破了我坚决不去北京的 FLAG。

整个秋招期间，我也在我的高中同学曾同学的指导下投递过央国企，也参加过选调生考试，进了老家某四线城市面试，但是最终也放弃了面试机会。得到了离家近钱也还算可以的国企 offer，最终权衡之下也没有去。究其原因，一是因为我想做大模型，二是因为今年互联网 LLM 薪酬确实很有竞争力，三是我确实对我现在所选的方向有信心，而且也觉得自己为之真的付出了很多很多的努力，不忍心丢掉。

有的时候也会想，如果换一条路走，说不定现在我会有非常圆满而无压力的生活，也不会让父母担心我失业，也不会常常焦虑，毕竟人总会觉得自己没有选择的道路上开满了鲜花。但是路已至此，整个秋招的结果于我而言已经非常满意，剩下的道路只能抛弃杂念勇敢向前了。

在此也对秋招期间给我提供巨大帮助的朋友们（ @企鹅火烈鸟 @大润发杀鱼工 还有好多朋友以及帮忙内推的贵人不知道知乎账号不一一艾特了，tb 在内心感谢大家）表示衷心的感谢。

秋招之后：科研与开源，MLsys 的黄金时代

在秋招后期，我得以沉下心来继续学习大模型推理相关知识， 那时我非常想去接触一下 P-D 分离相关的研究，正好 @Hsword 苗老师不嫌我菜，愿意带着我一起玩，在跟 Xupeng 和 Chunan 做远程科研期间再次学到了很多分布式的知识。在学习结束后，我开始去尝试接触SGLang，沿着 yineng 前辈之前给我指出的路想要给开源界做出贡献（第一个 PR 即将产出！），认识了 @Chayenne Zhao 等很多优秀的同行，交到优秀的新朋友感觉才是在做大模型中的一大乐趣。

行文至此，想到秋招之后小红书对于 MLsys 的一片唱衰（毕竟确实很难，薪酬在24 年之前也比不过算法），又想到如今大火的 Deepseek 的 open-source week 开源的均是 MLsys 的前沿技术，MLsys 在短时间内爆火到不少优秀的学弟学妹想主动入行，可以说，现在正处于 MLsys 大基建的黄金时代。技术风向的变化确实太快，但是我想，任何时候都不应抱着趋红踩黑的心态来规划自己的人生，技术是学不完的，但是解决问题的方法确是通用的。学到方法论，才是立身之本。

最后，保持好奇，保持精力，希望你我的前途都光明灿烂！
