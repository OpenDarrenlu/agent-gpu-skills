怎么使用 MLIR 框架提供的 API？

刚上手一个大型的 C++ 项目，通常情况下有一个问题是相当令人苦恼的，也就是关于 API 的使用。具体来说，通常又会有如下几个疑问：

我想实现某个需求可能会用到的 API 这个框架有提供吗？
我要如何调用 API？

这里我给出一些自己用到的用来熟悉 MLIR 框架 API 的方法。

Bug : error: operand #0 does not dominate this use

碰到这种问题最重要的 debug 方法是查看 mlir 代码，但是默认情况下有 error 时标准输出只会有 error 信息，一个小技巧是用 dump 方法，例如：

func::FuncOp &func
func.dump();


这个报错大概率是因为某个 op 用到了未定义的操作数，未定义可能是由于引用了后面的 SSA Value，或者引用了其他 Region 的 SSA Value，例如如下 mlir 代码：

      %10 = scf.while (%arg4 = %7) : (index) -> index {
        %11 = arith.cmpi slt, %7, %9 : index
        scf.condition(%11) %arg4 : index
      } do {
      ^bb0(%arg4: index):
        %11 = scf.for %arg5 = %7 to %9 step %c1 iter_args(%arg6 = %6) -> (f64) {
          %13 = arith.addi %arg5, %7 : index
          %14 = memref.load %1[%13] : memref<?xindex>
          %15 = memref.load %2[%13] : memref<?xf64>
          %16 = memref.load %3[%14] : memref<64xf64>
          %17 = arith.mulf %15, %16 : f64
          %18 = arith.addf %arg6, %17 : f64
          scf.yield %18 : f64
        } {"Emitted from" = "linalg.generic"}
        %12 = arith.addi %7, %c8 : index
        scf.yield %12 : index
      } attributes {"Emitted from" = "linalg.generic"}
      memref.store %11, %4[%arg3] : memref<32xf64>

其中 memref.store 用到的 %11 未定义。

具体的解决方法就是将问题 op 删除，或者将其移动到正确位置。那么问题来了，要怎么写代码来实现这两个操作呢？且听我细细道来。
