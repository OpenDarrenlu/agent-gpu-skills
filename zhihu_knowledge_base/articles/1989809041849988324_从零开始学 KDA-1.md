chunk_kda

对于 KDA 的基本公式而言：




不难发现，这个公式换算到 FMA的等效计算是 6 次 headdim x headdim 的 CUDA Cores 计算。读者可能忽视这个的计算复杂度，对于 head dim 128而言，这个在现代 GPU 上面是纯粹的 compute bound. 如果按这个写法来计算，KDA 很难在 Prefill打满访存带宽，被 cuda cores bound 住了，同时，由于 KDA同时拥有 deltarule 和遗忘门，这使得使用这个公式训练必须间隔存储 KDA的中间状态（类似于 KV Cache， 每个头大小是 128 x 128 的 FP32 矩阵）。这进一步加深了训练深层网络的显存压力和访存压力。初步估算内存带宽的极限大概是 1T/s。

在 KDA 的原始遗忘门激活函数下，使用简单点的内核还会引入数值不稳定的问题，这是因为使用检查点，反向会需要使用除法从后往前重建中间时刻的 St, 这也是无法接受的。在上述两个原因下，我们开源了 chunk_kda 的初始版本，这个版本使用了多种数值计算的方法，将 CUDA Cores 的计算压力变为 TensorCore 的访存压力。

理解 KDA 并行
暂时忽略遗忘门

上述公式给很多读者带来了比较大的心智负担。我们可以从下述角度来得到该公式的非完全并行变种。（小写 u 代表每个时刻的，大写U代表每16个Token的某种 pack矩阵）

假设我们的更新公式是没有遗忘门门控的，那么更新公式如下：S_t = S_{t-1} + \beta_t k_t v_t^\top - \beta_t k_t k_t^\top S_{t-1}
如果我们想利用矩阵乘，我们可以构造它的简化公式（最简单的linear attention） S_t = S_{t-1} + k_t \cdot u_t^\top ，如果我们能够构造/求解解出来这个 u ，我们就可以进一步转换 每 16 个时刻的u成U矩阵乘的方式，从而获得现代硬件的免费午餐(TensorCore 要求 M最小是16）。求解和构造这个是自然且简单的。

比较两个公式，我们会有如下的关系
k_t u_t^\top = \beta_t k_t v_t^\top - \beta_t k_t k_t^\top S_{t-1}
u_t^\top = \beta_t (v_t^\top - k_t^\top S_{t-1})

我们转置一下方便理解：
u_t = \beta_t (v_t - S_{t-1}^\top k_t)

我们期望得到每 16 个 token 一组的更新公式来利用硬件算力，我们展开公式，并把它称作 CHUNK 并行公式，公式如下：
S_{t-1} = S_{initial} + \sum_{j=0}^{t-1} k_j u_j^\top
带入到上面 u_t 的式子里面：，可以得到下面的这个关系
u_t = \beta_t v_t - \beta_t \left( S_{initial}^\top k_t + \sum_{j=0}^{t-1} (u_j k_j^\top) k_t \right)
整理一下成标准线性方程组的形式：
u_t + \beta_t \sum_{j=0}^{t-1} (k_t^\top k_j) u_j = \beta_t (v_t - S_{initial}^\top k_t)
相当于：
u_t + \sum_{j < t} \underbrace{\left[ \beta_t k_t^\top k_j \right]}_{\text{Interaction Matrix } L_{tj}} u_j = \beta_t (v_t - S_{initial}^\top k_t)
现在这个公式有良好的数学性质，因为这是一个标准的线性方程组：
(I + L) \cdot U = V_{base}
这里的U就是每个小的u_t拼起来
L = \text{tril}\left( (\beta \odot K) \cdot K^\top \right)
对这个严格下三角 I+L 求逆就能解出来 U 了，然后就可以chunk并行了！

把遗忘门加回来

在这个时刻，细心的读者可能发现，我们还没有考虑遗忘门（forget gate）对 St 更新的影响。我们在这里引入考虑，在 Kimi Linear 的模型结构中，g 在对数域的取值范围是(-无穷，0)，这会有一些细小的心智负担，但在本文并不考虑。

首先还是看初始状态对最后结果的贡献：

初始的状态经过了所有的衰减
S_{part1} = S_{in} \odot (g_0 \odot \dots \odot g_{C-1})^\top
对应代码：S_next = S_initial * total_decay。

后面每一个 \mathbf{k}_t \mathbf{u}_t^\top ，都会对结果产生贡献，我们考虑简单的从前往后扫的情况，假设当前t这个时刻已经更新到S里面了。
这个更新被写入状态后，会随着时间推移，继续经历t+1, t+2, \dots, C-的衰减。
t=0 产生的项，经历了 g_1 \dots g_{C-1} 。
t=1产生的项，经历了 g_{2} \dots g_{C-1} 。
t=C-1(最后一步) 产生的项，没有经历后续衰减（或者说经历了单位衰减）。
所以，最终累加到里的项是：

\Delta S = \sum_{t=0}^{C-1} \left( \prod_{j=t+1}^{C-1} \text{Diag}(g_j) \right) (\mathbf{k}_t \mathbf{u}_t^\top)
这里的连乘 prod_{j=t+1}^{C-1} g_i 可以用“每个 chunk 内总衰减”除以“直到当前 token t的衰减”来计算。
\text{DecayRemaining}_t = \frac{\prod_{j=0}^{C-1} g_j}{\prod_{j=0}^{t} g_j} 在 Log 空间中，就是减法： \log(\text{DecayRemaining}_t) = \text{SumTotal}_g - \text{CumSum_t}_g




Torch 实现

将上述公式翻译成代码：

def naive_parallel_kda_fwd(
    q: torch.Tensor,
    k: torch.Tensor,
    v: torch.Tensor,
    g: torch.Tensor,
    beta: torch.Tensor,
    scale: float | None = None,
    initial_state: torch.Tensor | None = None,
    output_final_state: bool = False,
    chunk_size: int = 16,
):
    q, k, v, g, beta = map(lambda x: x.float(), [q, k, v, g, beta])

    B, T, H, K, V = v.shape[0], q.shape[1], q.shape[2], k.shape[-1], v.shape[-1]

    if scale is None:
        scale = K ** -0.5
    q = q * scale

    assert T % chunk_size == 0, f"T={T} 不能被 chunk_size={chunk_size} 整除"

    n_chunks = T // chunk_size

    # 调整维度顺序方便并行: [B, H, n_chunks, chunk_size, D]
    def to_chunk(x):
        return x.view(B, T, H, -1).transpose(1, 2).view(B, H, n_chunks, chunk_size, -1)

    q_chunk = to_chunk(q)
    k_chunk = to_chunk(k)
    v_chunk = to_chunk(v)
    g_chunk = to_chunk(g)
    beta_chunk = to_chunk(beta).squeeze(-1)

    if initial_state is None:
        S = torch.zeros(B, H, K, V, device=q.device, dtype=torch.float)
    else:
        S = initial_state.clone().float()

    outputs_all = []
    S_all = []

    for i in range(n_chunks):
        qc = q_chunk[:, :, i]
        kc = k_chunk[:, :, i]
        vc = v_chunk[:, :, i]
        gc = g_chunk[:, :, i]
        bc = beta_chunk[:, :, i].unsqueeze(-1)

        k_source = kc * bc

        # 这里为了简单，先用了for循环来写 decay 项。实际是矩阵乘
        L = torch.zeros(B, H, chunk_size, chunk_size, device=kc.device, dtype=kc.dtype)
        M_qk = torch.zeros(B, H, chunk_size, chunk_size, device=k_chunk.device, dtype=k_chunk.dtype)
        for t in range(chunk_size):
            for j in range(t + 1):
                if j == t:
                    decay_vector = torch.ones(B, H, K, device=kc.device, dtype=kc.dtype)
                else:
                    decay_vector = gc[:, :, j+1:t+1, :].sum(dim=-2).exp()

                qk = (qc[:, :, t] * kc[:, :, j] * decay_vector).sum(dim=-1)
                M_qk[:, :, t, j] = qk

                if j < t:
                    # k_source[t] @ k[j].T * decay
                    kk_val = (k_source[:, :, t] * kc[:, :, j] * decay_vector).sum(dim=-1)
                    L[:, :, t, j] = kk_val

        # 求逆
        # 计算 I + L 的标准逆
        I = torch.eye(chunk_size, device=L.device, dtype=L.dtype).repeat(L.size(0), L.size(1), 1, 1)
        IL = I + L
        IL_inv = torch.linalg.inv(IL)

        g_sum_total = gc.sum(dim=-2, keepdim=True)
        g_cumsum = gc.cumsum(dim=-2)
        total_decay = g_sum_total.squeeze(-2).exp()
        decay_rem = (g_sum_total - g_cumsum).exp()
        decay_acc = g_cumsum.exp()

        k_decayed = kc * decay_acc
        s_proj = torch.matmul(k_decayed, S)
        v_eff = bc * (vc - s_proj)
        U = IL_inv @ v_eff

        q_decayed = qc * decay_acc
        o_inter = torch.matmul(q_decayed, S)
        o_intra = torch.matmul(M_qk, U)
        o_chunk = o_intra + o_inter
        outputs_all.append(o_chunk)

        k_restored = kc * decay_rem
        delta_s = torch.matmul(k_restored.transpose(-1, -2), U)

        # 这里会导致并行度上不去
        S_all.append(S)
        S = S * total_decay.unsqueeze(-1) + delta_s

    o_final = torch.cat(outputs_all, dim=2)
    o_final = o_final.transpose(1, 2)
    S_all_final = torch.cat([s.unsqueeze(0) for s in S_all], dim=0)

    if output_final_state:
        return o_final, S, S_all_final
    else:
        return o_final, None, S_all_final




到此时此刻，我们已经实现了一个超快的 fwd，下一步就是写一个 triton 来实现它了。这种计算方法的推导是简单且易于理解的。

遗忘门的问题

从循环到 TensorCore：forget gate 的数值雷区

在循环实现中，我们看似安全地写着：

decay = gc[j+1:t+1].sum().exp()   # exp(负数) ∈ (0,1]

但当我们试图将其拍平为两个矩阵乘、喂给 TensorCore 时，公式必须改写为：

L_{ij} = \underbrace{k_i \beta_i \exp\left(\frac{g_i}{2}\right)}_{\text{left factor}} \cdot \underbrace{k_j \exp\left(-\frac{g_j}{2}\right)}_{\text{right factor}}

虽然 g_i, g_j ≤ 0，但 exp(-g_j/2) 中的负号会把符号翻转。一旦某个 g_j < -85，exp(-g_j/2) 立刻超过 `exp(85)`，在 bfloat16 中直接变成nan。使得出来的结果是未定义的。

这样左右两个因子是数值安全的，但是这个在实践训练中会引入 loss spike，所以在 chunk_kda 的实现中，由于完全并行，我们可以构造 类似 causal attention matrix，只有对角线的块会遇到该问题，我们二次切分快，并在不能并行的地方回到 cuda cores 计算。
