背景

接上一篇，n_route_experts等于2的次幂时（1，2，4，8，16，32，64，128，256）向量版的softmax算子

topkGatingSoftmax：fuses the softmax, max and argmax into a single kernel

优化点：

去掉smem依赖，warp内共享数据，利用warp级reduce原语
每个warp可以处理多行数据，每行数据形成thread group组（sub-group）
softmax，max，argmax融合一个kernel

kernel launch参数

gridDim：每个warp可以处理多行(多个原始token），在kernel launch的时候首先计算需要num_warps（num_tokens对ROWS_PER_WARP做roundup）；计算出num_warps之后，再做一次对WARPS_PER_TB做roundup。
WARPS_PER_TB: 这里TB就是thread block或者cta
blockDim: 128个线程
blockIdx.x: [0, num_blocks)
threadIdx.x: [0, 32)
threadIdx.y: [0, 4)
template <int EXPERTS, int WARPS_PER_TB>
void topkGatingSoftmaxLauncherHelper(
    const float* input,
    const bool* finished,
    float* output,
    int* indices,
    int* source_row,
    const int num_rows,
    const int k,
    const int start_expert,
    const int end_expert,
    cudaStream_t stream) {
  static constexpr std::size_t MAX_BYTES_PER_LDG = 16;

  static constexpr int BYTES_PER_LDG = MIN(MAX_BYTES_PER_LDG, sizeof(float) * EXPERTS);
  using Constants = detail::TopkConstants<EXPERTS, BYTES_PER_LDG>;
  static constexpr int VPT = Constants::VPT;
  static constexpr int ROWS_PER_WARP = Constants::ROWS_PER_WARP;
  const int num_warps = (num_rows + ROWS_PER_WARP - 1) / ROWS_PER_WARP;
  const int num_blocks = (num_warps + WARPS_PER_TB - 1) / WARPS_PER_TB;

  dim3 block_dim(WARP_SIZE, WARPS_PER_TB);
  topkGatingSoftmax<VPT, EXPERTS, WARPS_PER_TB, BYTES_PER_LDG><<<num_blocks, block_dim, 0, stream>>>(
      input, finished, output, num_rows, indices, source_row, k, start_expert, end_expert);
}
about rows	meaning
ROWS_PER_WARP	rows manipulated within a warp
ROWS_PER_CTA	rows manipulated within a thread block
thread_row_in_warp	continuous threads within a warp are grouped together to work on a single row, so there' re threadIdx.x / THREADS_PER_ROW sub-groups, each sub-group works on a single row.
thread_row_in_warp means i-th row within a warp
thread_row	the i-th row in global view

sub-group中thread加载方式是interleaved，每个线程加载使用128bit向量指令加载。当n_routed_experts为256，每个线程加载4个expert，1个warp一共加载128个expert。1行数据是不能跨warp的，需要多次迭代，迭代次数就是LDG_PER_THREAD。

We pack multiple rows into a single warp

float row_chunk[8];

typedef AccessType float[4];

每个LDG加载row_chunk的4个float元素，迭代器ii的for-loop每次循环发出1个LDG指令。

gmem加载时，THREADS_PER_ROW个线程以interleaved方式加载连续4个float元素。同1个线程在for-loop循环的下一次迭代中加载的是stride为THREADS_PER_ROW * ELTS_PER_LDG的元素。

注：没有smem共享数据，所有kernel内部的变量都是register。

  // Now, we compute the group each thread belong to in order to determine the first column to start loads.
  const int thread_group_idx = threadIdx.x % THREADS_PER_ROW;
  const int first_elt_read_by_thread = thread_group_idx * ELTS_PER_LDG;
  const float* thread_read_ptr = thread_row_ptr + first_elt_read_by_thread;

  // Determine the pointer type to use to read in the data depending on the BYTES_PER_LDG template param. In theory,
  // this can support all powers of 2 up to 16.
  // NOTE(woosuk): The original implementation uses CUTLASS aligned array here.
  // We defined our own aligned array and use it here to avoid the dependency on CUTLASS.
  using AccessType = AlignedArray<float, ELTS_PER_LDG>;

  // Finally, we pull in the data from global mem
  float row_chunk[VPT];
  AccessType* row_chunk_vec_ptr = reinterpret_cast<AccessType*>(&row_chunk);
  const AccessType* vec_thread_read_ptr = reinterpret_cast<const AccessType*>(thread_read_ptr);
#pragma unroll
  for (int ii = 0; ii < LDG_PER_THREAD; ++ii) {
    row_chunk_vec_ptr[ii] = vec_thread_read_ptr[ii * THREADS_PER_ROW];
  }
计算max
float row_chunk[8];

首先线程内部先reduce求出线程的最大值thread_max，然后通过warp级的reduce原语得到整个warp的max，也是thread_max哈。到这里warp内部的thread_max同步完毕。

  // First, we perform a max reduce within the thread. We can do the max in fp16 safely (I think) and just
  // convert to float afterwards for the exp + sum reduction.
  float thread_max = row_chunk[0];
#pragma unroll
  for (int ii = 1; ii < VPT; ++ii) {
    thread_max = max(thread_max, row_chunk[ii]);
  }

// Now, we find the max within the thread group and distribute among the threads. We use a butterfly reduce.
#pragma unroll
  for (int mask = THREADS_PER_ROW / 2; mask > 0; mask /= 2) {
    thread_max = max(thread_max, SGLANG_SHFL_XOR_SYNC_WIDTH(0xffffffff, thread_max, mask, THREADS_PER_ROW));
  }

计算sum

跟刚才计算max类似，线程内部计算exp（x-max）和row_sum；然后通过warp级reduce原语得到整个warp的row_sum。到这里warp内部的row_sum同步完毕。

  // From this point, thread max in all the threads have the max within the row.
  // Now, we subtract the max from each element in the thread and take the exp. We also compute the thread local sum.
  float row_sum = 0;
#pragma unroll
  for (int ii = 0; ii < VPT; ++ii) {
    row_chunk[ii] = expf(row_chunk[ii] - thread_max);
    row_sum += row_chunk[ii];
  }

// Now, we perform the sum reduce within each thread group. Similar to the max reduce, we use a bufferfly pattern.
#pragma unroll
  for (int mask = THREADS_PER_ROW / 2; mask > 0; mask /= 2) {
    row_sum += SGLANG_SHFL_XOR_SYNC_WIDTH(0xffffffff, row_sum, mask, THREADS_PER_ROW);
  }

计算softmax

每个线程用exp(x-max)/row_sum，到这里每个线程得到row（原始token）中每个expert对应的softmax分数

reciprocal_row_sum：row_sum倒数，softmax计算的分母
  const float reciprocal_row_sum = 1.f / row_sum;

#pragma unroll
  for (int ii = 0; ii < VPT; ++ii) {
    row_chunk[ii] = row_chunk[ii] * reciprocal_row_sum;
  }
find topk

跟上一篇中算法类似，下面多重循环中最外层(迭代器k_idx)对应topk k-idx位置，当k_idx为0时，计算的是output[token-id][0]；当k_idx为1时，计算的是output[token-id][1]，以此类推。

但是，这个算法视角是warp级别的，每行数据（原始token）一定是同一个warp处理的。

We pack multiple rows into a single warp

这一段代码就是找到当前行的第k_idx个expert (k_idx < topk)，跳过已选中expert代码在最后实现。

跟max/sum过程类似，都是先在线程内部reduce出一个当前softmax值最大的value和对应的expert；然后通过warp级reduce原语选出warp（sub-group）中softmax值最大的value和对应的expert。到这里，当前行对应的k_idx个expert就选出来了呀。

注1：为了实现warp内一致的语义，当softmax值相等时选择expert id小的expert。

注2：为了简化代码，把这段代码记作find_k_idx_expert

    float max_val = row_chunk[0];
    int expert = start_col;
#pragma unroll
    for (int ldg = 0, col = start_col; ldg < LDG_PER_THREAD; ++ldg, col += COLS_PER_GROUP_LDG) {
#pragma unroll
      for (int ii = 0; ii < ELTS_PER_LDG; ++ii) {
        float val = row_chunk[ldg * ELTS_PER_LDG + ii];

        // No check on the experts here since columns with the smallest index are processed first and only
        // updated if > (not >=)
        if (val > max_val) {
          max_val = val;
          expert = col + ii;
        }
      }
    }
// Now, we perform the argmax reduce. We use the butterfly pattern so threads reach consensus about the max.
// This will be useful for K > 1 so that the threads can agree on "who" had the max value. That thread can
// then blank out their max with -inf and the warp can run more iterations...
#pragma unroll
    for (int mask = THREADS_PER_ROW / 2; mask > 0; mask /= 2) {
      float other_max = SGLANG_SHFL_XOR_SYNC_WIDTH(0xffffffff, max_val, mask, THREADS_PER_ROW);
      int other_expert = SGLANG_SHFL_XOR_SYNC_WIDTH(0xffffffff, expert, mask, THREADS_PER_ROW);

      // We want lower indices to "win" in every thread so we break ties this way
      if (other_max > max_val || (other_max == max_val && other_expert < expert)) {
        max_val = other_max;
        expert = other_expert;
      }
    }
LDG_PER_THREAD	how many LDG instructions issued by single thread
COLS_PER_GROUP_LDG	column stride for per LDG instruction

执行到这里，所有线程的expert都是刚刚选中的k_idx个expert，通过sub-group中的0号线程把结果保存起来，过程跟上个文章类似哈。

注3：为了简化代码，把这段代码记作store_k_idx_expert

如何跳过已选中的expert：把对应的softmax值清掉啦。见证奇迹时刻到了！

执行到这里warp内这个sub-group中的所有线程得到一致的expert变量。所以，store_k_idx_expert和thread_to_clear_in_group在sub-group内所有线程看到都是一致的。但是row_chunk是register，每个线程加载的数据在gmem上的offset是不一样的。

在我们的场景中n_routed_experts为256，THREADS_PER_ROW为32，那么32线程中只有1个线程需要把row_chunk对应的expert清掉。那个线程就是在k_idx轮次中胜出的（maxnum）的线程，也就是expert变量对应的线程。

  // Now, we compute the group each thread belong to in order to determine the first column to start loads.
  const int thread_group_idx = threadIdx.x % THREADS_PER_ROW;
  const int first_elt_read_by_thread = thread_group_idx * ELTS_PER_LDG;
  const float* thread_read_ptr = thread_row_ptr + first_elt_read_by_thread;  

  // Now, softmax_res contains the softmax of the row chunk. Now, I want to find the topk elements in each row, along
  // with the max index.
  int start_col = first_elt_read_by_thread;
  static constexpr int COLS_PER_GROUP_LDG = ELTS_PER_LDG * THREADS_PER_ROW;

  for (int k_idx = 0; k_idx < k; ++k_idx) {
    find_k_idx_expert();
    store_k_idx_expert();

    // Finally, we clear the value in the thread with the current max if there is another iteration to run.
    if (k_idx + 1 < k) {
      const int ldg_group_for_expert = expert / COLS_PER_GROUP_LDG;
      const int thread_to_clear_in_group = (expert / ELTS_PER_LDG) % THREADS_PER_ROW;

      // Only the thread in the group which produced the max will reset the "winning" value to -inf.
      if (thread_group_idx == thread_to_clear_in_group) {
        const int offset_for_expert = expert % ELTS_PER_LDG;
        // Safe to set to any negative value since row_chunk values must be between 0 and 1.
        row_chunk[ldg_group_for_expert * ELTS_PER_LDG + offset_for_expert] = -10000.f;
      }
    }
  }

回过头来看这个实现，同一个row的expert是在同一warp（sub-group）中处理的，reduce操作可以依赖warp级reduce原语。所以，这个实现中没有使用smem变量。
