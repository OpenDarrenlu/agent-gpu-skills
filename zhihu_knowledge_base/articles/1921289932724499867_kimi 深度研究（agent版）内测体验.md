幻觉 vs 清醒剂

我觉得应对 LLM 幻觉和安全，是最能体现硬核技术和产品思维差异的两个地方。

agent 这类软件很容易混淆实体词，例如论文 SeedBench 和 SEEDBench 侧重点不同；叫 SeedLLM 跟头条没一毛钱关系，我们的 seed 就是本意“植物种子”。

按造应用的常理说，是不是应该做个图谱啊、formally 表达啊，靠 CRUD pipeine 的方式，把重要的实体词先隔离开；再在 retrieval 期间避免语义冲突？

但 kimi 不，扎扎实实地，在一轮又一轮的 retrieval 中区分实体间的概念。

虽然最后的报告里写了头条的东西... 但这个技术是对味的！最终报告还写了论文不存在的“种子包衣”，也就那样了，用户不会骂娘的

0x03 高考志愿填报

问个热点问题，高考报新闻学，还是计算机？

高考刚出分，我是江苏考生，全国卷总分 606 分，选科组合是历史政治地理。我想填报上海高校的新闻学专业，不知道专业前景如何。
- 请帮我出一份新闻学就业的竞争力报告。希望能和计算机专业做对比。
- 我这个分数也不高，到底是选个好学校更好，还是应该选个差点的学校，但专业更好。

我咨询了旁边的江苏实习生，模型对大学排名的认知不太对哈，日志里有的在江苏差了 2 个档次（看执行日志，最终结果没体现）。

生成的 UI 是真不错，但 html 需要处理一下手机屏幕适配，这很重要
对这个话题，生成的结论更激进、直白点好。车轱辘话产品至少在国内没用
