回顾历史

2013 年，我还是一名软粉。作为读大二的穷学生，月生活费只有 1000 的我斥巨资 2000 多元买下了一个丐版 Surface RT。

Surface RT 淘宝订单

当时的我手里拿着 Windows Phone，桌上放着 Surface RT，一心想着赶 Metro App 开发的“潮流”。虽然后来这个“潮流”并没有出现。。。

Surface RT

他陪伴我度过了躺床上玩舰娘的日子，陪我在回家的火车上看完了 C# language specification，陪我开发了 Tomato Music。虽然他 4 核 A9 + 2GB DDR3L 1500 的性能在 2023 年来看有点羸弱，但我还是割舍不下他（其实是咸鱼才能卖 200，有点亏）。

安装 Windows 10 on ARM

2019 年泄露出一版 Windows 10 Creators Update Build 15035，这也是唯一一版 ARMv7 指令的 Windows 10。后续有大佬从后续版本的 WinPE 中拿了一些组件过来，版本号更新到了 17763，不过系统的主要组件还是 15035。

这个版本因为从来没有正式发布过，有一些问题，比如驱动问题，不支持被远程桌面连接等，详细的列表可以在Open Surface RT 找到。

存在的问题
实在是太卡了，4 核 A9 + 2GB 内存跑 Win10 实在是有点难为他了。资源管理器启动都会白一会，Word 打字反应也很慢。于是我放弃了在 Surface RT 上继续使用 Win10。
安装 Raspberry Pi OS

Surface RT 使用的是 Nvidia 的 Tegra 3 SoC。有一个叫 grate-driver/linux 的 fork 增加了不少 Tegra SoC 的支持。Open Surface RT 项目也向其中 upstream 了 Surface RT 的 device treee 和一些驱动。于是我们可以在 Surface RT 上 boot linux。

预编译好的 linux kernel 可以从 https://openrt.gitbook.io/open-surfacert/surface-rt/linux/kernel/prebuilt-binaries 下载。目前最新的 kernel 是 5.17。

存在的问题

因为 Surface RT UEFI 固件的问题，L2 Cache、动态调压等功能需要 kernel 通过 ACPI 去控制，但我们用的是 device tree，所以这些功能都没有。尤其是没有 L2 Cache 导致性能很差。通过 APX 启动可以解决这个问题，但是太折腾了，每次启动都需要一台 linux PC 把 UBoot 传进去。

2023 年的 Windows RT 8.1 生态如何呢？

应用商店已经装不了东西了，不过有很多大佬们移植的 Desktop App。

压缩解压 - 7Zip

官方支持 7zip for Windows on ARM，点 Download 7zip 22.01 .exe installer (32-bit ARM32)。

7Zip
文本编辑 - Notepad 3

目前的版本是 5.21.1129.1。

Notepad 3
PDF 阅读 - SumatraPDF

新版本需要 WebView2，所以能用的版本最多到 3.3.3。

SumatraPDF
远程访问 - TightVNC

因为 ARMv7 版本的 Windows 不支持接入远程桌面（但可以远程访问其他机器），只能用 VNC 之类的替代。TightVNC 在 Win 上的 Server 已经很久不维护了，似乎被 UltraVNC 取代了。有一些 bug 先凑合用，以后有空可以移植下 UltraVNC。

TightVNC Server
SNES 模拟器 - Snes9X

有个 1.53 的版本，实测可以玩牧场物语，有点小卡。以后有空可以优化下。

Snes9X
模拟器平台 - RetroArch

界面非常的卡，进了游戏还可以接受，似乎是用了 GLDirect 的原因。SurfaceRT 不支持 opengl，Direct3D11 也只支持 featureLevel 9.1。

RetroArch
DOS 模拟器 - DOSBox-X

又是一个官方支持。实测巨卡无比，Win98装了几个小时没装完。似乎是因为 dynrec 的 JIT 引擎在 RT 下有问题，RT 只支持 Thumb-2 指令。有 issue 提到 Win10 ARM 支持 ARMv7 指令是没问题的。

DOSBox-X
