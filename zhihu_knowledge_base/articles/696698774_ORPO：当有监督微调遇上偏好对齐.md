# ORPO：当有监督微调遇上偏好对齐

**作者**: 养生的控制人​浙江大学 控制科学与工程博士

**原文链接**: https://zhuanlan.zhihu.com/p/696698774

---

大语言模型通常会有预训练、有监督微调（supervised fine-tuning, SFT）、偏好对齐（preference alignment）三个训练阶段。虽然说在很多应用场景下有监督微调就已经够用了，但对于一些面向用户的公众模型，偏好对齐还是很有必要的（不然模型说了什么不该说的话可能这个产品甚至公司都要完蛋）。

比较经典的偏好对齐的做法就是基于人类反馈的强化学习（RLHF）那一套，根据人类偏好/反馈数据训练一个“奖励模型”，并使用该模型作为强化学习中的奖励函数，再通过类似PPO之类的强化学习算法来优化大语言模型的输出。

不过对于大多数非公司级的大语言模型来说，不愿意折腾RLHF，毕竟偏好数据不好收集、还要额外训一个奖励模型、还要搞训练不稳定的强化学习。这些成本都是很高的。

当然，都2024年了，RLHF的替代已经有不少了，比如有用AI来收集偏好数据的RLAIF，有省略奖励函数建模的直接偏好优化 (DPO)，今天的主角则是考虑将偏好对齐融入SFT阶段的几率比偏好优化（ORPO，odds ratio preference optimization）。

\mathbf{OR}_\theta(y_w,y_l)=\frac{\mathbf{odds}_\theta(y_w|x)}{\mathbf{odds}_\theta(y_l|x)} \\

对于偏好对齐来说，我们就是希望提高比较好的输出的概率，降低比较不好的输出的概率，也就是最大化\mathbf{OR}_\theta(y_w,y_l)。

对于除法一般难计算，那我们取个对数就可以将除法转为乘法了！考虑到深度学习里我们一般都是去最小化某个目标函数，因此可以考虑再套一个负的log sigmoid函数，把最大化问题转为最小化问题

\mathcal{L}_{OR}=-\log\sigma\left(\log\frac{\mathbf{odds}_\theta(y_w|x)}{\mathbf{odds}_\theta(y_l|x)}\right) \\

这样我们就可以最小化\mathcal{L}_{OR}来实现偏好对齐了！

当然这里跟SFT并没有脱离开，我们只需要把这一项加到SFT的目标函数中，就能够实现在SFT的阶段进行偏好对齐

\mathcal{L}_{ORPO}=\mathbb{E}_{(x,y_w,y_l)}\left[\mathcal{L}_{SFT}+\lambda\cdot\mathcal{L}_{OR}\right] \\

其中SFT的目标函数就是常规的交叉熵损失

\begin{aligned}\mathcal{L}&=-\frac{1}{m}\sum_{k=1}^m\log P(\mathbf{x}^{(k)},\mathbf{y}^{(k)})\\[1ex]&=-\frac{1}{m}\sum_{k=1}^m\sum_{i=1}^{|V|}y_i^{(k)}\cdot\log(p_i^{(k)})\end{aligned} \\

不过，虽然ORPO能够将偏好对齐融入SFT阶段，也存在一定的局限性，那就是常规的SFT数据集不好用了，得是偏好数据集才能来进行ORPO训练。

代码实现

理解上面的原理后，代码实现看起来还是比较直观的

把正负样本分别送入模型，正样本就是“好”的，负样本就是相对“不好”的

outputs_neg = model(**{'input_ids': inputs['negative_input_ids'],
                               'attention_mask': inputs['negative_attention_mask'],
                               'labels': neg_labels,}, output_hidden_states=True)      
outputs_pos = model(**{'input_ids': inputs['positive_input_ids'],
                               'attention_mask': inputs['positive_attention_mask'],
                               'labels': pos_labels,}, output_hidden_states=True)


先拿到常规SFT的损失

# Calculate NLL loss
pos_loss = outputs_pos.loss


计算输出的对数概率

def compute_logps(self, prompt_attention_mask, chosen_inputs, chosen_attention_mask, logits):
    mask = chosen_attention_mask[:, :-1] - prompt_attention_mask[:, 1:]
    per_token_logps = torch.gather(logits[:, :-1, :].log_softmax(-1), dim=2, 
                                       index=(mask * chosen_inputs[:, 1:]).unsqueeze(2)).squeeze(2)
    return torch.mul(per_token_logps, mask.to(dtype=torch.bfloat16)).sum(dim=1).to(dtype=torch.float64) / mask.sum(dim=1).to(dtype=torch.float64)
        
# Calculate Log Probability
pos_prob = self.compute_logps(prompt_attention_mask=inputs['attention_mask'], 
                                  chosen_inputs=inputs['positive_input_ids'], 
                                  chosen_attention_mask=inputs['positive_attention_mask'], 
                                  logits=outputs_pos.logits)
neg_prob = self.compute_logps(prompt_attention_mask=inputs['attention_mask'], 
                                  chosen_inputs=inputs['negative_input_ids'], 
                                  chosen_attention_mask=inputs['negative_attention_mask'], 
                                  logits=outputs_neg.logits)


计算对数几率

# Calculate log odds
log_odds = (pos_prob - neg_prob) - (torch.log(1 - torch.exp(pos_prob)) - torch.log(1 - torch.exp(neg_prob)))
sig_ratio = torch.nn.functional.sigmoid(log_odds)
ratio = torch.log(sig_ratio)


融入偏好对齐的SFT最终损失由两部分组成

# Calculate the Final Loss
loss = torch.mean(pos_loss - self.alpha * ratio).to(dtype=torch.bfloat16)
