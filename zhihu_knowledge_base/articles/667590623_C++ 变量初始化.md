1. 使用 auto
不要使用类型前置，如果没有初始化会躲避编译器的检查

auto关键字要求变量在声明时必须进行初始化，否则会导致编译错误。

struct C {
    int a;
    int b;
};

int main(int argc, char ** argv) {
    auto c = C{};
    cerr << c.a << endl;
    return 0; 
}


C 类型的对象是作为栈上的局部变量创建的。

2.使用智能指针

有可能出错的初始化方式：

auto p1 = new C
对于内置类型（如 int, float, char 等）和 POD（Plain Old Data，即没有构造函数、析构函数、虚函数等的旧式结构体或类）类型的对象，使用 new 进行默认初始化时，并不会对成员进行初始化。这意味着这些成员的值是未定义的
auto c = new C;
cerr << c->a << endl;


make_unique的好处，保证new和构造unique_ptr不会被别的异常打断，从而达成异常安全、避免资源泄露

auto c = make_unique<C>();

3.使用类型后置

可以有效的防止隐式类型转化

以下代码可能会陷入死循环
    auto a = std::vector<int>(0xfffffffffff);
    for(auto i = 0; i < a.size(); ++i) { 
        cerr << a[i];
    }

使用类型后置
    for(auto i = size_t(0); i < a.size(); ++i) { 
        cerr << a[i];
    }

使用 static_cast 可以防止传入 nullptr
    for(auto i = static_cast<size_t>(0); i < a.size(); ++i) { 
        cerr << a[i];
    }

4.字面常量
const char *无法相加
    auto s = "hello";
    s = s + "world";
    cerr << s << endl;


C++ 14 新增了字面常量

    auto s = "hello"s;
    s = s + "world"s;
    cerr << s << endl;

5.函数参数顺序解决方法

C++20 新增的特性可以有效避免默认的变量初始化，排除代码更改后可能会带来的初始化问题。

使用指定初始化器（designated initializers）可以使结构体初始化更加清晰和安全。

struct D {
    int age = 42;
    std::string name = "hello"s;
};

    auto d = D{.age = 112};
    cerr << d.age << endl;
    cerr << d.name << endl;


struct lerp {
    float from = 0;
    float to = 1;
    float fac = 0.5f;

    float operator()() const { 
        return fac * (to - from) + from;
    }
};

    auto x = lerp{.from = 0, .to = 2, .fac = 0.6f}();
    cerr << x << endl;
