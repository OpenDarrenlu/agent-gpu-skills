社区技术朋友经常将 PyTorch 动态图和TensorFlow 及 MxNet 等框架的静态图进行对比。但实际上理解 PyTorch 动态性最简单的方式是先抛弃图的概念！

根据 PyTorch 社区主页的定义，PyTorch 实际上就是一个 Python 计算库 + AutoGrad:

PyTorch is a Python package that provides two high-level features:
+ Tensor computation (like NumPy) with strong GPU acceleration
+ Deep neural networks built on a tape-based autograd system

简单说就是一个能够自动求导和运行在 GPU 上的 NumPy 库。

如果要问怎么扩展 NumPy 做深度学习？我直接告诉你，用 NumPy 定义个深度学习网络；然后给每个 ndarray 的计算加个 hook，把计算过程记录下来。再根据记录下来的前向图生成反向的计算就可以了。(当然还需要考虑模型参数更新，因为不影响讨论动态图机制，我们姑且控制下这个变量~)

如果你还要问我怎么让 NumPy 跑在 GPU 上？直接全部重写一遍 NumPy 每个计算的 GPU Kernel 不就行了 :P。相信到这大家，就明白了这个应该是期末大作业的水平(虽然这个工程量不小)。

原理上 PyTorch 是这么实现的，只不过 AutoGrad 的实现稍稍有点不同（Eager模式下，反向图是每执行一个计算就生成一部分）。

什么是动态图？

既然前面模型定义没有图啥事，为啥还要引入动态图的概念？

这要从早期的 Caffe, TensorFlow 1.x, MxNet 等深度学习框架讲起。这些框架拥有的共同点就是将深度学习网络定义为静态图。所提供的 Python API 仅仅是为了方便构造一个深度学习网络图。

这种先定义静态图再执行的方式，不仅给编程和调试设置了门槛。而且静态图的对动态性表示能力本身也存在限制。举例来说，在 TensorFlow 1.x 中定义一个 RNN 类型的网络结构，必须先确定序列的最大长度；也同时意味着对于动态的序列来说所有的序列必须对齐和 padding 到固定的大小。

相比而言，PyTorch 深度学习网络定义实实在在就是 Python 程序，以 RNN 为例:

class RNNCell(nn.Module):
    # you can also accept arguments in your model constructor
    def __init__(self, data_size, hidden_size, output_size):
        super(RNNCell, self).__init__()
        self.hidden_size = hidden_size
        input_size = data_size + hidden_size
        self.i2h = nn.Linear(input_size, hidden_size)
        self.h2o = nn.Linear(hidden_size, output_size)

    def forward(self, data, last_hidden):
        input = torch.cat((data, last_hidden), 1)
        hidden = self.i2h(input)
        output = self.h2o(hidden)
        return hidden, output

class RNN(nn.Module):
    def __init__(self):
        super(RNN, self).__init__()
        self.rnn_cell = RNNCell(50, 20, 10)
        self.loss_fn = nn.MSELoss()

    def forward(self, batch, hidden, target):
        seqlen = batch.size(0)
        batch_size = batch.size(1)
        loss = 0
        for t in range(seqlen):
            # yes! you can reuse the same network several times,
            # sum up the losses, and call backward!
            hidden, output = self.rnn_cell(batch[t], hidden)
            loss += self.loss_fn(output, target[t])
        return loss

这个例子中和静态图方式存在的一个明显的差异就是循环控制流直接使用了 Python 高层语言的 for 语句:

for t in range(seqlen):
    hidden, output = self.rnn_cell(batch[t], hidden)

讲到这里可能还有点晦涩，这些跟动态图有啥关系呢？我们再回顾下，前面说的 PyTorch 在前向执行过程中会执行轨迹相关的记录。这种方式我们通常会称为 tracing，就是代码实际执行一遍，隐藏在其后的数据流计算图(DataFlow Graph)就很容易得到了。但是这种方式会由于输入不一样，每次的 trace 可能不一样! 这就是 PyTorch 动态图。

理解 AutoGrad 的实现机制

到现在我们只说清楚了什么是动态图。究其原因还是因为没有计算图表示，自动求导难以实现。PyTorch 中在 Eager 模式下对模型进行求导的机制称为 AutoGrad。AutoGrad 机制在模型每次执行前向计算过程中，同时动态生成反向计算图；执行 loss.backward() 函数触发反向图的计算。

从概念上讲，AutoGrad 在由 Backward Function 对象组成的有向无环图 (DAG) ；记录了数据和所有执行的操作。在这个 DAG 中，叶子是输入张量，根是输出张量。 通过从根到叶子节点执行 DAG 图的过程就是使用链式法则自动计算梯度的过程。

https://pytorch.org/tutorials/beginner/blitz/autograd_tutorial.html
PyTorch Dispatcher 机制

到这可能有的已经好奇了，PyTorch 是如何通过这么简单的 API 实现这些逻辑的运行的？这里我推荐看看 PyTorch 核心开发者 Edward Z. Yang 的 Blog Let’s talk about the PyTorch dispatcher. 还有，PyTorch Github 的 Wiki PyTorch Dispatcher Walkthrough.

简单说，当我们调用 PyTorch Variable 上面的某个 API 比如 torch.add/Tensor.add 操作时，事情发生并不是那么直接（Tensor 的加法并不会直接调用到底层的数学运算），而是通过一个叫 Dispatcher 的机制完成了操作的路由转发。

AutoGrad Dispatch Example(from ezyang&#39;s blog)

对应我们这里的例子，torch.add 的 AutoGrad 的实现首先会被调用到，在这里面会执行反向图的构建；然后，在经过一系列其他的路由；最终会 dispatch 到 CPU/GPU 对应的 Kernel 实现上。整个过程，Dispatcher 按照内部的优先级，结合用户程序的全局设置及局部状态，确定路由的过程。充分体现了 PyTorch 执行的动态性。
