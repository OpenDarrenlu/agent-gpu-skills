AdaCluster：面向视频生成中稀疏注意力的自适应查询-键聚类方法

作者：Haoyue Tan, Shengnan Wang, Yulin Qiao, Juncheng Zhang, Youhui Bai,Ping Gong, Zewen Jin, Cheng Li

​Adacluster核心算法流程图
内容总结：视频扩散Transformer（DiTs）已成为视频生成领域的主流范式，凭借强大的可扩展性和高保真合成能力，广泛应用于娱乐、广告等多个领域。但该类模型存在显著的推理延迟瓶颈，核心原因在于注意力机制的二次计算复杂度——随着视频分辨率提升和帧数增加，输入序列长度会急剧增长，导致注意力计算负担过重。实验表明，在CogVideoX-2B等模型上生成高分辨率视频时，注意力计算占总生成时间的75%以上，严重限制了DiTs的实际落地应用。

为解决这一问题，现有稀疏注意力方法试图通过挖掘注意力的固有稀疏性加速计算，但普遍存在局限：部分方法忽略token间的语义相似性，仅对连续token进行分组，导致注意力精度下降；另一部分方法采用统一聚类策略，无法适配不同层token的异质分布，同样会造成模型性能受损。针对这些痛点，中国科学技术大学等团队提出AdaCluster，这是一种无需训练的自适应聚类框架，通过为查询和键设计定制化聚类策略，在大幅提升DiTs生成速度的同时，最大限度保留视频生成质量，实现效率与保真度的平衡。

本文了AdaCluster，这是一种无需训练的自适应聚类框架，为查询（Query）和键（Key）设计定制化聚类策略，其中查询聚类基于查询-键分数的相对大小与查询长度无关的特性，先归一化查询向量再采用角度相似性聚类以实现高压缩比，键聚类则针对不同层键向量分布差异大的问题，提出分层自适应K-means聚类，包含聚类数量分配、阈值自适应聚类和关键聚类高效选择，还利用相邻去噪步骤的分布一致性加速聚类过程，而TensorQuest则改进了Quest方法，适配GPU张量核心以高效识别关键聚类，相比原始方法提速最高达4.31倍。

Q&A

Q：AdaCluster 聚类效果更好，所以比 SVG2 精度和速度都更高吗？有没有指标能直接衡量聚类好坏？

A：可以算聚类误差、轮廓系数等，但没有统一公认的评估指标，而且各层聚类数不一样，很难直接对比。目前只看最终任务精度来验证效果，不单独评价聚类本身。

Q：latent token 之间在视频或图像的空间位置上是否存在相关性？图像 patch 内像素的上下左右位置关系，与 attention 中的 token 是如何对应的？

A：存在对应关系。latent token 本质就是图像 / 视频分块（patch）的特征表示，一个 token 对应一块局部区域，因此相邻 token 在空间上就是相邻的 patch，天然带有上下左右的位置关系。

Q：PPT 里说 Q 聚类只看角度、K 只看模长，原因是什么？注意力同时包含 Q、K 夹角和模长，为什么可以这样拆分？

A：因为稀疏聚类是按注意力分数排序选 K，只关心相对大小。Q 等比例缩放只会让所有 QK 内积同比例放大，不改变排序结果，所以 Q 只需要角度信息。
