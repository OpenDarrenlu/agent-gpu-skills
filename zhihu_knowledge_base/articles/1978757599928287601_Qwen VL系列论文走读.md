Qwen3-VL
一、核心定位与整体优势

Qwen3-VL 是 Qwen 系列当前最先进的视觉 - 语言模型（VLM），于 2025 年 11 月发布，核心优势体现在三大支柱：

更强的纯文本理解能力：在多项任务中超越同类纯文本模型，兼顾多模态与语言基础能力；
原生超长上下文处理：支持文本、图像、视频交织的 256K tokens 上下文，可精准检索、关联长文档与长视频信息；
先进多模态推理：在单图 / 多图 / 视频任务中表现领先，尤其在 MMMU、MathVista、MathVision 等复杂推理基准上达到 SOTA。




模型家族涵盖两类变体，适配不同 latency - 质量需求：

密集型（Dense）：2B/4B/8B/32B 参数；
混合专家（MoE）：30B-A3B（总参数 30B）、235B-A22B（总参数 235B，单 token 激活 22B 参数）。
二、模型架构升级

Qwen3-VL 延续 “视觉编码器 + VL 融合器 + LLM 解码器” 三模块架构，核心创新集中在以下三点：

2.1 关键架构改进
改进方向	具体方案	解决问题
Interleaved MRoPE	将时间（t）、水平（h）、垂直（w）维度的位置编码均匀分布在高低频带，替代原 MRoPE 的维度分区策略	原 MRoPE 频率谱不均衡，导致长视频理解性能下降
DeepStack 跨层融合	提取视觉编码器（ViT）多中间层特征，通过轻量级残差连接注入 LLM 前三层对应隐藏态	强化视觉 - 语言对齐，保留从低到高的细粒度视觉信息
文本化视频时间戳	用显式文本时间戳（如 <3.0 seconds>）标记视频帧组，支持秒 / HMS 两种格式	替代 Qwen2.5-VL 的 T-RoPE 绝对时间编码，解决长视频时间 ID 稀疏、训练数据成本高的问题




2.2 各模块细节
LLM backbone：基于 Qwen3 系列，旗舰模型 Qwen3-VL-235B-A22B 超越多数 VLM，且在文本基准上优于纯文本模型；
视觉编码器：采用 SigLIP-2 架构（动态分辨率训练），默认用 SigLIP2-SO-400M，2B/4B 小模型用 SigLIP2-Large（300M），通过 2D-RoPE 适配动态输入尺寸；
SigLIP-2: https://arxiv.org/pdf/2502.14786
CoMP: https://arxiv.org/pdf/2503.18931
VL 融合器：两层 MLP 压缩 ViT 的 2×2 特征为单视觉 token，同时支持 DeepStack 专用融合器。
三、训练流程与数据优化
3.1 预训练：四阶段渐进式训练
阶段	目标	训练对象	Token 预算	序列长度	核心动作
S0	视觉 - 语言对齐	仅 VL 融合器（其余冻结）	67B	8192	用高质量图-caption对、视觉知识集合、OCR 数据建立跨模态基础
S1	多模态预训练	全参数（视觉编码器 + 融合器 + LLM）	~1T	8192	混合 VL 数据（图文交错 / grounding / VQA / STEM/少量视频）与纯文本数据，保留语言能力
S2	长上下文预训练	全参数	~1T	32768	增加纯文本占比提升长文本理解，加入更多视频 / Agent 数据
S3	超长上下文适配	全参数	100B	262144	聚焦长视频 / 长文档任务，固化 256K 上下文能力




3.2 预训练数据：质量与多样性升级

覆盖 9 大类核心数据，关键改进如下：




图像caption：用 一个专门为recaption微调的Qwen2.5-VL-32B 重生成细粒度字幕，语义去重保留视觉多样性；数据平衡：用纯文本embedding去重，再进行聚类、识别稀疏区域、增强。
图文交错：用专门微调的轻量级内容识别器来进行内容领域分类，过滤低质量、有害数据，比如广告、促销内容和标题党；对超长的书籍规模的交错数据，进行了严格的质量控制。
世界知识：百科全书式的，定义明确的，以实体为核心的大规模数据集。并进行了基于重要性的重平衡。
OCR：支持 39 种语言（较 Qwen2.5-VL 新增 29 种），3000 万合成 + 100 万真实样本；
长文档理解：有HTML、Markdown两种解析方式，从而支持对异构格式的解析。
grounding：bbox、point、计数。坐标系采用[0,1000]的归一化坐标系。
3D 空间理解：关系标注、功能标签（可抓取、可坐）、需要规划的动作条件查询（要达到xx目的，我需要先干啥？）
3D 与定位：构建 3D 边界框标注数据集（如 Omni3D），都统一到Omni3D式虚拟相机坐标系下。
coding：多模态编码
视频数据：密集描述合成、时空视频定位；
STEM 推理：合成的大规模grounding和VQA数据集，合成和验证都是高度自动化、组合多个专用模型的；生成 6000 万 K12 / 本科习题，1200 万多模态 CoT 样本，严格过滤错误推理；包含Qwen3的所有纯文本推理数据。
Agent 数据：覆盖桌面 / 移动 / Web GUI，生成多步任务轨迹，加入工具调用（搜索 / 函数）数据。
3.3 后训练：三阶段精调与优化
监督微调（SFT）：
分两阶段：32K 上下文初始化 → 256K 上下文聚焦长文档 / 长视频；
数据拆分：“非思考” 模型用标准格式，“思考” 模型用 Chain-of-Thought（CoT）格式，共 120 万样本（1/3 纯文本，2/3 多模态）。
短文本
从开源界、互联网等各个领域整合了多个子任务，包括但不局限于具身智能的空间推理、细粒度视觉理解、基于视频中的时空定位的目标跟踪、长文本技术文档理解。
多语种
质量控制：query过滤、response过滤。
长文本
难度、多模态必要性、质量控制。
强到弱蒸馏（Strong-to-Weak Distillation）：
用强教师模型的文本输出蒸馏学生模型 LLM backbone，提升文本与多模态推理能力。




强化学习（RL）：
推理 RL：针对数学、编码、视觉 grounding 等可验证任务，用 SAPO 算法优化；过滤简单任务
通用 RL：优化指令遵循（格式 / 长度约束）与人类偏好（ helpfulness / 准确性），修正 SFT 阶段的错误先验。




“图像思考” Agent能力训练

两阶段训练，先训 Qwen2.5-VL-32B 视觉 Agent，再蒸馏生成 12 万多轮数据精调 Qwen3-VL，用 “答案准确性 + 多步推理 + 工具调用” 三重奖励约束。

四、核心评估结果

Qwen3-VL 在全尺寸模型、全任务类型上表现领先，尤其是长视频理解和文本能力保留。

空间重置。新的图片的h、w都是从0开始，而不是从t_0开始。好处在于：大模型通常有attention sink，注意力会集中在小id处，空间重置使得有更多的小id位置（我觉得这条比较牵强）；相对位置的表达更清晰，时空不会纠缠在一起；
频率交替分配。很简单，就是将t、h、w三个维度在频率维度上依次分配，形成交错局面。原生的MRoPE是块状分配的。好处在于：原本设计中，t是集中在高频的，这样会导致施加在时间上的注意力随长程快速衰减；h、w的频率表现不一样，不对称；
Omni3D虚拟相机坐标系

不同相机的内参存在差距（焦距、主点、畸变系数等），如果将来源于不同相机的数据集直接用来训一个模型，三维检测模型会因尺度 - 深度歧义性（Scale-Depth Ambiguity）无法泛化。虚拟相机坐标系的设计目标是：消除硬件差异，将所有数据投影到一个统一的、无尺度偏差的参考系中。

1. 原始数据预处理：相机标定与坐标系转换
各数据源独立标定：
对每个数据集的图像 / 点云，使用传统标定方法（如棋盘格标定板）获取内参矩阵（焦距、主点、畸变系数）和外参矩阵（相机位姿）。
例如，KITTI 的相机外参描述车辆坐标系到世界坐标系的变换，而 SUN RGB-D 的外参描述室内相机到场景坐标系的变换。
坐标转换到世界坐标系：
对每个三维物体标注（如 3D 框），使用外参矩阵将其从相机坐标系转换为世界坐标系下的绝对坐标。




2. 虚拟相机坐标系的构建
定义虚拟相机内参：
设定统一的虚拟相机内参矩阵，例如：
焦距 fvirtual​：取所有数据集焦距的中位数或均值；
主点 (cx​,cy​)：通常设为图像中心；
忽略畸变（假设数据已完成畸变校正）。
该内参矩阵不对应任何真实相机，仅作为统一投影的参考。




虚拟深度计算：
对每个世界坐标系下的点 (X,Y,Z)，计算其在虚拟相机坐标系下的归一化深度zvirtual​：
zvirtual​=Zoriginal​⋅foriginal​Z⋅fvirtual​​
其中，Zoriginal​ 是原始深度值，foriginal​ 是原始相机焦距。
该公式的本质是将原始深度按虚拟焦距进行缩放，从而消除焦距差异的影响。




3. 统一投影到虚拟相机坐标系
三维点投影：
对每个世界坐标系下的点 (X,Y,Z)，使用虚拟相机内参矩阵投影到二维图像平面：
u=zvirtual​X⋅fvirtual​​+cx​,v=zvirtual​Y⋅fvirtual​​+cy​
此时，不同数据集的点云 / 图像中的物体在虚拟相机坐标系下具有相同的像素尺度（如同一辆车在 KITTI 和 SUN RGB-D 中的投影大小一致）。




二维图像归一化：
对原始图像进行缩放和裁剪，使其分辨率与虚拟相机的图像平面匹配（如统一为 1024×1024 像素）。
同时，通过双线性插值等方法校正因缩放产生的图像畸变。
