4.4 图形处理单元/Graphics Processing Units

GPU起源于图形加速器，有着优秀的图形处理性能。

GPU编程
CUDA是解决算法中并行表示问题的一个优雅解决方案，尽管它并不适用于所有算法，但足够重要。它以某种方式与我们思考和编码的方式产生共鸣，使得在任务级别之外更容易、更自然地表达并行。
Vincent Natol, “Kudos for CUDA,” HPC Wire (2010)

GPU程序员面临的挑战不仅是在GPU上获得良好的性能，还包括协调系统处理器和GPU上的计算调度以及系统内存和GPU内存之间的数据传输。此外，GPU几乎具备编程环境可以捕捉到的所有类型的并行：多线程、MIMD、SIMD，甚至指令级并行性（ILP）。

NVIDIA开发了一种类似C的语言和编程环境，通过解决异构计算和多方面并行性的挑战，提高GPU程序员的生产力，名为CUDA（统一计算设备架构，Compute Unified Device Architecture）。CUDA为系统处理器（host）生成C/C++代码，并为GPU（device）生成C和C++方言（dialect）。所有这些形式的并行统一是CUDA线程。

编译器和硬件可以将数千个CUDA线程组合在一起，利用GPU内部的各种并行性：多线程、MIMD、SIMD和ILP。CUDA编程模型归类为单指令多线程（SIMT）。这些线程被组合在一起，并以线程块的形式执行。执行整个线程块的硬件称为多线程SIMD处理器。

在给出CUDA程序的示例之前，需要知道一些细节:

为了区分GPU（设备）上的函数和系统处理器（主机）上的函数，CUDA使用__device__或__global__来表示前者，使用__host__来表示后者。
使用__device__声明的CUDA变量被分配到GPU内存，可以被所有多线程SIMD处理器访问。
在GPU上运行的函数名的扩展函数调用语法为： name <<<dimGrid, dimBlock>>> (... parameter list…) 其中dimGrid和dimBlock指定了代码的线程块的维度和在block中线程的维度。
除了用于块的标识符blockIdx和每个块中线程的标识符threadIdx之外，CUDA还提供了一个关键字用于表示每个块的线程数blockDim，来自于dimBlock参数。

在CUDA代码之前，还是以DAXPY循环（Y=a*X+Y）的常规C代码为例。

// Invoke DAXPY
 daxpy(n, 2.0, x, y); 

// DAXPY in C 
void daxpy(int n, double a, double *x, double *y) 
{ 
 for (int i = 0 ; i < n ; ++i) 
  y[i] = a * x[i] + y[i]; 
} 

以下是CUDA版本。启动n个线程，每个向量元素一个线程，在多线程SIMD处理器中每个线程块有256个CUDA线程。GPU函数首先基于块ID、每个块的线程数和线程ID计算相应的单元索引i。只要这个索引在数组内(i < n)，就执行乘法和加法。

// Invoke DAXPY with 256 threads per Thread Block 
__host__ 
int nblocks = (n + 255) / 256; 
daxpy<<<nblocks, 256>>>(n, 2.0, x, y);

 // DAXPY in CUDA
 __global__ 
void daxpy(int n, double a, double *x, double *y) 
{ 
 int i = blockIdx.x * blockDim.x + threadIdx.x; 
 if (i < n) 
  y[i] = a * x[i] + y[i]; 
} 

通过指定grid维度和每个SIMD处理器的线程数来确定CUDA中的并行度。此外，每个元素分配单独的线程，当结果写回内存时不需要在线程之间进行同步。

GPU硬件处理并行执行和线程管理，不需要通过应用或者操作系统。为了简化硬件调度，CUDA要求线程块能够独立并以任意顺序地执行。不同的线程块不能直接通信，除非在全局内存中使用原子内存操作进行协调。

许多GPU硬件的概念在CUDA中并不明显。编写高效的GPU代码要求程序员从SIMD操作角度思考，尽管CUDA编程模型看起来像是MIMD。

同时，像许多并行系统一样，CUDA在生产力和性能之间做了妥协，包含内置函数（intrinsics），让程序员可以在硬件上指定控制。

机器对象：SIMD指令线程、SIMD指令

SIMD指令线程：只包含在多线程SIMD处理器上执行的SIMD指令的传统线程。
SIMD指令：在SIMD通道上执行的SIMD指令

硬件创建、管理、调度和执行的机器对象是SIMD指令线程。这些SIMD指令线程有独立的PC，并在多线程SIMD处理器上运行。

处理硬件：多线程SIMD处理器、线程块调度器、SIMD线程调度器、SIMD通道

多线程SIMD处理器：执行不依赖于其他SIMD处理器的SIMD指令线程的多线程SIMD处理器。
线程块调度器：将多个线程块分配给多线程SIMD处理器的单元。
SIMD线程调度器：调度并发射SIMD指令线程的单元，包括记录SIMD线程执行的scoreboard。
SIMD通道：负责在单个SIMD指令线程中执行单个元素的操作，结果根据mask存储。

由于SIMD指令线程在定义上是独立的，SIMD线程调度器可以选择任何准备好的SIMD指令线程进行执行，不用按照线程程序的顺序。SIMD线程调度器中的scoreboard，可以记录最多64个SIMD指令线程，用于确定哪个SIMD指令已经准备就绪。如图4.16所示，SIMD线程调度器以不同的顺序选择SIMD指令线程执行。

图4.16 SIMD指令线程调度

为了能够执行许多SIMD指令线程，当SIMD指令线程创建时，每个SIMD处理器会动态分配一组物理寄存器；当SIMD线程退出时，物理寄存器会被释放。

例如，可以有一个线程块用了16个SIMD线程，每个线程36个寄存器；另一个线程块用了32个SIMD线程，每个线程20个寄存器。

虽然不同变化会导致部分寄存器不可用，但是实际上，对于给定的可向量化循环（Grid），大多数线程块使用相同数量的寄存器。此外，硬件需要知道每个线程块的寄存器在register file中的位置，以每个线程块为基准记录。

NVIDIA GPU指令集架构

与大多数系统处理器不同，NVIDIA编译器的指令集目标是硬件指令集的抽象。PTX（Parallel Thread Execution）为编译器提供了稳定的指令集，已经不同代GPU之间的兼容性。

硬件指令集对于程序员是不可见的。PTX指令描述单个CUDA线程的操作（通常与硬件指令一对一映射），但一个PTX指令可以扩展为多个机器指令，反之亦然。

PTX没有一次写入寄存器的数量限制，编译器必须进行寄存器分配，将PTX寄存器映射到实际设备中可用的固定数量的读写硬件寄存器。随后通过优化器，可以进一步减少寄存器的使用，还会做消除dead code ，合并指令，并计算分支可能发散和汇聚的位置等操作。

虽然x86微架构和PTX之间存在一些相似性，都会转换为内部形式，但区别在于：在x86上，这种转换发生在硬件执行过程的运行时（runtime）中；而在GPU上，是在软件和加载时发生的。

PTX指令的格式：opcode.type d, a, b, c;

其中，d是目标操作数；a、b和c是源操作数；操作类型type是以下之一:

源操作数是32位或64位寄存器或常数值。目标操作数是寄存器，除了存储指令外。

图4.17展示了基础的PTX指令集。所有指令有1位谓词寄存器，可以通过设置谓词指令（setp）进行设置。控制流指令包括函数调用（call）和返回（return）、线程退出（exit）、分支（branch）和线程块内的屏障同步（bar.sync）。在分支指令前面放一个谓词可以实现条件分支。

编译器或PTX程序员将虚拟寄存器声明为32位或64位有类型或无类型值。例如，R0、R1等用于32位值，RD0、RD1等用于64位寄存器。虚拟寄存器到物理寄存器的分配是在PTX加载时发生的。

与向量架构不同，GPU没有单独指令用于连续数据传输、带步长数据传输和聚集-散布数据传输。GPU中所有的数据据传输都是聚集-散布（gather-scatter）。

为了获得连续（单位步长）数据传输的效率，GPU内的特殊地址合并硬件会识别什么时候SIMD指令线程内SIMD通道共同产生连续地址；然后，运行时硬件通知内存接口单元请求32个连续word的块传输。通过相邻的CUDA线程同时访问相邻的地址，可以被合并到一个或几个内存或缓存块中。

GPU中的条件分支

在PTX汇编器层，一个CUDA线程的控制流由PTX指令分支、调用、返回和退出描述，加上每个指令的单独线程通道谓词（由1位谓词寄存器指定）。PTX汇编器分析PTX分支图并将其优化为最快的GPU硬件指令序列。每个线程可以根据自己的决策进行分支，不需要步调一致。

在GPU硬件指令层面，控制流包括分支、跳转、索引跳转、调用、索引调用、返回、退出和管理分支同步栈的特殊指令。GPU硬件为每个SIMD线程提供私有栈；栈的项包含一个标识符token、目标指令地址和目标线程活动的mask。GPU硬件指令也有每个通道单独的谓词（启用/禁用），由每个通道的1位谓词寄存器指定。

PTX汇编程序通常会优化PTX分支指令的简单外层IF-THEN-ELSE语句，为单独的谓词GPU指令，而不是任何GPU分支指令。更复杂的控制流通常混合使用谓词和带有特殊指令和标记的GPU分支指令（当一些通道分支到目标地址时，使用分支同步堆栈push堆栈项，其他通道则失败）。

PTX汇编器识别循环分支并生成分支到循环的顶部的GPU分支指令，以及特殊的栈指令来处理脱离循环的单独通道，并在所有通道完成循环时，汇聚SIMD通道。

GPU索引跳转和索引调用指令push项到栈中，所以当所有通道完成switch语句或函数调用时，SIMD线程汇聚。

GPU设置谓词指令（setp）用于评估IF语句的条件部分。PTX分支指令依赖于该谓词。如果PTX汇编器生成的谓词指令没有GPU分支指令，它会用每个通道的谓词寄存器来为每条指令启用或禁用SIMD通道。

IF语句的THEN部分，线程内的SIMD指令将操作广播到所有SIMD通道。这些谓词设置为1的通道会执行操作和存储结果，而其他SIMD通道不执行操作或存储结果。

对于ELSE语句，指令使用谓词的补集（相对于THEN语句），所以之前空闲的SIMD通道现在执行操作并存储结果。在ELSE语句结束时，指令取消谓词，初始计算可以继续进行。

在底层的细节层面，PTX汇编器会在适当的条件分支指令设置一个“分支同步”标记，将当前活动的mask push到每个SIMD线程内的栈上。如果条件分支发生分化（通道分支预测情况不一致），会push到栈中，并根据条件设置当前的内部活动的mask。分支同步标记在ELSE部分之前pop出分化的分支项并翻转mask比特位。在IF语句结束时，PTX汇编器添加另一个分支同步标记，将先前的活动的mask从栈中pop出到当前的活动的mask中。

如果所有的mask比特位都被设置为1，那么THEN部分末尾的分支指令将跳过ELSE部分的指令。对于所有掩码位都为0的情况，THEN部分也有类似的优化，因为条件分支会跳过THEN部分的指令。

并行的IF语句和PTX分支通常使用全体一致的分支条件（所有通道都遵循相同的路径），这样SIMD线程就不会分化到不同单独通道控制流中。PTX汇编器会优化这样的分支，跳过SIMD线程中任何通道都不执行的指令块。

else

NVIDIA GPU内存结构

如图4.18所示，展示了NVIDIA GPU的内存结构。

在多线程SIMD处理器中的每个SIMD通道都有私有内存。（1）用于堆栈帧、溢出寄存器和寄存器中不适合的私有变量。（2）SIMD通道之间不共享私有内存。（3）GPU会将这个私有内存缓存在L1和L2缓存中，以帮助寄存器溢出和加速函数调用。

多线程SIMD处理器专属的片上内存称为本地内存。（1）作为小的临时内存。（2）具有一个低延迟（几十个时钟周期）、高带宽（每个时钟周期128 byte）的特点。（3）用于同一线程或线程块中线程之间的数据存储或复用。线程块之间的不共享数据。（4）当多线程SIMD处理器创建线程块时，动态分配本地内存的一部分给线程块，并在线程块中的所有线程退出时释放内存。（5）动态分配的本地内存对线程块是私有的。

GPU和所有线程块共享的内存为共享内存。片外DRAM内存。

系统处理器，也称为主机（host）对于GPU内存的访问。（1）可以读取或写入GPU内存；（2）本地内存对主机不可用，是每个多线程SIMD处理器私有的；（3）私有内存对主机不可用。

图4.18 GPU的内存结构。GPU内存所有Grid共享，本地内存线程块内的所有SIMD指令线程共享，私有内存单个CUDA Thread私有。

GPU传统上使用较小的流式缓存，通过依赖大量的SIMD指令线程的多线程来隐藏访问DRAM的长延迟，而不是依赖大型缓存。基于使用多线程来隐藏DRAM延迟，系统处理器中大型L2和L3缓存的芯片面积被用于计算资源和大量寄存器，为了保存许多SIMD指令线程的状态。

虽然将内存延迟隐藏在许多线程中是GPU和向量处理器的最初思想，但是近期的GPU和向量处理器都用缓存来减少延迟。这个论点遵循了排队论中的Little定律：延迟越长，内存访问期间需要运行更多的线程，从而需要更多的寄存器。因此，GPU增加缓存来降低平均延迟，从而掩盖寄存器数量可能短缺的问题。

PS：利特尔定律（Little's Law）是一个在排队理论中使用的公式，用于描述排队系统中的平均项目数量、进入排队系统的速率和项目在队列中的平均时间之间的关系。

L：在排队系统中平均项目数量
λ：单位时间内到达排队系统的平均项目数量
W：项目在排队系统中花费的平均等待时间
Pascal GPU架构的创新点

Pascal架构的多线程SIMD处理器比图4.20中简化的版本更加复杂，图4.19展示了双调度器发射指令的情况，图4.20展示了Pascal GP100 GPU的多线程SIMD处理器的框图，下面是一些特点。

为了增加硬件利用率，每个SIMD处理器有2个SIMD线程调度器（有多个指令分配单元）。
SIMD线程调度器选择SIMD指令线程，并向每个线程的两组16个SIMD通道、16个加载/存储单元或8个特殊功能单元发射指令。
每个时钟周期可以调度两个SIMD指令线程，允许64个通道处于活动状态。
线程是独立的，在指令流中不需要检查数据依赖性。

图4.20 Pascal GPU的多线程SIMD处理器的框图。每个SIMD通道都有一个流水线浮点单元、一个流水线整数单元，一些用于将指令和操作数分配给这些单元的逻辑，以及一个用于保存结果的队列。双精度ALU（DP单元）执行64位浮点运算，特殊功能单元（SFU），用于计算平方根、倒数、正弦和余弦等函数。

每一代新GPU架构通常都会增加一些新功能，以提高性能或使程序员更容易使用。以下是Pascal 的四大创新点:

快速单精度、双精度和半精度浮点运算。
高带宽内存。HBM2。
高速片间互联。由于GPU的协处理器性质，当尝试使用多个GPU与一个CPU进行通信时，PCI总线可能成为通信瓶颈。Pascal GP100引入了每个方向最高20 GB/s数据传输的NVLink通信通道。每个GP100有4个NVLink通道，有160 GB/s的片间峰值带宽，用于多GPU系统内GPU之间的通信。NVLink为所有相互连接的GPU和CPU之间提供了内存一致性，还提供了缓存到缓存的通信，而不是内存到内存的通信。
支持统一虚拟内存和paging。

（PS：算力和高带宽内存等，在现在看来参数上没什么可比性。可能主要还是NVLink，确实超前）。
