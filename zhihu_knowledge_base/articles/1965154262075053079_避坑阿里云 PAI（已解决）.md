# 避坑阿里云 PAI（已解决）

**作者**: 白牛github tpoisonooo

**原文链接**: https://zhuanlan.zhihu.com/p/1965154262075053079

---

看，又一个被阿里云逼疯的甲方。

作为 2014 年首批 ecs 用户，和 2024 年所谓”大客户”，我应该是看到了帝国腐烂的过程。

10 年前阿里云会为自己的故障写道歉信，现在嘛..

1. 价格是乱算的

我们之前采购了灵骏 gpu 服务，6w/月。账户里有 20w 元。24 年 10 月因为欠费停机。

整个事奇葩在..gpu 服务是预扣费的。账号还有 2w 的时候，扣除 6w 欠费了。然后又因为账户欠费， gpu 自己停了！

服务没完整提供，我还倒欠他 4w，我心里..真是万马奔腾。

事情过去一年了，作为”大客户”已经熬走了俩销售。在甲方的不续约威胁下，他们每次都说好好好，仍没彻底解决。

2. 资源是瞎堆的

现在打开 pai 资源组，购买 8cpu+30G 内存。实际使用时，只有 7cpu+27G 内存。

这是要九出十三归？

东西不便宜，我们已经为云服务额外付费了（租单卡A10 5000/月，真的不便宜）。用的时候还九出十三归？

便不便宜自行搜淘宝

类似的资源错配还有。让 eas 访问外网，发几个 http 请求，必须要买公网 nat，最低 240/月。

对对对，实习生写代码，需要个架构师指导。你怎么不去抢呢？！

3. 玩套路是来搞笑的

不知哪位 P8 大聪明，在公共资源配置上，设计了竞价策略。出价高的实例可被调度。

可真按当前 3.7/小时出价 A10 (折合2700 / 月)，又提示”库存不足”。只能再把配置改回去。

菜，想玩，又玩不起。白白浪费我部署时间。

菜又爱玩，破发了又耍赖
4. 处理工单是不带脑子的

提工单像打太极。

认真描述问题，对面是不看的。不管你说啥，先回复个套话:

“亲，我们公网 nat 支持按量付费哦”

但凡动点脑子，看看所谓按量是多少钱么..




岂有此理..我们这付款资质，还得受这鸟气。明年换供应商，结束。

更新于 20251027

周六日收到了阿里云运营+技术的电话、销售那边也说本周完成调账。等一年了，再等一周也没啥。

最难绷是评论区部分技术人员。他们似乎在尝试给我科普稳定（guo shi）了应该有三十年的 tcp/NAT/VPC 来同情服务商。干脆买到烂尾楼也共情开发商好了。
