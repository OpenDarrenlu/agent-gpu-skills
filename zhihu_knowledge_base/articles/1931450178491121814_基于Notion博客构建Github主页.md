# 基于Notion博客构建Github主页

**作者**: SnoopyBug划水划呀划~

**原文链接**: https://zhuanlan.zhihu.com/p/1931450178491121814

---

1. 准备

创建Github主页，例如“http://username.github.io”。

2. 拷贝Notion博客模版

将Notion博客模版拷贝到自己的Notion空间，并发布该页面，获得页面链接。

3. Fork仓库：notablog-starter

将notablog-starter仓库fork到自己的Github仓库，将config.json中的url改为上面得到的页面链接。

4. 设置ACCESS_TOKEN

前往Github，在Settings中找到Developer settings，然后在Personal Access Tokens中新建Tokens。期限可选“no expiration”，scops选择“repo”即可。生成token，复制下来。如果一段时间后执行失败，记得检查token是否因为某些原因失效或者过期。

回到自己的notablog-starter仓库，在settings中找到secrets and variables，点击actions，在右边页面新建repository secret。name填写“ACCESS_TOKEN”，secret填写上面生成的token。

5. 创建Workflow

在自己的notablog-starter仓库中，找到Actions，新建Workflow后选择“set up a workflow yourself”，此时会生成main.yml。

将main.yml文件替换为以下内容，需要修改“repository-name”参数为自己的主页（例如username/http://username.github.io）。

name: Github Pages

on:
  schedule:
    - cron: '0 23 * * *'
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v2

      - name: Build
        run: |
          npm i -g notablog
          notablog generate .
          ls -R public  # 验证输出

      - name: Deploy
        uses: JamesIves/github-pages-deploy-action@v4.2.5
        with:
          token: ${{ secrets.ACCESS_TOKEN }}
          repository-name: username/username.github.io
          branch: gh-pages
          folder: public
          exclude: ".git, .github, .ssh"
6. 执行Workflow

在自己的notablog-starter仓库中，找到Actions，左侧出现“Github Pages”，点击后在页面右侧执行“Run Workflow”。

main.yml中也设置了每天标准时间23点（北京时间6点）执行，自动更新Notion内容到主页。

7. 切换主页分支

由于main.yml中设置将目标网页资源更新到username/http://username.github.io的gh-pages分支，需要前往http://username.github.io仓库，在settings中找到Pages，将“branch”设置为“gh-pages”，并保存。

8. 查看主页

Notion中的内容编辑很自由，但是需要按照模版来组织内容，逻辑比较清楚。

每次更新完Notion内容后，可以进行步骤6直接更新，也可以等待每日6点自动更新。

Tips

需要注意的是，目前notablog对Notion中的图片支持似乎有点问题，我还没弄清楚原因。因此推荐先将笔记中使用的图片使用知乎等平台上传到网络，然后使用“图片链接”的方式嵌入到Notion中。
