GDN Chunked Prefill 的一些数学变形

符号约定：

Q_iK_iV_i 是第 i 个 chunk 的输入，尺寸为 C \times D_K 和 C \times D_V ，默认 C=64 ， D_K=D_V=128
A_i 是第 i 个 chunk 在 WY-Representation 过程中矩阵求逆的结果，尺寸为 C \times C
\alpha_i\beta_i 分别为第 i 个 chunk 的 gate 和 writing strength，是长度为 C 的向量（即每个 token 上有一个标量），且取值范围为 (0, 1]
\gamma_i 是在第 i 个 chunk 内部对 \alpha_i 的累乘，是长度为 C 的向量
\gamma_{i,C-1} 是 \gamma_i 的最后一位，也是第 i 个chunk内所有 \alpha_i 的乘积 \Pi_{t=0}^{C-1}\alpha_i ，是标量

考虑原版 GDN Chunked Prefill 递推流程：

1. A_i \gets \left(I+\mathrm{StrictLower}\left( \mathrm{diag}(\beta_i)(\Gamma_i \odot K_iK_i^\intercal) \right)\right)^{-1}
2. \left\{\begin{aligned} W_i &\gets A_i\mathrm{diag}(\beta_i)\mathrm{diag}(\gamma_i)K_i \\ U_i &\gets A_i\mathrm{diag}(\beta_i)V_i \end{aligned}\right.
3. \left\{\begin{aligned} V_i' &\gets U_i-W_iS_i \\ S_{i+1} &\gets \gamma_{i,C-1}S_i + K_i^\intercal\mathrm{diag}\left(\frac{\gamma_{i,C-1}}{\gamma_i}\right)V_i' \end{aligned}\right.
4. O_i \gets \mathrm{diag}({\gamma})Q_iS_i + \left(\mathrm{Lower}(\Gamma_i) \odot Q_iK_i^\intercal\right)V_i'

对于第1步，由于 \left(\mathrm{diag}({\gamma})X\mathrm{diag}\left(\frac{1}{\gamma}\right)\right)^{{-1}} = \mathrm{diag}({\gamma})X^{-1}\mathrm{diag}\left(\frac{1}{\gamma}\right) ，有

\begin{aligned} &\left[I+\mathrm{StrictLower}\left( \mathrm{diag}(\beta_i)(\Gamma_i \odot K_iK_i^\intercal) \right)\right]^{-1}\\ =&\left[I+\Gamma_i \odot\mathrm{StrictLower}\left( \mathrm{diag}(\beta_i) K_iK_i^\intercal \right)\right]^{-1}\\ =&\left[\Gamma_i \odot I+\Gamma_i \odot\mathrm{StrictLower}(...)\right]^{-1}\\ =&\left[\Gamma_i \odot (I+\mathrm{StrictLower}(...))\right]^{-1}\\ =&\left[\mathrm{diag}(\gamma_i) (I+\mathrm{StrictLower}(...))\mathrm{diag}\left(\frac1{\gamma_i}\right) \right]^{-1}\\ =&\mathrm{diag}(\gamma_i)\left[I+\mathrm{StrictLower}(...)\right]^{-1}\mathrm{diag}\left(\frac1{\gamma_i}\right)\\ =&\Gamma_i \odot \left[I+\mathrm{StrictLower}(...)\right]^{-1} \end{aligned}

也就是说这个 \Gamma_i 可以拆出来。拆出来有什么好处呢？我们知道由于累乘的数值精度问题，实际保存的数据是\log{\gamma_i} 而不是 \gamma_i ，那么 \Gamma_{i,s,t} 就等于 \exp(\log \gamma_{i,s}) / \exp(\log \gamma_{i,t}) 。又因为这个除法有溢出风险，所以实际计算采用的是 \Gamma_{i,s,t} = \exp(\log \gamma_{i,s} - \log \gamma_{i,t}) ，就需要在 SFU 上做 C^2 次 \exp 操作，还是有一定开销的。我们把 \Gamma_i 从第1步拆出来放到后面计算 S_i 和 O_i 的 kernel 中，由于第4步本来也需要计算 \Gamma_i ，这样可以省去重复计算的开销。

另外第2、3步 V_i'=U_i-W_iS_i=A_i\mathrm{diag}(\beta_i)(V_i-\mathrm{diag}(\gamma_i)K_iS_i) ，这样时间复杂度要小一些，在 主流 Attention 在 GPU 上的优化实践 中亦有记载。

综上我们得到一个新的递推流程：

1. A_i \gets \left(I+\mathrm{StrictLower}\left( \mathrm{diag}(\beta_i)K_iK_i^\intercal\right)\right)^{-1}
2. \left\{\begin{aligned} V_i^\Delta &\gets V_i - \mathrm{diag}(\gamma_i)K_iS_i \\ V_i' &\gets \left(\Gamma_i \odot A_i\right)\mathrm{diag}(\beta_i)V_i^\Delta \\ S_{i+1} &\gets \gamma_{i,C-1}S_i + K_i^\intercal\mathrm{diag}\left(\frac{\gamma_{i,C-1}}{\gamma_i}\right)V_i' \\ O_i &\gets \mathrm{diag}({\gamma_i})Q_iS_i + \left(\mathrm{Lower}(\Gamma_i) \odot Q_iK_i^\intercal\right)V_i' \end{aligned}\right.

有眼尖的小朋友可能会发现，如果把 \Gamma_i \odot A_i 写成 \mathrm{diag}(\gamma_i)A_i\mathrm{diag}\left(\frac1{\gamma_i}\right) 的形式，新流程的第二步还可以化简为：

\left\{\begin{aligned} V_i^\Delta &\gets \mathrm{diag}\left(\frac1{\gamma_i}\right)V_i - K_iS_i \\ V_i' &\gets A_i\mathrm{diag}(\beta_i)V_i^\Delta \\ S_{i+1} &\gets \gamma_{i,C-1}(S_i + K_i^\intercal V_i') \\ O_i &\gets \mathrm{diag}({\gamma_i})Q_iS_i + \mathrm{Lower}\left(\mathrm{diag}(\gamma_i) Q_iK_i^\intercal\right)V_i' \end{aligned}\right.

这样似乎就不再需要计算 \Gamma_i 了。那这样是否可行呢？答案是不行的。考虑到 \gamma_i \approx 0 时 \frac1{\gamma_i} 可能溢出，这个 \frac1{\gamma_i} 是不能显式地计算出来的。

Gate 与 Sliding Window

其实这一切的起源都是我们在去年11月的某一天顿悟到有 gate 就有 sliding window 这件事，FlashQLA 也可以说是为了这盘醋包的饺子。博客中已经介绍了如何利用 sliding window 做序列并行，这里不再赘述。回答几个大家可能关心的问题：

有没有 sliding window 区别大吗？其实不大。因为只要这层有一个 head 不具备 sliding window 的性质，从 S_0 开始递推的那个 thread block 就会拖慢整个 kernel 的运行时间。我们也尝试了增加并行度使得非 sliding window 的 thread blocks 可以填满整个 GPU，但这样又会让后续的矫正 kernel 变慢。短期内的计划是在 tilelang_prepare_h_kernel 中支持切分 D_V 。
通过 gate 累乘判断 sliding window 大小的阈值怎么取？我们目前采用的是 \exp(-10.0) ，即 \sum_i\log\gamma_{i,C-1} < -10.0 ，但这个阈值其实并不太影响窗口大小，因为 gate 的累乘是指数衰减的，改成 \exp(-8.0) 也只会让窗口缩短 1/5。
Sliding window 只能用于 CP 预处理这一件事吗？不是。大家可以自行发挥想象。
KDA 有 sliding window 吗？直觉上 per-channel gate 更难形成 sliding window，但也说不准实际训练出来在某些 head 上是有的。有心人可以去验证一下。
序列并行中的代数等价

回顾 DeltaNet如何做序列并行 中的 CP 流程，对于 CP rank j ：

\left\{\begin{aligned} S^\ast_{j,i+1} &\gets \gamma_{j,i,C-1}S^\ast_{j,i} + K_{j,i}^\intercal \mathrm{diag}\left(\frac{\gamma_{j,i,C-1}}{\gamma_{j,i}}\right) V'_{j,i} \\ M_{j,i+1} &\gets \left( \gamma_{j,i,C-1} I -K_{j,i}^\intercal \mathrm{diag}\left(\frac{\gamma_{j,i,C-1}}{\gamma_{j,i}}\right) W_{j,i} \right) M_{j,i} \end{aligned}\right.
S_{j,0} \gets S^\ast_{j,0} + M_{j,0}S_{j-1,0}

乍一看这个 M 矩阵递推的代价是很高的，每个 chunk 要做 128 \times 128 \times 128 的矩阵乘法。但我们知道矩阵连乘在不同的结合顺序下具有不同的时间复杂度，这个 M 和 S 看起来也没有本质区别（？）应该是有更快的数学等价的。让我们操作一下：

\begin{aligned} M_{j,i+1} &= \left( \gamma_{j,i,C-1} I -K_{j,i}^\intercal \mathrm{diag}\left(\frac{\gamma_{j,i,C-1}}{\gamma_{j,i}}\right) W_{j,i} \right) M_{j,i} \\ &= \left( \gamma_{j,i,C-1} I -K_{j,i}^\intercal \mathrm{diag}\left(\frac{\gamma_{j,i,C-1}}{\gamma_{j,i}}\right) (\Gamma_{j,i} \odot A_{j,i})\mathrm{diag}(\beta_{j,i})\mathrm{diag}(\gamma_{j,i})K_{j,i} \right) M_{j,i} \\ &= \gamma_{j,i,C-1}\left( I - K_{j,i}^\intercal A_{j,i}\mathrm{diag}(\beta_{j,i})K_{j,i} \right) M_{j,i} \\ &= \gamma_{j,i,C-1}\left( M_{j,i} - \left(K_{j,i}^\intercal A_{j,i}\mathrm{diag}(\beta_{j,i})\right) \left(K_{j,i} M_{j,i}\right) \right) \end{aligned}

通过结合律，我们获得了和 S 矩阵相同的递推复杂度。这里还可以 call back 上面的 sliding window，我们把 \gamma_{j,i,C-1} 提到外面意味着 M 矩阵也在随 gate 衰减，那么只要 \gamma_{j,i,C-1} 累乘到足够小，序列并行的第二步 S_{j,0} \gets S^\ast_{j,0} + M_{j,0}S_{j-1,0} 就变成了 S_{j,0} \gets S^\ast_{j,0} ，这就是在 chunk 视角下 sliding window warmup 的原理。

有聪明的小朋友可能还会想，这个 M 矩阵的递推形式和 S 矩阵如此相似，会不会有数据复用的可能？有的，兄弟，包有的。我们在前面还留了一个伏笔，\frac1{\gamma_i} 可能溢出但 \frac{\gamma_{i,C-1}}{\gamma_i} 不会，因此只算 S 不算 O 时我们是可以进一步变换以跳过计算 \Gamma_i 的。具体来说就是：

S^\ast_{j,i+1} \gets \gamma_{j,i,C-1}S^\ast_{j,i} + K_{j,i}^\intercal A_{j,i}\mathrm{diag}(\beta_{j,i}) \left( \mathrm{diag}\left(\frac{\gamma_{j,i,C-1}}{\gamma_{j,i}}\right) V_{j,i} - \gamma_{j,i,C-1}K_{j,i} S^\ast_{j,i} \right)

因此我们可以在同一个 thread block 中通过复用 K^\intercal A \mathrm{diag}(\beta) 同时计算 S 和 M 。流程总结为：

\left\{\begin{aligned} X_{j,i} &\gets -\mathrm{diag}(\beta_{j,i}) A_{j,i}'^\intercal K_{j,i} \\ Y_{j,i} &\gets \gamma_{j,i,C-1} K_{j,i} S^\ast_{j,i} - \mathrm{diag}\left(\frac{\gamma_{j,i,C-1}}{\gamma_{j,i}}\right) V_{j,i} \\ Z_{j,i} &\gets K_{j,i} M_{j,i} \\ S^\ast_{j,i+1} &\gets \gamma_{j,i,C-1}S^\ast_{j,i} + X_{j,i}^\intercal Y_{j,i} \\ M_{j,i+1} &\gets \gamma_{j,i,C-1} \left( M_{j,i} + X_{j,i}^\intercal Z_{j,i} \right) \end{aligned}\right.

Backward 中的代数等价

Backward的优化点比较零碎，这里挑几个比较有意思的：

求 d\beta 需要用到 d(A\mathrm{diag}(\beta)) \odot A ，而求 d\Gamma 需要用到 d(\Gamma \odot A\mathrm{diag}(\beta)) \odot (A\mathrm{diag}(\beta)) ，但由于我们实际上要算的是 d\log \gamma ，所以 d\Gamma 后面还要再乘上 \Gamma ，我们变换一下 \begin{aligned} &d(\Gamma \odot A\mathrm{diag}(\beta)) \odot (A\mathrm{diag}(\beta)) \odot \Gamma\\ = &d(\Gamma \odot A\mathrm{diag}(\beta)) \odot \Gamma \odot A\mathrm{diag}(\beta)\\ = &d(A\mathrm{diag}(\beta)) \odot A\mathrm{diag}(\beta) \end{aligned} ，那么 d(A\mathrm{diag}(\beta)) \odot A 这一部分又可以复用了
从 d\Gamma 到 d\gamma 会经过 d\gamma = \sum_s d\Gamma - \sum_t d\Gamma 这样在两个维度上分别 reduce 再相减的形式，又因为 reduce 最好是在内层，所以这里怎么着都是要通过 shared memory 转置一次的。那么我们可以这样： d\gamma = \sum_s (d\Gamma - d\Gamma^\intercal) 减少一次 reduce 操作。
梯度从 A_T=KK^\intercal 回传到 K 这一步是不需要做两次矩阵乘法的，可以这样： dK = (dA_T+dA_T^\intercal)K
