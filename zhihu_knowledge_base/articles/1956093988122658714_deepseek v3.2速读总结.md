# deepseek v3.2速读总结

**作者**: 菽陌松囿

**原文链接**: https://zhuanlan.zhihu.com/p/1956093988122658714

---

刚扫了下paper：

模型结构：引入DSA解决seq^2计算复杂度，在deepseek v3.1 terminus上continue pretrain和post train，DSA原理就是从query token筛选出哪些token进行attention计算，其实就是再做一次attention score计算， 然后选择topK的token attention score，不过比mla的计算量小很多（64*128）。

推理方面：

推理成本肯定大大降低，未来可以支持更长的context，并且ttft/tpot/吞吐理论上都会好不少，值得注意的是，indexer attention score 还是seq^2复杂度，但是维度降低了（相比于mla），移除了softmax，几乎蜕变为gemm

算子引入tilelang去实现dsa，未来triton/tilelang/cutedsl可能是潮流， ai编译器重要性也更重要了，一定上缓解某些非cuda兼容平台的压力。

然后再看看训练细节：

warmup stage：

将3.1模型增加indexer模块，没有topk selection，将3.1参数freeze ，对齐indexer attention 和dense attention score输出，比较token attention score KL散度，1000个step，2.1B tokens

Sparse training stage：

对齐完之后，添加了fine grained topk token selection， 再次进行 attention score对齐

后训练：

先基于3.2 base model训练领域专家，然后蒸馏不同领域的专家cot数据，用后RL进行知识合并

然后用不同领域cot数据进行大规模混合RL训练，还是用grpo，不过有具体reward model对cot长度&语言一致性和准确性进行一个打分，来给予奖励信号

意义：

在MLA引入稀疏注意力，降本巨大（MLA降低kvcache显存，DSA降低计算量），并且精度持平，价值挺大的了，最近阿里提出混合注意力，其实意图一样，以前swa啥也好是瞎蒙，注定效果没保障。

提出了基于现有模型去改造做续训练和后训练，不用重头开训，大大减少训练成本，很有意思，我记得有些人在moe模型基础提了一些upcycling的做法，一搬小厂拿到开源模型是不能动模型结构的，重训成本太高，最多就是剪枝。

看到了deepseek对国产芯片大力支持，day0支持，首先要把vllm/sglang pr+模型权重提前给寒武纪和ascend，先跑起来， 其实跑起来不是啥问题，也不要高潮，NV几分钟的事，国产慢点而已，尤其涉及新算子且用native kernel会慢很多，dsl是通过编译器屏蔽软硬件细节，可以跨平台。

补充：其实indexer的attention score计算复杂度其实也不低啊，不考虑移除softmax的收益，纯看qk计算量，还是O（seq^2），不过head num从128变为64了，head dim从576变为128，总计算量减少了9倍
