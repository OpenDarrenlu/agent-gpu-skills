# 关于deepseek R1盈利的计算

**作者**: 菽陌松囿

**原文链接**: https://zhuanlan.zhihu.com/p/27339870003

---

官方公布盈利:
收入为: 单日400w元
cost profit 盈利比例: 545%
实际盈利: 400w / (6.45) * 5.45 = 337w




盈利理论计算

其实细心的读者是可以读出R1的盈利的, 本文讲述一种计算方案, 有好事者可以自己算

成本: H800 单卡成本, 大概2美金/hour, 为15元/hour=0.004元/s
R1 token单价: 16元/百万token
单卡收入: 0.02元/s, 可看下文分析
盈利为: (0.02 - 0.0004) / 0.0004 = 400%
单日总盈利: 因为采用了约2200张卡, 2200 * (0.02-0.0004)元/s * 3600 * 24 = 300w, 相差gap为37W

然后再看理论盈利怎么推算的

为了分析deepseek R1推理收入, 因为R1非常火爆, 我们认为R1服务是满载的(白天/黑夜都是如此, 而且晚上针对流量低峰进行了缩容), 所以我们要分析其单卡能提供的吞吐, 关注到V3论文有个关键数据, 单个expert batch size 是256 token(其实按照这个数推算kvcache显存, 其实是合理, 具体怎么算, 自己摸索), 因为一个token 被dispatch 8张卡, 所以在集群推理模式下, 单张卡能推理256/8 = 32路batch请求

那单卡满载的推理吞吐是多少呢?

按照官网api说明是40 token/s, 所以单卡吞吐32 * 40 token/s=1280 token/s, 其实如果满负荷跑latency变大吞吐更高 按照单价16 元/百万token, 单卡满载的收入: 1280 token/s * 16 / (1000000 元/ token) = 0.02元/s

盈利比例是多少呢(0.02元/s-0.004元/s) / 0.0004 = 400%, 因为是2200张卡, 所以单日盈利(0.02 - 0.004) * 2200 * 24 * 3600 = 300w, 因此和官方公布数据gap仅为37W

以上数据纯个人推断, 感兴趣可以参考自己验证讨论.
