item 2: 尽量别用宏
2.1 常量声明使用const
#define PI 3.14很蠢，因为报错的时候就报一个3.14，也不会告诉你是PI这里出了问题。

2.2 类中数组长度也别define

// using static const int 
class GamePlayer { 
private:
    static const int NumTurns = 5; // constant declaration rather than definition 
    int scores[NumTurns];
};
// const int GamePlayer::NumTurns; // definition. put it into .cpp file, not .h file, if address of NumTurns is taken

// using enum hack
class GamePlayer {
private:
    enum { NumTurns = 5 }; // NumTurns a symbolic name for 5
    int scores[NumTurns]; // fine
};


这里蕴含着几个知识点：

C++要求你对任何东西都进行定义，而class-specific constants that are static and of integral type(int, char, bool)是一个例外，如果你不想取他们的地址的话。想取地址就把注释的那行删了。
对enum取地址是非法的，这使得enum表现的更像#define

item 3: 善用 const

对于变量、函数返回值、函数参数，能用const就用const

3.1 指向常量的指针 or 指针常量

char greeting[] = "Hello";
const char *p1 = greeting; // non-const pointer; const data. valid for p=new_addr
char const *p11 = greeting; // equal to p1,
char * const p2 = greeting; // const pointer, non-const data. valid for p[0]+=1
const char * const p3 = greeting;; // const pointer and data


3.2 const and non-const member funtion

如果你想实现这两个函数且避免代码重复，non-const可以调用const，因为const提供的是安全版，从安全扩展为不安全是合法的。
char& operator[](std::size_t position) {
  // now just calls const op[]
  return const_cast<char&>(
    static_cast<const TextBlock&>(*this)[position] 
  );
}

3.3 bit-wise const的局限性
WHAT is bit-wise const: 如果一个对象是const的，那么他的成员的值就是无法修改的。但是下面这段阴间代码却是合法的
class C {
public:
  ...
  char * pc;
}
const C c("Hello");
C.pc[0] += 1; // it's legal since it doesn't modify value of pc

如果我觉得就算对象是const的，某一个成员也是可以修改的，该怎么做？-》添加mutable关键字
class C {
public:
  ...
  char * pc;
  mutable std::size_t pc_length;
  std::size_t length() const { // function for const 
    pc_length = std::strlen(pc);
    return pc_length;
  }
}
const C c("Hello");
auto length = C.length();

item 4: 总是初始化变量

4.1: use intialization list instead of assignment

初始化列表中变量的顺序最好和变量被声明的顺序一致
WHY：观察下面这个在构造函数中使用了assignment的例子，B的构造其实执行了三遍。
class B {
public : 
    B():n(0) {
        std::cout << "construct B:" << n << std::endl;
    }
    B(int n_):n(n_) {
        std::cout << "construct B:" << n << std::endl;
    }
    B operator=(const B& rhs){
        std::cout << "copy B:" << rhs.n << std::endl;
        return B(rhs.n);
    }
    int n;
};

class A {
public:
    A(B b_){ // equal to A(B b_):b() {,
        b = b_;
    }
    B b;
};
B b(1);
A a(b);
// we get: 
// construct B:1
// construct B:0
// copy B:1
// construct B:1


4.2: data members that are const or are references must be initialized

4.3： the relative order of initialization of non-local static objects defined in different translation units is undefined.

WHAT is non-local static: 在函数中被定义的static是local static，其他的全局变量、类中static变量就是non-local static
// file_a.cpp
extern A a;
// file_b.cpp
static B b(a); // dangerous! a may not be initialized

解决办法：因为C++编译器保证在get_a调用时，其中的local static已经被初始化了。
// file_a.cpp
A& get_a() {
  static A a;
  return a;
}
// file_b.cpp
static B b(get_a());

item 5 & 6: 关于编译器会默认生成的函数

5.1 哪些函数会被默默声明/调用

默认构造函数 & 析构函数: 简单的初始化 non -static 成员
拷贝构造函数(copy constructor) & 赋值运算符(copy assignment): 简单的复制 non-static 成员

5.2 什么情况下编译器不会生成这些函数

如果你定义了构造函数，编译器就不会写个默认构造函数了
如果有成员是引用，编译器不会构造赋值运算符。同时，因为引用类型必须在定义时就初始化，所以当创建类对象时，必须在构造函数的初始化列表中对引用类型的成员进行直接初始化，而不能在构造函数中初始化。
class Person{
public:
    string & name;
    Person(string& str): name(str){ } // correct. error {name = str;}
};


6.1 显示地禁止那些不需要的默认函数

如果想禁止复制，可以声明构造函数和赋值运算符为私有方法，且不去定义它。C++ iostream 库中ios_base, basic_ios等就是这么做的。(这种连参数名字都不写的方法竟然可以编译过，也是醉了 orz)
class HomeForSale {
private:
    HomeForSale(const HomeForSale&); // declarations only
    HomeForSale& operator=(const HomeForSale&);

    Uncopyable& operator=(const Uncopyable&);

class HomeForSale: private Uncopyable {
    ...
