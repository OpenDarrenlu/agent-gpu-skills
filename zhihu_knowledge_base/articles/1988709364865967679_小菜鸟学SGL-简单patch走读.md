kernel比较简单：

input是experts序的c2
output是token序的，每个原始token经过expert计算之后的权重加权和
permutation是dst2src索引

每个block处理原始token中的某一个token，block中至少256线程处理每个原始token对应的topk激活值（expert计算结果）；input[m * topk]的行向量 element-wise mul permutation[m * topk](标量，专家权重)

注：循环中i表示原始token id，j表示topk的专家id

然鹅，这个patch有两处bug：

if (src_row >= m) continue; src_row是m*topk，这个判断引起一些case的truncate
blockDim不够，当row_stride大于256，[256,row_stride) column没有累加到output中

+

+
+  for (int j = 0; j < topk; ++j) {
+    int index_2d = i * topk + j;
+    int src_row = permutation[index_2d];

+    scalar_t val = input_tensor[src_row * row_stride + d];

+    scalar_t factor = 1.0;

+      factor = factors[index_2d];
+    }
+

+  }
+
+  output_tensor[i * row_stride + d] = sum_val;

修复patch

patch：c797322280b48b9cf88bd18992f3f628f85d86b5

对应的pr：fix: fix apply_shuffle_mul_sum by mickqian · Pull Request #7444 · sgl-project/sglang

清晰标注了每个tensor的shape
input_tensor的load和output_tensor的store使用了向量指令，向量长度16字节（我理解)
假设bf16情况，vec_size是8，每个向量操作处理8个element
stride = blockDim.x表示block中的线程个数，等于min（向量load次数，1024）
跟之前patch类似，i表示原始token id，j表示topk的专家id
src_row是permutation转换过的，转换成（当前的原始token id，expert id）二元组对应的c2输入，permutation没问题的话，一定是合法的
最后还处理partial的情况，[row_stride / vec_size*vec_size，row_stride)的部分
