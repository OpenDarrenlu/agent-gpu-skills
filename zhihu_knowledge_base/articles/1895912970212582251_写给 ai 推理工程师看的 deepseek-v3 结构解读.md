动机

Deepseek-V3 以及其强化学习版 Deepseek-R1 火了好长一段时间了，关于其模型各种角度解读也很多，但是在查找资料的过程中，笔者发现对于其模型结构的描述都不是很直观，比如其完整模型有多少层，每一层的 hidden size 是多少，MoE 具体是从多少中选多少 ，以及相互之间怎么连接的。

Forward 流程总结

参考论文中的 hyper-parameter，我们可以总结其单层的 transformer layer forward 如 Fig. 1 ，整个模型由 61 层 mla + moe transformer layer 组成，其中前三层 FFN 为传统 FFN，其余层采用 MoE。MoE 模块，由 1 shared expert + 256 routed expert 组成，每次计算从 256 个 expert 中选择 8 个。

deepseek transformer layer 结构

Deepseek 主要改了 transformer 结构，如 Fig. 2 所示，在 attention 部分，其用 Multi-head Latent Attention 替代原有的 multi-head attention 来减少 kv cache 的存储开销，在 ffn 部分使用了 MoE （mixture of experts）方案来实现更少计算量达到大尺寸模型同等效果，并且使得分布式推理更自然。

Figure 2. deepseek transformer layer 结构图[1]，用 MLA 替代 multi-head attention，用 MoE 替代 feedforward
MLA

原始的 atten 结构的 infer 过程中，decode 阶段通常是 LLM 消耗显存的主要部分，而其消耗的大部分显存主要是为了存储每次 decode 产生的新的 kv cache，这个 cache 的大小随着 decode 的文本长度变长而线性增加。

考虑 Llama-2-7B bf16 版本，其 hidden size 为 4096，考虑 qga（4 q 对 1 kv），每个 token 需要增加的显存大小为：2 (kv) * 1024 * 32 layers * 2Bytes = 131 KB，而对于一个线上服务来说，通常都是多 batch 多 token 的场景，kv cache 会消耗大量显存。而更大的模型 decode 所需要的显存也会更高。

如何去节省 kv cache 的空间，deepseek 提出了 MLA，其本质是不直接存储 kv，而是对要存储的 kv 做一次“压缩”，然后在实际要用 kv 的时候再“解压”出来，如 Fig.2 所示。假设 batch size 为 1，代入 v3 的具体数值，每次 decode 具体做法如下：

具体计算过程，如 Fig.3 所示。

h (1, 1, 7168) -> kv 时，实际上先做一个 7168 -> 512 的 down proj，压缩后的 512 维 tensor c^{KV} 被称为 compressed latent vector，然后保存这个 c^{KV}。
在实际做 attention 的时候，
要得到 kv ，需要先 up-proj 到 128 * 128 channel，作为位置无关的 kv 信息。
我们希望 kv cache 的时候还能带上 rope 信息，所以保留 64 channel 由输入 h 经过另一个 dow-proj 得到，标记为 k^{R} ，然后对 k^{R} 做旋转位置编码，在和原来的 128 * 128 channel 拼接在一起即可得到 128 * 192 channel。注意这个拼接每个 head 拼接同一个 k^{R}，所以 k^{R} 的 channel 可以是非常小的，比如这里的 64。
v 就不用考虑 rope，所以就直接用一个 up-proj 即可。

MOE
如 Fig 4. 所示，MoE 对原本的一个 dense ffn 拆成了 N 份，每一份把 intermedia size 给减少到原来的 1 / N，保持实际运算的计算量不变，每次都是先用一层 router + topk 来决定会走哪些 ffn 层（每个 ffn 被称为一个 expert），最后结果按router score 相加起来给到下游，ffn 输出的尺寸也不会变。
对应到 671B 模型，会定义 1 个 shared expert，256 个 routed expert，每次计算都会经过所有的 shared expert，以及 top 8 个 routed expert。
这个方案还可以做成分布式推理架构，每个 expert 可以放在不同卡上推理，为了优化通讯开销，工程上还在训练时做了一个限制：每次要找的 routed expert 会尽可能分发到同一个 node 上（简单理解：一个 node 为 一个 8 卡机）
