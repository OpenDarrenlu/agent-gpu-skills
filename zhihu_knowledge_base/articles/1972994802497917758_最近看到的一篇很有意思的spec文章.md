Title: Steering Pretrained Drafters during Speculative Decoding

Link: https://arxiv.org/pdf/2511.09844

虽然最近忙着rebuttal，但 inbox 推送的文章还是会扫一眼。这篇paper可以说今年以来，为数不多的让我觉得有意思的paper了。

TL, DR; 一种全新的利用 target model 的 hidden states 的方式。

先看摘要，其实第一眼看上去会觉得有些云里雾里。"steering" 这个词个人觉得比较少见，但这个computed from the verifier's hidden states and injected into the pretrained drafter 着实让人难以理解。于是直接拉到论文的图示：

说实话，更云里雾里了。这篇paper的presentation真的没被审稿人challenge吗？verifier 到 drafter 中间的线是什么意思？生成 draft tokens吗？

读到这里，我才豁然开朗——这也是一种利用 target model 的 hidden states的方法！

那它到底是怎么做的呢？我们可以跟传统方法来比较一下（这个时候就能看懂上面那张图了）：

 vanilla speculative decoding: draft model 只接收 target model 的 accept / reject 信息；
eagle3: draft model 接收 target model 的 hidden states, 并作为inputs; 注意这里的draft model 通常是一个单层layer;
SD2: draft model 接收 target model 的 hidden states, 并加到 draft model 本身的 activation上。注意这里的draft model 通常是一个完整的LLM；

到这里，我会觉得这是一步非常精彩的变形！

hidden states 不再作为 一个独立的 inputs, 毕竟concat的操作虽然intuitive，但不够elegant；
hidden states 还是可以指导 draft model. 通过注入 activation, 居然能够提高 draft model的效果；
draft model 保持一个完整的 LLM 形态！不再需要魔改各种架构
只需要训练一个从 target model hidden states -> draft model activation 的连接器！这也是一个很轻量的训练。

Additional Points:

文章我还没有细看，只是粗略读了一下觉得很有趣，如有错漏欢迎补充；
在现有 draft model 上继续增强，是我觉得很promising的一个方向。对于目前的工业界来说，draft model的能力远远不够支持生产级别的提速；
这篇 paper的evaluation 仍然不够。仍然是 vanilla speculative decoding的evaluation，并没有真正的测出draft model的提升，因为vanilla speculative decoding 仍然受限于固定的gamma; 举例，假设 原来的 drafter能预测的 draft tokens是 8，2; 而 gamma被固定在5. 如果经过steering vector注入之后，预测的draft tokens变为了5，5，那就会造成MAT提高的表象；
这篇paper的idea我觉得还是很好的，但是presentation真的有点差...
我更好奇，在PEARL这种能真正unlock drafter能力的框架下，steering vector的提升会不会更大
