对应到 torch 源码里

# 已知输入
log_probs = log_softmax(gen_logits, dim=-1)                # (B, Tgen, V)

# 计算过程
# 计算每条序列的 log π_θ(y|x)
token_logp = torch.gather(
        log_probs, dim=-1, index=targets.unsqueeze(-1)     # (B, Tgen, 1)
).squeeze(-1)                                              # (B, Tgen)
seq_logp = token_logp.sum(dim=1)                           # (B,)

# 序列级目标  J_seq = E[ R * log π_θ(y|x) ]
loss = -(reward * seq_logp).mean()                         # 标量

计算过程是“一次性”的。计算中要累加拿到 seq_logp，就得先算 token_logp，显存峰值和 Tgen 相关。分块优化的压力转给 triton 或 GPU kernel 开发。




如果应用前面的结论，按 arXiv miniRL 所述，GRPO 训练中每次 rollout 一条 sequence，loss 是 token level 计算的。

即最终的： \nabla_\theta{(\frac{\pi_\theta(y|x)}{\pi_{\theta_{old}}(y|x)} )} = \nabla_\theta(\prod_{i=1}^{|y|} (1+\delta_{i})) \approx \nabla_\theta{( \sum_{t=1}^{|y|} \frac{\pi_\theta(y_t|x,y_{<t})}{\pi_{\theta_{old}}(y_t|x,y_{<t})} )}

物理含义：朴素地给整个 seq 算梯度，等价于给每个 token 算梯度再累加。

原本的并行代码可以调整成串行：

# 已知输入
logits    # (B, Tgen, V)

# 计算过程
for t in range(Tgen):
    lt = log_softmax(logits[:,t,:])          # (B,V)      ①
    pt = gather(lt, ids[:,t]).squeeze(-1)    # (B,)       ②
    loss_t = - (is_ratio[:,t] * reward * pt)              ③
    loss += loss_t / Tgen

每个 token 的 loss 是比较独立的，拿到 loss_t 就可以释放之前的显存。峰值和 Tgen 无关，削减一个维度。
