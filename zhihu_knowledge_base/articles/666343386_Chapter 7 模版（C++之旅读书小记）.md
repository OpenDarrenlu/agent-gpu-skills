1. 模版可以提升代码的抽象层次
提升抽象层次可以将代码从具体的类型和操作中抽离出来，使得代码更灵活，可以重复性的使用
2. 模型的类型是安全的，但是无约束的模版，编译期可能无法进行类型检查
2.1 C++20 引入了 Concepts
template<Element t>
class Vector {
private:
    T * elem;
    int sz;
    // ...
}

template<Element t> ，Element 是一个谓词，用于检查T 是否符合 Vector所需要的特性，在模版中指定一个概念，这叫做 受限模版参数。
2.2 Type Traits
使用 Type Traits 来在编译时检查类型属性，比如 std::is_integral 并通过静态断言（static_assert）提供有意义的错误消息。
template <class T>
T f(T i) {
    static_assert(std::is_integral<T>::value, "Integral required.");
    return i;
}
2.3 SFINAE（Substitution Failure Is Not An Error）
SFINAE的工作机制基于模板参数的替换。当模板实例化时，编译器尝试用实际的类型或值替换模板参数。如果这个替换在某个上下文中没有意义（例如，尝试执行一个不合法的操作），这种替换失败不会产生编译错误。相反，这个特定的模板实例会被丢弃，编译器会继续查找其他可能的模板实例。(From GPT)
一个常见的使用方式是是在返回类型上使用表达式SFINAE，其中表达式使用逗号运算符，逗号运算符的左子表达式是正在检查的那个（转换为void以确保不选择返回类型上的用户定义逗号运算符），右子表达式具有函数应该返回的类型。
template<class C, class F>
auto test(C c, F f) -> decltype((void)(c.*f)(), void()) {
    std::cout << "(1) Class/class reference overload called\n";
}

template<class C, class F>
auto test(C c, F f) -> decltype((void)((c->*f)()), void()) {
    std::cout << "(2) Pointer overload called\n";
}

void test(...) {
    std::cout << "(3) Catch-all overload called\n";
}

int main() {
    struct X { void f() {} };
    X x;
    X& rx = x;
    test(x, &X::f);  // (1)
    test(rx, &X::f); // (1), creates a copy of x
    test(&x, &X::f); // (2)
    test(42, 1337);  // (3)
}

output:
(1) Class/class reference overload called
(1) Class/class reference overload called
(2) Pointer overload called
(3) Catch-all overload called
3. 函数对象可以作为函数的参数
函数对象（仿函数），可以用来定义对象，该对象可以像函数一样被调用
函数对象的美观之处在于自带了用于比较的值，不需要为每个类型写一个单独的比较函数，也不需要用全局变量来保存函数
template<typename T>
class Less_than {
    const T val; // 待比较的值
public:
    Less_than(const T& v): val(v) { }
    bool operator()(const T& x) const { return x < val;}
};


Less_than lti {42};
Less_than lts {"Backus"s};

void fct(int n, const string& s) {
    bool b1 = lti(n);
    bool b2 = lts(s);
}

template<typename C, typename P>
int count(const C& c, P pred) {
    int cnt = 0;
    for(const auto& x: c) {
        if(pred(x)) {
            ++cnt;
        }
    }
    return cnt;
}

void f(const Vector<int>& vec, const list<string>& lst, int x, const string& s) {
    cout << "numbers of values less than " 
        << x << ": " << count(vec, Less_than{x}) << '\n';

    cout << "numbers of values less than "
        << s << ": " << count(lst, Less_than{s}) << '\n'; 
}
4. 匿名函数作为函数参数
void f(const Vector<int>& vec, const list<string>& lst, int x, const string& s) {
    cout << "numbers of values less than " << x 
    << ": " << count(vec, [&](int a){ return a < x}) << '\n';

    cout << "numbers of values less than " << s 
    << ": " << count(lst, [&](const string& a){return a < s}) << '\n'; 
}

6. 没有析构函数但需要清理的类型，如何提供 RAII
可以定义一个 finally 函数，在作用域结束时实行
void old_style(int n){
    void* p = malloc(n * sizeof(int));
    auto act = finally([&]{free(p);});
}

template<class F>
[[nodiscord]] auto finally(F f) {
    return Final_action{f};
}

template<class F>
struct Final_action {
    explicit Final_action(F f) :act(f) { }
    ~Final_action() { act(); }
    F act;
};
7. [[nodiscord]] 的作用
[[nodiscard]] 是 C++17 引入的一个属性，用于标记函数或方法的返回值不应被忽略。当在函数声明中使用 [[nodiscard]] 属性时，如果调用者没有使用该函数的返回值，编译器将发出警告。
错误状态检查：对于可能报告错误或状态的函数，使用 [[nodiscard]] 可以确保调用者检查并处理这些返回值，从而避免忽略重要的错误信息。
关键操作结果：如果函数执行了一些关键操作，其结果通过返回值反馈，使用 [[nodiscard]] 可以促使调用者关注这些结果。
[[nodiscard]] int computeSomethingImportant();

int main() {
    computeSomethingImportant(); // 这里将产生编译器警告，因为返回值被忽略了
    int result = computeSomethingImportant(); // 没有警告，因为返回值被使用了
}
8. 模版别名的用处
所有的标准容器库都提供了 value_type 作为元素类型的别名
using size_t = unsigned int;

template<typename T>
class Vector {
public:
    using value_type = T;
    //...
}

template<typename C>
using Value_type = C::value_type; // C 的元素类型

template<typename Container>
void algo(Container& c) {
    Vector<Value_type<Container>> vec; // 结果保存在此
    // ...
}
别名机制可以通过绑定模版的部分参数或者全部参数来定义新的模版

9. If constexpr 条件编译提供提供实现
某种操作可以用两个不同的函数
除了使用派生类，还可以使用编译时 if，避免多态开销，提升性能
is_trivially_copyable_v<T> 表明类型是否可以较小的代价被拷贝
template<typename T>
void update(T& target) {
    if constexpr(is_trivially_copyable_v<T>)
        simple_and_fast(target); // POD 类型
    else 
        slow_and_safe(target); // 复杂类型
}
