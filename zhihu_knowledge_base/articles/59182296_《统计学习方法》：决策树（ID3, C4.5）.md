决策树是一种比较直观的分类和回归方法，显然决策树具有树形结构，其根据输入数据的各个维度进行判断，每个判断即产生一个内部节点，而判断如果能给出结论，则产生一个叶子节点 ，即完成分类。

直观的看，决策树和心理测试题很像，都是如果xxx，则跳转到xxx的模式。可以视作一系列if-else规则的组合。每次对一个特征进行判断，故最多判断

决策树每一次判断选取的特征维度都需要保证其具有一定的分类能力，即其对最终的分类是有贡献的。如果某个维度的信息对分类结果没有影响，则这一信息是可以丢弃的。而选取特征时，需要尽量选取分类能力最大的那一维度，一般来说使用信息熵的增益来进行选择。

实现

实现中为了简洁清晰起见，我依然采用了pandas.crosstab()函数对频数进行统计，大大降低代码量。以书中P59，表5-1为例，crosstab能进行如下的统计：

故计算熵和条件熵可以在crosstab的统计结果的基础上轻松完成：

def calH(self, count):
    # cal entropy
    p = count/count.sum()
    p = p[p != 0]
    return -p @ np.log2(p)

def calg(self, cb):
    # cal gained entropy
    w = cb.sum(1)/cb.sum().sum()
    Hi = cb.apply(self.calH, axis=1)
    return w @ Hi

根据计算的熵和条件熵即可计算出信息增益，选取增益最大的维度进行分组，这里用到了df.groupby()，并将这一维度的数据去除。实现中可以用df.drop()去掉切分的数据。最后将切分维度的名字，各分组对应的切分维度的数据，以及分组打包返回。

def SplitData(self, df):
    labels = df.iloc[:, -1]
    data = df.iloc[:, :-1]
    # use crosstab to count the frequency
    cbs = (pd.crosstab(data.iloc[:, i], labels)
            for i in range(data.columns.size))
    y_c = labels.groupby(labels).count()
    # entropy of y
    HD = self.calH(y_c)
    HDA = [self.calg(cb) for cb in cbs]
    if self.method == "ID3":
        g = HD-HDA
    elif self.method == "C4.5":
        g = 1-HDA/HD
    if g.max() < self.eps:
        return None
    # the split location
    split = g.argmax()
    name = df.columns[split]
    # divide into parts
    gp = df.groupby(df.iloc[:, split])
    return ((name, i, d.drop(name, axis=1)) for i, d in gp)

有了分组信息，即可递归地构建决策树。递归的停止条件即为达到了叶子节点，有以下两种情况：

如果经过剔除后只剩下类别信息，则选取类别的众数作为其类别。
如果分组中类别信息均相同，即只有一类，则该类为其类别。

其他时候均为中间节点，则递归的进行分类。由于决策树各个特征维度的取值集合不同，可以使用字典这一数据结构来保存生成的决策树。具体的实现如下：

def creatTree(self, df):
    # all of one class
    if df.iloc[:, -1].unique().size == 1:
        return df.iloc[0, -1]
    if df.columns.size == 1:
        return df.mode().iloc[0, 0]
    res = {}
    # creat the tree recursively
    for name, i, d in self.SplitData(df):
        if name not in res:
            res[name] = {}
        res[name][i] = self.creatTree(d)
    return res
