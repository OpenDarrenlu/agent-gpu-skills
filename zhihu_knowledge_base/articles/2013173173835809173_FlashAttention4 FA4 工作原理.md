FlashAttention-4：非对称硬件扩展的算法与内核流水线协同设计

像Blackwell GPU这样的现代加速器延续了非对称硬件扩展（asymmetric hardware scaling）的趋势，其中张量核心（tensor core）吞吐量的增长速度远快于其他资源，例如共享内存带宽、用于指数等超越运算的特殊功能单元（SFU），以及通用整数和浮点ALU。例如，从Hopper H100到Blackwell B200，BF16张量核心的吞吐量从 1 PFLOPs 增加到了 2.25 PFLOPs，而SFU数量和共享内存带宽则保持不变。

这种扩展不对称性对于为Blackwell架构优化注意力（attention）等复杂内核有着深远的影响。其核心在于，注意力包含了两个 GEMM（

对注意力的一种天真看法可能是，GEMM 的速度完全控制了内核性能，人们可以有效地忽略这些其他注意力组件，至少在一阶近似上是这样。然而，对B200进行“进给与速度（feeds and speeds）”分析实际上显示了相反的情况：主要的性能瓶颈不在于张量核心执行 MMA（矩阵乘加）的速度有多快，而在于（a）在前向（FWD）计算期间负责计算 softmax 指数的SFU单元，以及（b）在后向（BWD）计算期间的共享内存流量。

在这篇博客文章中，我们提出了 FlashAttention-4，这是一种最大化矩阵乘法与这些其他资源瓶颈之间重叠的算法和内核协同设计。在B200上使用BF16时，它最高可达 1605 TFLOPs/s（71%的利用率），比 cuDNN 9.13 版本快达 1.3 倍，比 Triton 快达 2.7 倍。

我们主要的算法和内核协同设计理念如下：

用于最大化重叠的新流水线：新的前向和后向软件流水线，利用 Blackwell 完全异步的 MMA 和更大的分块（tile）大小，将张量核心、softmax 指数和内存操作重叠起来。
前向（FWD）传递：在 FMA 单元上通过多项式近似实现指数函数的软件模拟，以缓解指数计算瓶颈，并加上条件在线 softmax 重缩放。
后向（BWD）传递：将中间结果存储在张量内存（tensor memory）中以缓解共享内存流量，结合 Blackwell 新的 2-CTA MMA 模式以进一步减少共享内存流量并将原子归约（atomic reduction）减半，并额外支持确定性执行模式以实现可重现的训练。
调度：新的分块调度器（tile scheduler），以缓解由因果掩码和可变序列长度带来的负载不平衡。

Blackwell 上的新硬件特性
张量内存 (TMEM)：在B200上，148个 SM 中的每一个都拥有 256 KB 的 TMEM，这是一种直接连接到张量核心的片上暂存器，用于 warp 同步的中间存储。
完全异步的第五代张量核心：tcgen05.mma 是异步的并在 TMEM 中进行累加。对于 BF16 和 FP16，最大的单 CTA UMMA 分块为 128x256x16，这比 Hopper 最大的 WGMMA 原子大约大 2 倍。UMMA 由单线程启动，减轻了寄存器压力，使得更大的分块和更深的流水线变得切实可行，且没有 Hopper warpgroup MMA 的溢出痛点。这也使得 warp 特化更加可行，一些 warp 移动分块，而另一些则发出 MMA 指令，以将矩阵乘加与 softmax 和内存流量重叠。tcgen05.mma 还可以从 TMEM 中获取操作数 A。
2-CTA MMA：Blackwell 可以跨同一集群中的一个 CTA 对执行一个 UMMA，跨越这两个对等 CTA 的 TMEM。领导者 CTA 中的一个线程启动 MMA，但两个 CTA 在其飞行（运行）期间必须保持活跃。这通过在这一对 CTA 之间拆分 M 和 N，将 MMA 分块维度扩展到高达 256x256x16，减少了冗余流量并降低了每个 CTA 的内存占用。CTA 组大小（1 或 2）在一个内核内的所有 TMEM 和张量核心操作中必须保持恒定。

主要结论：前向受限于计算和指数运算，后向受限于共享内存带宽。因此，我们在前向传递中将 softmax 与 MMA 重叠，并在后向传递中减少共享内存流量。

前向传递：带条件重缩放的新 softmax 流水线

前向传递有两个矩阵乘法：QK^T 和 PV。在 Blackwell 上，张量核心变得快得多，但指数单元 (MUFU.EX2) 却没有。因此，softmax 不再仅仅是“两个矩阵乘法之间的那点事”——它是一个必须精心进行流水线处理的瓶颈。

前向传递简而言之：

pingpong调度每个CTA 2个Q和2个O分块：最大化 MMA 和 Softmax 之间的重叠。
2个softmax warpgroup：每个分块的 softmax 带有同步，以在计算指数时不发生重叠。
2^x 的软件模拟：将指数计算分布在硬件的 MUFU 和 FMA 上的软件模拟之间。
分阶段将P存储在TMEM中：减轻寄存器压力。
校正 warpgroup：指定“校正”warpgroup 来执行重缩放，将其移出关键路径。
在线 softmax（条件）重缩放：降低重缩放频率，最小化非矩阵乘法操作。
流水线：乒乓 Q 分块及专用校正阶段

FlashAttention-4 在每个 CTA 计算两个查询分块——Q^H 和 Q^L——每个覆盖 128 个查询 token，并以乒乓调度交替处理它们。

Blackwell 改变了 softmax 映射。S = QK^T 的累加器分块为 128x128，位于张量内存中；然而，在被读入寄存器后，硬件规定我们为每一行分配一个线程来对分块进行分区。我们使用两个 128 线程的 warpgroup，每个 Q 分块一个，每个 softmax warpgroup 执行以下操作序列：

每个线程从张量内存中将 S 的 128 个元素的一行加载到寄存器中
归约获得 rowmax（行最大值）和 rowsum（行和）
使用可调参数，决定这 128 个元素中哪一部分使用硬件的 MUFU 与软件模拟的 e^x
计算 P = \text{softmax}(S) 并转换为 BF16 精度
分阶段将 P 存回张量内存以减轻寄存器压力（而不是同时持有 S 的 128 个元素和 P 的 64 个 BF16 元素）
一旦 P 的 3⁄4 块被存储，就触发相应的 PV 矩阵乘法。

关键细节在于 exp 是受限部分。我们显式同步这两个 softmax warpgroup，使它们不会同时评估 exp，从而减少 MUFU 争用。

为了使重缩放远离关键路径，内核将其分配给一个专用的 warpgroup。校正 warpgroup 计算：

仅在最大值跳跃幅度较大时才进行重缩放：
O_j = \begin{cases}\exp(m_{j-1}-m_j)\,O_{j-1} + \exp(S_j-m_j)\,V_j, & \text{if } m_j - m_{j-1} > \tau,\\O_{j-1} + \exp(S_j-m_{j-1})\,V_j, & \text{otherwise.}\end{cases}
在迭代结束时应用最终归一化：O_{\text{final}} = \frac{O}{l_{\text{final}}}。
可选择计算并存储 LSE（Log-Sum-Exp）。

在最后我们仍然使用真实的最终统计数据进行归一化，因此跳过微小的重缩放步骤保留了最终输出的正确性，同时从关键路径中删除了许多向量计算。我们在 warp 粒度上做决定以避免分歧。

更快的指数运算：在 MUFU.EX2 和 FMA 上分布 2^x 计算

Softmax 需要大量的指数运算，而 MUFU 的吞吐量远低于张量核心的吞吐量。FlashAttention-4 通过在硬件 MUFU.EX2 路径旁边并行运行 exp2 的软件模拟来增加有效指数吞吐量，利用了原本未充分利用的 FMA 单元。

范围缩减 (Cody-Waite)：我们使用 Cody-Waite 范围缩减的经典技术，将指数计算分解为整数部分和分数部分：2^x = 2^{n} \cdot 2^{f}。在 IEEE 754 float32 中，乘以 2^n 仅仅是一个指数更新。
2^{x_{\text{frac}}} 的多项式近似 (Horner方法)：为了近似 2^f，我们将其重写为 Horner 形式以便高效求值。
2^{x_{\text{frac}}} \approx p_0 + p_1 x_{\text{frac}} + p_2 x_{\text{frac}}^{2} + p_3 x_{\text{frac}}^{3}

系数 p_0 = 1.0, p_1 \approx 0.6951, p_2 \approx 0.2276, p_3 \approx 0.0771 是使用 Sollya 软件包选取的，以最小化 [0, 1) 范围内的相对近似误差。
指数位移和相加：最后一步是将整数部分 n 和分数近似值 2^f 组合起来，形成 2^x \approx 2^n \cdot 2^f。由于 2^f \in [1,2) 的 float32 指数部分为 127，乘以 2^n 只是将整数 n 移位到指数部分，然后加上 2^f 的尾数位。
后向传递：共享内存流量主导的地方

优化 FlashAttention 的后向传递感觉就像往房间里塞一张过大的地毯：按平一个角落，另一个角落又翘了起来。后向传递计算的张量核心工作量大约是前向传递的 2.5 倍，它将五个 MMA 操作链接起来以重新计算 S，并为 dQ、dK、dP 和 dV 运行 QK 和 PV 的梯度 MMA，外加 P 和 dS 的逐元素工作。在 Blackwell 上，FLOPs 并不是后向传递的限制因素；共享内存带宽才是。

流水线：将 MMA 与 softmax 重叠

Hopper 时代的 FlashAttention-3 将 MMA 累加器保存在寄存器中，因此寄存器压力往往会迫使采用更串行的调度。在 Blackwell 上，累加器存在于 TMEM 中，这使得在保持多个 MMA 运行的同时，让 CUDA 核心处理 P 和 dS 的逐元素工作成为现实。因为在我们分析的 roofline（性能上限）中，指数吞吐量与两个 MMA 相当，隐藏它是值得的。

核心的重叠机制很简单：当我们在计算分块 j 的 softmax 时，我们已经发出了分块 j-1 的 dK 和 dQ 的 MMA。

为了减少共享内存流量，后向传递中重新计算的 S 和 P 相对于前向传递是转置的分块，因此中间变量已经是 S^T 和 P^T。然后我们可以直接将 P^T（以及随后的 dS^T）以完全符合 dV 和 dK MMA 所消耗的操作数 A 的布局存储在 TMEM 中。

TMEM 不能同时容纳五个完整的累加器和中间变量，因此 FA4 跨阶段重用 TMEM 列：S 和 P 共享一组列，dP、dS 和 dQ 共享另一组列。

2-CTA 后向传递：减少共享内存流量和全局原子加法
共享内存流量（Shared memory traffic）。 即使有了改进的流水线，并且十个 GEMM 操作数中有两个保存在张量内存中，后向传递仍然受限于共享内存带宽。我们利用 Blackwell 的 2-CTA MMA 模式来缓解这个问题，该模式在 CTA 对之间划分输出累加器。使用 M=256 且 N=K=128 时，两个 CTA 作为一个分块合作：每个 CTA 暂存一半的操作数 B 并仅保留其自己的累加器切片。这大致将操作数 B 的共享内存流量减半。
归约轴冲突（Reduction axis conflict）。 我们在五个后向 GEMM 中使用 M=256 和 N=K=128 的 MMA 分块来减少 B 流量，但 dQ MMA 的性质引入了不匹配。在 FlashAttention 的后向传递中，每个 CTA 拥有一个固定的 KV 分块（在外循环中跨 N 个 CTA 并行化），并在内循环中遍历 M 个分块。dQ 更新在外循环的 KV 序列上进行归约。2-CTA MMA 拆分的是输出分块，而不是归约操作，并且 dQ 的归约维度是 N，这已经在 CTA 对之间拆分了。每个 CTA 仍然需要针对其拥有的行进行完全的归约。
解决方案：DSMEM 交换。 我们通过使用集群内的分布式共享内存（DSMEM）在两个 CTA 之间交换一半的 dS 来解决这个问题。这重新打包了 dS，使其沿着非归约轴进行分区：每个 CTA 拥有 M/2 行，同时持有完整的 2N 归约。每个 CTA 的 dQ MMA 变成 (M/2, 2N) x (2N, d)，在张量内存中累加一个 (M/2, d) 分块。在 2-CTA 模式下，S、dP、dV 和 dK 的 MMA 保持 M=256，而 dQ 使用 M=128，且归约加倍到 2N=256。然后我们重新排序流水线以隐藏 DSMEM 延迟：在计算前一个分块的 dQ 之前，先计算当前分块的 dP。由于 dQ 分块可以与 P 一起放入 TMEM，它可以重用用于 S 的 TMEM 区域，因此 dP 和 dQ 不再像 1-CTA 模式中那样共享区域。通过这种排序，当前分块的逐元素 dS 与上一次迭代的 dQ MMA 重叠。
dQ 原子加法。 作为附带的好处，dQ 分解将全局原子归约的数量减半。原子操作具有不确定性且代价高昂，并且它们在每次内循环迭代中都会发生。因此，在 2-CTA 后向传递中，每个 CTA 只写 dQ 分块的一半，执行的全局原子归约数量也只有对应 1-CTA 版本的一半。
确定性模式（Deterministic mode）

不确定性的根源是 dQ 的全局原子累加。FA4 提供了一种确定性模式，使用信号量风格的锁和内存栅栏来串行化全局归约，以强制执行固定的累加顺序。然而，确定性并不意味着“一切停止”。FA4 通过 CTA 交错映射（swizzling）减少锁争用，并为因果掩码使用最短处理时间优先（SPT）排序来减少停顿。在实践中，确定性后向传递在我们的基准测试中能达到不确定性吞吐量的约 85-90%。

调度

因果掩码和可变序列长度使得注意力的负载不平衡，因为不同的工作块具有不同的主循环长度，因此 FA4 改进了网格线性化，并应用最长处理时间优先（LPT）调度来减少尾部时间。事实上，这些想法并非针对 Blackwell 或任何特定的 GPU 架构，我们在 FA3 中也使用了它们。

对于因果掩码（causal masking），标准网格顺序 (mblocks, heads, batches) 次优地按从短到长的顺序处理分块，所以 FA4 将 batch-heads 交错映射成 L2 缓存大小的段，并按 batch-head 段遍历网格，逆序迭代 mblocks，然后再迭代每个段内的 batch-heads。

对于可变序列长度（variable sequence length），由于不同的批次涉及不同数量的工作，从 LPT 调度启发式的角度来看，给定的批次处理顺序通常是次优的。为了纠正这个问题，我们可以启动一个预处理内核，根据每个工作块的最大执行时间对批次进行排序，并写入一个虚拟到实际批次索引的映射，注意力内核使用该映射按排序后的顺序遍历批次；此外，元数据可以被缓存，以便排序不会带来性能损失。

语言和框架：CuTe-DSL

FA4 完全在 CuTe-DSL（CUTLASS 的 Python 内核领域特定语言）中实现。内核是用 Python 编写的；DSL 会被降低为 PTX，然后 CUDA 工具包将其编译成 GPU 机器码。该编程模型反映了 CuTe/CUTLASS 抽象，并带有 PTX 转义后门，同时将编译时间相比 C++ 模板减少了约 20-30 倍。

注意力基准测试

我们展示了 FlashAttention-4 在 B200（BF16）上的结果，并将其与 FlashAttention-2，以及在 Triton、Gluon 和 cuDNN 中的实现进行了比较。对于 cuDNN，我们与 cuDNN 9.13 及最新版本 9.19.1.2 进行了对比。从 9.13 和 9.14 版本开始，我们与 cuDNN 团队合作，将 FlashAttention-4 中的一些技术整合到 cuDNN 中，以便我们的工作能够惠及尽可能多的从业者。

对于前向传递，FlashAttention-4 比 cuDNN 9.13 快 1.1-1.3 倍，比 Triton 快 2.1-2.7 倍。对于后向传递，在大序列长度下，FlashAttention-4 始终优于其他基准方案。
