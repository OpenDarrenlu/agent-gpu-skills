PyTorch Distributed API
DeviceMesh

Getting Started with DeviceMesh — PyTorch Tutorials 2.4.0+cu121 documentation

DeviceMesh: 管理process_group的抽象，用户可以用来创建inter-node和intra-node的processgroups。主要好处是说对多维并行例如2-D，3-D并行的情况提供了一个更加直观和简单的processgroup 划分方案，device mesh会自动setup NCCL communicators。否则user需要自己定义和创建NCCL communicators，过程相对复杂，很容易出错。

如何创建？

from torch.distributed.device_mesh import init_device_mesh
mesh_2d = init_device_mesh("cuda", (2, 4), mesh_dim_names=("replicate", "shard"))

如何访问创建好的DeviceMesh？

# Users can access the underlying process group thru `get_group` API.
replicate_group = mesh_2d.get_group(mesh_dim="replicate")
shard_group = mesh_2d.get_group(mesh_dim="shard")
Dtensor

pytorch/torch/distributed/_tensor/README.md at main · pytorch/pytorch (github.com)

顾名思义，是Tensor的distributed表示，是torch.Tensor的子类，可以通过from_local和to_local的互相转换

提供了一个统一的tensor sharding layout来表示不同的并行状态；
通过SPMD style进行shard计算
是PyTorch native 张量并行API的backend

DTensor 有三种placement类型：

Shard(tensor_dim)：在DeviceMesh指定的维度，沿着tensor_dim进行shard

Replicate：在DeviceMesh 维度进行tensor 复制

Partial：是一个中间表示。DeviceMesh指定的device之间，tensor shape相同，但是每个device都拥有部分的数值，这些数值需要进一步通过reduce（例如rum，max）来得到distributed tensor。

High-level APIs

对于一个已经存在的tensor或者tensor作为参数的module（例如nn.linear），提供了两种把参数转为DTensor的API：

distribute_tensor(tensor)

2. distribute_module(module, device_mesh, partition_fn, input_fn, output_fn)

这里的partition_fn提供了对每个parameter转为DTensor的函数，例如在ColwiseParallel种对linear weight的shard：

def _partition_linear_fn(self, name, module, device_mesh):
    # colwise shard weight/bias to Shard(0), weight be Shard(0)
    # means Colwise as Linear is input * weight^T + bias, where
    # weight would become Shard(1)
    for name, param in module.named_parameters():
        dist_param = nn.Parameter(
            distribute_tensor(param, device_mesh, [Shard(0)])
        )
        module.register_parameter(name, dist_param)
    ########################
    if isinstance(module, nn.Linear):
        partition_fn = self._partition_linear_fn

行切分和列切分

defined in /home/bduser/zhuhong/pytorch/torch/distributed/tensor/parallel/style.py




我们先简单复习一下按行切分和按列切分：

行切分 RowwiseParallel：

f和g是共轭算子，在forward和backward方法不同。

	f	g
Forward	Split	Allreduce
Backward	Allgather	No communication

Backward过程的理解：

g的backward：假定现在我们要对 W_i 求梯度，则可推出 \frac{\partial L}{\partial W_i} = \frac{\partial L}{\partial Y} * \frac{\partial Y}{\partial Y_i} * \frac{\partial Y_i}{\partial W_i} = \frac{\partial L}{\partial Y} * \frac{\partial Y_i}{\partial W_i} ，经过g 的forward，每块GPU上已经拥有了 Y 和各自的 Y_{i}，两块GPU就可以独立计算各自的梯度，不需要额外通信。
f的backward：forward过程 X 进行了切分，backward要计算X的完整梯度，需要把 X_{1} 和 X_{2} 的梯度通过allgather拼接起来， \frac{\partial L}{\partial X} = concat[\frac{\partial L}{\partial X_1}, \frac{\partial L}{\partial X_2}]

Pytorch 源码：

    def __init__(
        self,
        *,
        input_layouts: Optional[Placement] = None,
        output_layouts: Optional[Placement] = None,
        use_local_output: bool = True
    ):
        super().__init__()
        self.input_layouts = (input_layouts or Shard(-1), ) # 这里定义了输入X的layout，要在last dimension进行切分
        self.output_layouts = (output_layouts or Replicate(), ) # 输出的layout是replicate
        self.use_local_output = use_local_output

对输入X的切分处理：

通过把X转成Dtensor的layout来处理

    @staticmethod
    def _prepare_input_fn(input_layouts, desired_input_layouts, mod, inputs, device_mesh):
        input_tensor = inputs[0]
        if not isinstance(input_tensor, DTensor):
            input_tensor = DTensor.from_local(input_tensor, device_mesh, input_layouts, run_check=False)

        if input_layouts != desired_input_layouts:
            input_tensor = input_tensor.redistribute(placements=desired_input_layouts, async_op=True)
        return input_tensor

对weight和bias的处理：

对weight： 直接行切分，但是做linear的时候weight通常要转置，所以这里变成了列切分Shard（1）；bias是直接在linear的结果矩阵上加偏置，这里是replicate的语义。

    def _partition_linear_fn(self, name, module, device_mesh):
        # Rowwise shard weight to Shard(1), bias to Replicate(), weight be Shard(1)
        # means Rowwise as nn.Linear is input * weight^T + bias, where
        # weight would become Shard(0)
        module.register_parameter("weight", nn.Parameter(
            distribute_tensor(module.weight, device_mesh, [Shard(1)])
        ))
        if module.bias is not None:
            module.register_parameter("bias", nn.Parameter(
                distribute_tensor(module.bias, device_mesh, [Replicate()])
            ))
列切分 ColwiseParallel：

Forward和backward的操作和通信：

	f	g
Forward	Copy	Allgather
Backward	Allreduce	Split

Backward过程的理解：

g的backward：\frac{\partial L}{\partial W_i} = \frac{\partial L}{\partial Y_i} * \frac{\partial Y_i}{\partial W_i} ，其中 \frac{\partial L}{\partial Y_i} 可以通过 \frac{\partial L}{\partial Y} 沿着列做split来得到，每张GPU在forward的allgather之后就有了完整的 Y ，可以直接计算出 \frac{\partial L}{\partial Y}
f 的backward：因为对于损失 L ， X 既参与了 Y1 的计算，也参与了 Y2 的计算。因此有 \frac{\partial L}{\partial X} = \frac{\partial L}{\partial X}|1 + \frac{\partial L}{\partial X}|2 。其中 \frac{\partial L}{\partial X}|i 表示第i块GPU上计算到X时的梯度。
Example：列切分+行切分

PyTorch提供了一个例子方便理解整个过程：

examples/distributed/tensor_parallelism/tensor_parallel_example.py at main · pytorch/examples (github.com)

主要步骤：

对input X，直接to到每个GPU上，模式为replicate
然后列切分weight A，模式为Shard(dim=0)（考虑转置的因素），调用scatter 把切分的weight发送到不同的GPU上
接下来行切分weight B，模式为Shard(dim=1)（考虑转置的因素），调用scatter 把切分的weight发送到不同的GPU上
最后对输出结果做allreduce，模式为Partial(sum)，调用allreduce对所有GPU的计算结果汇总。
Transformer block的切分策略
MLP层
MLP

考虑一般的MLP block，有两个linear，采用的weight切分策略是对A进行列切分，对B进行行切分

f 的forward：把输入X拷贝到两块GPU上，每块GPU即可独立做forward计算。
g 的forward：每块GPU分别进行forward计算，得到Z1和Z2，GPU间做一次AllReduce，得到结果Z。
g 的backward：每块GPU都可以直接计算出 \frac{\partial L}{\partial Z} ，各自独立做梯度计算。
f 的backward：计算\frac{\partial L}{\partial X} 时，两块GPU需要做一次AllReduce，把各自的梯度 \frac{\partial L}{\partial X}|1 和 \frac{\partial L}{\partial X}|2 相加即可。

MLP的通信量：

2次allreduce，每次Allreduce的通信量为 2\Phi （reduce-scatter和allgather 各为 \Phi ），则总通信量为 4\Phi=4bsh

为什么对A进行列切分，对B进行行切分？

因为整体的原则是尽量保证各GPU上的计算相互独立，减少通讯量。在对A完成linear运算后，需要进行一次GELU的激活计算，但GELU是非线性的，如果A采用行切分，在GELU前需要做一次Allreduce，产生额外的通信。

GELU(Y)=GELU(Y1+Y2)\ne GELU(Y1)+GELU(Y2)\\GELU(Y)=[GELU(Y1), GELU(Y2)]\\

Attention层
Self-Attention

Self-Attention对number head进行切分，也就是对 W^Q, W^K, W^V 进行列切分，对线性层 W^O 进行行切分

方式和MLP基本一致，forward和backward原理也一致

总通信量：

和MLP一致，2次allreduce，每次Allreduce的通信量为 2\Phi （reduce-scatter和allgather 各为 \Phi ），则总通信量为 4\Phi=4bsh

Embedding

输入embedding

对于第一个nn.Embedding layer，embedding table使用行切分，输入X replicate到每个GPU；也就是说，每个GPU上维护部分词表，token在每个GPU上查找的结果进行一次allreduce，就能得到最终的输入。

输出层embedding：

这个embedding本质上是一个线性层，nn.Linear layer，也就是把输出的logits映射回词表，得到每个位置的次。这里对词表使用列切分。

关于和sequence parallel 的结合使用：

考虑norm 需要使用sequence parallel的问题，tok_embeddings的输出要在sequence 维度进行切分，所以设置了output_layouts=Shard（1）；
而最终的embedding linear接收的输入也是沿着sequence切分的，所以input_layouts=Shard（1）；
model = parallelize_module(
    model,
    tp_mesh,
    {
        "tok_embeddings": RowwiseParallel(
            input_layouts=Replicate(),
            output_layouts=Shard(1),
        ),
        "norm": SequenceParallel(),
        "output": ColwiseParallel(
            input_layouts=Shard(1),
            output_layouts=Replicate()
        ),
    }
)




Loss Parallel / Cross-Entropy 层

输出层做完embedding后，通常我们会通过allgather收集所有GPU上的计算结果，然后对每一行做softmax，就可以得到当前位置每个词出现的概率。然后用这个概率和真实值做cross-entropy计算。

allgather产生的通信量是 b*s*v ，当词表增大时，会带来更大的通信开销。同时增加了memory的占用。所以PyTorch 提出了Loss parallel，其实就是Megatron-LM种的cross-entropy的并行计算。

这个图画的有点抽象，我尝试写下我自己的理解：

蓝色块表示sharded tensor，也就是存在一个GPU上的tensor；
黄色块表示partial values，也就是需要不同GPU之间做allreduce的值；
绿色块表示replicated tensor，也就是allreduce之后所有的GPU上都有的值。

左上角的input就是完成了embedding的计算，得到的 b*s*v 的logits输出，这里简化为二维，把batch和sequence length合并，统称为batch，shape变成了 batch*vocab 。这里的logits的含义，我们可以理解成 X*WE^{T} 的结果就是输出 X 与词表 WE 的相似度。也就是说，对于特定行，表示了改token与词表种每个词的相似度。

从Loss Parallel的角度，词表维度vacab进行了列切分。这里的一个蓝色块就是沿词表切分后在一个GPU上的logits。

PyTorch里面计算Cross-Entropy 分成两个主要步骤：计算log_softmax和nll_loss_forward。cross_entorpy 是 log_softmax 和 nll_loss 两个函数的组合，log_softmax 负责进行 softmax 归一化及取对数，nll_loss 负责计算交叉熵。

Pytorch 的实现在

cpu: pytorch/aten/src/ATen/native/SoftMax.cpp at main · pytorch/pytorch (github.com)

cuda: pytorch/aten/src/ATen/native/cuda/SoftMax.cu at main · pytorch/pytorch (github.com)

为了解决溢出问题，需要减去最大值

logsoftmax = log(softmax(x_i))=log(\frac{e^{(x_i-M)}}{\sum_{j=0}^{n}{e^{(x_{j}-M)}}})

M 为所有 x_i 的最大值。

首先对log_softmax的计算，

最左侧第一个黄色块，表示对当前GPU上的logits数值求得的局部最大值，然后GPU间做allreduce可以得到绿色块，也就是全局最大值。
根据全局最大值，在当前GPU上计算 e^{(x_i-M)} ，然后求和计算 \sum_{j=0}^{n}{e^{(x_{j}-M)}}，也就是第二个黄色块，GPU之间做allreduce，得到全局所有词的总和绿色块
蓝色块根据全局总和计算得到log softmax值

接下来是nll_loss_forward的计算

loss(x, target)=-log(\frac{e^{(x[target]-M)}}{\sum_{j=0}^ne^{(x[j]-M)}})

4. 根据target的token idx，然后得到partial value也就是每个GPU上的local target token idx，也就是右侧的第一个黄色块，第2个黄色块是在当前GPU上根据前面的local target idx计算得到的loss（x，target），最后通过allreduce 得到全局的loss结果绿色块。

Sequence Parallel

主要应用的layer：nn.LayerNorm,nn.Dropout,RMSNormPython, etc.

这个和Megtron-LM中的序列并行是一致的，主要作用是降低activation的memory。

张量并行后，需要做相应的 AllReduce 操作得到完整中间结果，如下图的 \bar{f} (forward为allreduce，backward no operation）。

f 操作是forward是no operation，backward是allreduce； \bar{f} 操作forward是allreduce，backward是no operation。

事实上，Dropout（部分随机）和LayerNorm（隐层维度求和） 不依赖完整结果，所以可以拆分到不同的GPU上来完成，也就是对sequence 进行切分。在不影响通信量的情况下，Allreduce 可以被拆分为 reduce-scatter 和 allgather，如下图所示：

g 操作是forward是allgather，backward是reduce-scatter； \bar{g} 操作forward是reduce-scatter，backward是allgather

PyTorch的实现参考SequenceParallel，也就是在sequence dim进行切分，默认是1。

def _prepare_input_fn(sequence_dim, mod, inputs, device_mesh):
    input_tensor = inputs[0]
    if isinstance(input_tensor, DTensor):
        return inputs
    elif isinstance(input_tensor, torch.Tensor):
        return DTensor.from_local(input_tensor, device_mesh, [Shard(sequence_dim)], run_check=False)
    else:
        raise ValueError(f"expecting input of {mod} to be a torch.Tensor or DTensor, but got {input_tensor}")

对于transformer block，主要是针对每个layer定义TP和SP的plan：

for layer_id, transformer_block in enumerate(model.layers):
    layer_tp_plan = {
        "attention_norm": SequenceParallel(),
        "attention": PrepareModuleInput(
            input_layouts=(Shard(1), None),
            desired_input_layouts=(Replicate(), None),
        ),
        "attention.wq": ColwiseParallel(),
        "attention.wk": ColwiseParallel(),
        "attention.wv": ColwiseParallel(),
        "attention.wo": RowwiseParallel(output_layouts=Shard(1)),
        "ffn_norm": SequenceParallel(),
        "feed_forward": PrepareModuleInput(
            input_layouts=(Shard(1),),
            desired_input_layouts=(Replicate(),),
        ),
        "feed_forward.w1": ColwiseParallel(),
        "feed_forward.w2": RowwiseParallel(output_layouts=Shard(1)),
        "feed_forward.w3": ColwiseParallel(),
    }

    # Adjust attention module to use the local number of heads
    attn_layer = transformer_block.attention
    attn_layer.n_heads = attn_layer.n_heads // tp_mesh.size()
    attn_layer.n_kv_heads = attn_layer.n_kv_heads // tp_mesh.size()

    # Custom parallelization plan for the model
    parallelize_module(
        module=transformer_block,
        device_mesh=tp_mesh,
        parallelize_plan=layer_tp_plan
    )
