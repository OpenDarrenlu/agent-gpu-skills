0x01 如何建模群友/好友？

首先 OO/AOP 太经典不想用。

想象中，每个人特质不一样。应该有不同的 rust trait 而不是固定抽象。

于是实现了 coding 建模，对每个微信好友，实时生成对应的 class，用的时候热加载。

想象很丰满，真做完之后，我不知道为啥要这么做.. 做科研已经讲了太多故事，这么设计实用意义在哪儿？除了对模型要求更高了。

目前是人物传记模式（像三国志）。会提取基本信息/聊天记录形成人物小传。得益于 LLM 可解释性，目前生成的准确率挺高。

0x02 如何设定性格？

完成了建模，就要开始交互。

作为负责任的群主，肯定优先满足群友喜好：弄个”毒舌+高傲+温柔+大波浪”妹子！

但实测下来，直接赋予 LLM 某种性格会有致命缺陷。无论什么场合，AI 总是表现出高傲+温柔，这非常不自然。

所以就在想，到底什么是性格？

就像有的人因为评委沙雕，在晋升答辩现场也刚；但有的就很服软，这种性格差异来自哪儿？无非立场和经历不同。

总结下来：性格是人面对事件的一组反应。抛开初始设定差异，人生选择的概率密度坍缩后，当前性格其实就是已有的经历。

因此，MirrorWe 中写了一段 doll.md 的自传，目的是生成想要的性格。测下来显著优于直接设定。

HuixiangDou 已经写了 123 版本，开发过程中显然很多组件可以复用。例如 model 输出的 think 部分，模型选择方式，tpm/rpm 约束..但实在懒得造新轮子。

直接复用了 moonshot/kosong 的接口设计。不满足需求的地方先 adapter ，打算后续给 kosong 发 PR。目测作者应该挺好。
