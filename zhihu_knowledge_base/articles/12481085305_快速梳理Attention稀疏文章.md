# 快速梳理Attention稀疏文章

**作者**: NULL让智能更廉价: AI芯片软件栈与异构计算系统

**原文链接**: https://zhuanlan.zhihu.com/p/12481085305

---

静态丢弃：

Big Bird：https://arxiv.org/abs/2007.14062

StreamingLLM ： https://arxiv.org/abs/2309.17453

动态丢弃(不可召回的 KV 缓存)：

H2O: https://arxiv.org/abs/2306.14048

SnapKV：https://arxiv.org/abs/2404.14469

全注意力动态稀疏(针对Attention算子，prefill/decode都可用)：

Deja Vu：https://arxiv.org/abs/2310.17157

SParQ：https://arxiv.org/abs/2312.04985

SampleAttention：https://arxiv.org/abs/2406.15486

全缓存动态丢弃(可召回的 KV 缓存)：

You Only Cache Once： https://arxiv.org/abs/2405.05254 （解码器生成全局 KV 缓存，此篇不够典型但也有类似含义）

Quest：https://arxiv.org/pdf/2406.10774 （建立缓存页为粒度召回）

RetrievalAttention：https://arxiv.org/abs/2409.10516（GPU静态Prefill+最近窗口静态+CPU建索引动态）

Post-Training Sparse Attention with Double Sparsity https://arxiv.org/abs/2408.07092 （post静态+动态，通道和token两个稀疏粒度）

ClusterKV: https://arxiv.org/abs/2412.03213 (以语义簇为粒度召回)

LoRC：https://arxiv.org/abs/2410.03111 （低秩近似压缩， perlayer动态大小混合压缩， 类似与DeepseekV2压缩的细粒度后训练版本）

一旦使用可召回策略，那么KV Cache的GPU Memory Offload调度也就开始重要起来了：

FlexGen： https://arxiv.org/abs/2303.06865

InstInfer： https://arxiv.org/abs/2409.04992

MoE-Lightning：https://arxiv.org/abs/2411.11217

ShadowKV：https://arxiv.org/abs/2410.21465 (低秩压缩 + Offload)
