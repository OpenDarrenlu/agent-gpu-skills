Overview

本章节主要是关于，如何在Linux命令行下，将网络硬盘通过WebDAV、Samba等协议进行远程挂载的方法。常用的场景是，家用NAS盘提供了可挂载的硬盘，我们需要在局域网环境下、在Linux机上进行挂载、使用。

我们希望它是可读写的、开机自启动的、失败了也不会影响到正常使用。

本文主要从客户端的角度出发。

通过NFS挂载

这个太简单了，就不讲了。如果有NFS挂载，那是最方便的。

sudo apt-get update
sudo apt-get nfs-common
通过WebDAV挂载

WebDAV，全称Web Distributed Authoring and Versioning，是基于HTTP协议的一个网络文件系统，提供了读写功能。

笔者不推荐使用WebDAV协议来在高速局域网环境下挂载NAS硬盘。此协议的版本、缓存功能在高速局域网环境下其实并没有用，还会降低大文件的拷贝速度。举例子，每次将大文件从本地上传至云端硬盘时，WebDAV会先将文件拷贝到本地的一个缓存场所，然后再上传。那么，一次复制变成了两次复制，耗时将会翻倍。

# 安装davfs2，是一个用来支持WebDAV协议的程序。
sudo apt-get update
sudo apt-get install davfs2

# 指定服务器地址和账号密码。
# 编辑密钥文件，增加一行，对应 服务器地址(一定是https或者http协议开头) / 账号 / 密码(用双引号圈起来、防止特殊字符的干扰)
# 密钥文件是/etc/davfs2/secrets
# 比如：
http://192.168.1.5:5005 mike "eearfs#1231"

# 如果环境变量里有PROXY，而我们又不需要通过PROXY连接网络硬盘，则需要修改/etc/davfs2/davfs2.conf文件。
增加以下一行：
use_proxy 0

# 新建一个文件夹，作为挂载到的本地地址。
mkdir -p /mnt/nas_webdav

# 设置开机自动挂载。
# 编辑/etc/fstab文件。增加一行，对应：服务器地址（和上面的相同），挂载到的本地文件夹，协议（不用改），参数
# _netdev,x-systemd.automount,很重要，提示这是个网络硬盘, 系统会在网络就绪后多次重试挂载硬盘；
# nofail很重要，表示就算这个硬盘加载失败，系统也能正常启动; 
# uid=1000、gid=1000会让系统将文件权限设置为对应的本地机器账号可用。用id -u看当前账号的uid，id -g看当前账号的gid。
# http://192.168.1.5:5005 /mnt/nas_webdav davfs _netdev,user,rw,nofail,uid=1000,gid=1000 0 0

# 测试是否能正常挂载，如果不能，请修改/etc/fstab文件直到可以，否则不要轻易重启。
sudo mount /mnt/nas_webdav
通过Samba挂载

Samba是很早的协议，各个系统都有兼容。在局域网里用的比较多，一般不在公网使用，安全性有问题。

Samba协议比较简单，在局域网环境里用比较方便，比WebDAV更适合用于在局域网环境里挂载NAS硬盘。

# Linux原生支持Samba挂载，如果发现自己没有mount-cifs程序，才需要安装
# 测试是否有该程序
mount-cifs
# 没有的话，再装
sudo apt-get update
sudo apt-get install cifs-utils

# 新建一个文件夹，作为挂载到的本地地址。
mkdir -p /mnt/nas_samba

# 新建一个文件，用来存储samba的密钥。其实也可以直接写到/etc/fstab文件，但如果账号密码里有特殊字符，那么就必须像这样写到一个单独的文件里。
# 位置随意，但是权限必须设置为仅root可见，防止泄密。最好放到/root里。
sudo vim /root/.smbcredentials # 编辑。
sudo chmod 600 /root/.smbcredentials
# 文件内容是你的账号密码，格式如下：
username=Mike
password=blabla@123


# 设置开机自动挂载。
# 编辑/etc/fstab文件。增加一行，对应：服务器地址，挂载到的本地文件夹，协议（不用改），参数
# nounix,file_mode=0777,dir_mode=0777是用来提高兼容性的。
//192.168.1.5/nvme12-132XXXX0363 /mnt/nas_samba cifs _netdev,nofail,credentials=/root/.smbcredentials,iocharset=utf8,uid=1000,gid=1000,nounix,file_mode=0777,dir_mode=0777 0 0

# 测试是否能正常挂载，如果不能，请修改/etc/fstab文件直到可以，否则不要轻易重启。
sudo mount /mnt/nas_samba
