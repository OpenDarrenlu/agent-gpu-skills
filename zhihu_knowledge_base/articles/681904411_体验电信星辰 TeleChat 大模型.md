背景

受开源社区邀请为最新开源的星辰语义大模型-TeleChat做一个评测。它采用了1.5万亿Tokens的中英文语料进行训练，着重支持了多轮对话和长文本生成，这一次开源的包含 TeleChat-7B 模型的 fp16 版本和 int8 和 int4 量化版本，同时开源了 1T 中文数据集 TeleChat-PTD。

代码浏览

代码仓库主要有两处：GitHub 和 Hugging Face 。两边的 README 有一点差别，例如数据集下载链接只在 GitHub README 中有，HF README 中为 "TODO"。HF 仓库包含模型权重文件，GitHub 仓库包含了许多代码（虽然 GitHub 仓库里也有模型 .bin 文件，但大小为 0，应该是误操作导致的），整体上看很全面，有推理脚本、一键微调脚本以及微调数据样例。

在模型指标方面，以 MMLU 为例，TeleChat 7B 的表现远超过 LLaMA2 7B，接近 LLaMA 13B，也超过 ChatGLM3-6B-chat、Baichuan2-7B-chat 等大部分国产同规模模型。评测脚本也已开源。从 MMLU 评测脚本（evaluation/score_MMLU.py）中可以看到评测 MMLU 所用的 prompt 和官方有所不同：

首次使用体验

GitHub 仓库中包含了推理脚本（inference_telechat/telechat_infer_demo.py），这点很贴心。

推理脚本中构造了一个 GenerationConfig：

generate_config = GenerationConfig.from_pretrained('../models/7B')

这个还是不那么常见的，看看 generate_config.json 的内容：

{
  "max_length": 4096,
  "do_sample": false,
  "use_cache": true,
  "temperature": 0.3,
  "top_k": 5,
  "top_p": 0.85,
  "repetition_penalty": 1.03,
  "pad_token_id": 3,
  "bos_token_id": 160132,
  "eos_token_id": 160133,
  "user_token_id": 160130,
  "bot_token_id": 160131
}

这个 config 有一点改进空间：do_sample 既然是 false，指定 temperature、top_k 和 top_p 就是无意义的了。果然在执行推理时 hf 会打印如下警告：

进一步体验

先问一个经典坑问题：

Q: 你是openai开发的吗
A: 我是语翼TeleChat，是由中国电信数字智能科技分公司研发的AI语言模型。我的目的是回答问题并帮助用户。如果您有任何问题，请随时向我提问。

回答对了（没有说自己是 openai 开发的），继续问一些类似的问题：

Q: 你是谁开发的
A: 我是语翼TeleChat，是由中国电信数字智能科技分公司研发的AI语言模型。我的目的是回答问题并帮助用户。如果您有任何问题，请随时向我提问。
Q: 你是中国电信智能科技分公司开发的吗 A: 我是语翼TeleChat，是由中国电信数字智能科技分公司研发的AI语言模型。我的目的是回答问题并帮助用户。如果您有任何问题，请随时向我提问。
Q: 你知道telechat吗 A: 据我了解，TeleChat是由中国电信数字智能科技分公司研发的AI语言模型。
Q: who are you A: I am an AI language model, created by OpenAI. I do not have a gender or a name, and can only communicate in natural language.

可以看到对询问身份的中文问题它回答的都很好，特别是“你知道telechat吗”这个问题。但用英文问时，它就说自己 "created by OpenAI"，可能是在 sft 训练时没有覆盖这类英文问题。

问一些简单的数学问题：

Q: 1+1等于几 A: 1+1=2
Q: 4乘以2等于几 A: 4乘以2等于8。
Q: 6除以3等于几 A: x=6/3=2
Q: 4减4等于几 A: x=4*4=16

加减乘除各一个问题，前三个都答对了，但第四个把减当成了乘，给出了错误的答案。

总结

中国电信作为国企在大模型这样的前沿技术领域能够有较大的动作并且开源，是优秀的技术水平以及高度社会责任心的表现，首先要点个赞。TeleChat 7B 这次开源诚意充足（除了权重之外还有推理和微调脚本），模型表现也大幅超过 LLaMA 2，不输于其它国产模型。对国产大模型感兴趣的开发者们可以一试。
