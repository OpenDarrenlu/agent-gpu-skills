# 大模型训练：如何优化MFU

**作者**: Lin Zhang​香港科技大学 计算机科学技术博士

**原文链接**: https://zhuanlan.zhihu.com/p/717509355

---

对大模型训练来说，Model FLOPs Utilization（MFU）是衡量训练效率最重要的指标。我相信很多人都熟悉MFU的计算公式，也知道MFU分数达到多少会是一个不错的水平。

然而，仅仅停留在这样的认识显然是不足的。对于从业者来说，如何适配新模型，知道往什么方向优化MFU是一门必修课。

笔者会从实践的角度出发，教大家如何拆解MFU公式，逐步提高大模型的训练效率。

公式中的第一项对应transformer layers中的线性层，第二项对应注意力层，而第三项对应最后的输出层。

对于非长序列训练场景[3]来说，简单起见，我们可以忽略第二项，使用6NBs来快速估计model FLOPs。其中，N为模型的参数量（对MoE模型来说则是激活参数量）。

得到model FLOPs，我们将其除以设备数，然后除以每个设备的理论算力，最后除以每个iteration的时间，我们就得到了MFU分数。

下面是一个具体的例子[4]：在64卡A100上训练GPT-175B模型（B=64、s=2048），如果iteration time为13.75秒，我们可以快速估计得到MFU分数为：

MFU = 6 * 175e9 * 64 * 2048 / 64 / 312e12 / 13.75 = 50%. 

这和论文中使用更加精确的公式得到的结果（51.4%）十分接近。

另外，在近期Deepseek发布的技术报告[5]中，笔者发现两组有趣的数据：

1）对于LLaMa-13B（B=4096、s=4096、设备数=512），我们估计得到MFU为84.3%，如此高的MFU并不现实（原因后面会展开解释）。这里有可能汇报的是LLaMa-7B的数据，对应MFU为45.4%。

2）对于DeepSeekMoE-16B（激活参数量=2.8B、B=4608、s=4096、设备数=640），我们估计得到的MFU为24.3%，这个结果显然还有很多优化空间（Fine-Grained MoE结构要比普通MoE更难优化）。

MFU的计算拆解

当我们实际训练一个大模型时，会得到一个或高或低的MFU分数。那么，到底是哪些因素导致出现这个分数呢？如果出现一个不够满意的MFU分数时，我们又该往哪些方向优化训练的效率？

为了回答以上问题，我们需要对MFU的计算进行拆解。

由于MFU反映的是理论的模型Matmul FLOPs计算时间和实际的模型训练时间之间的比例，我们可以对这个比例进行如下的拆解：

\text{MFU} = \frac{\text{Optimal Matmul Time}}{\text{Matmul Time}} \times \frac{\text{Matmul Time}}{\text{Compute Time}} \times \frac{\text{Compute Time}}{\text{Iteration Time}}.

由此可见，MFU的分数由以下三项分数决定。

1）Matmul算子效率，其中也包括FlashAttention算子中的Matmul。一方面，我们可以对单算子效率进行profile，例如CUTLASS中的Matmul算子在A100上能够取得大概 260/312=83.3%的效率。另一方面，我们可以对训练步骤中所有的Matmul和FlashAttention算子的时间进行汇总，用来估计第一项分数的数值。

2）Matmul算子比重，即Matmul类算子的时间占所有计算算子（例如vector类算子）时间的比例。第二项分数的大小反应出我们对其他类型算子的优化程度，而对这些小算子采取算子融合的策略往往能够大幅提高Matmul算子的比重。

3）实际计算比例，即所有算子的计算时间占总训练时间的比例。其中，非计算的部分包括，不可隐藏的通信开销，流水线并行中的bubble开销，和host-bound导致的CPU侧的开销等等。

对于任何一个训练任务，我们需要对这三项指标分别进行监督和优化。

具体来说，对于一个表现不错的稠密模型训练任务来说，我们大概可以得到80% x 85% x 75% = 51%的MFU分数。其中，如果能实现100%的实际计算比例，MFU分数可以达到68%。

对于大部分从业者来说，检查前两项指标，也就是算子的效率往往是工作的重点。

特别是对于不够成熟的硬件系统，Matmul算子或者FlashAttention算子在小shape、或者非常规shape下的性能往往会出现严重的劣化。

另外，对于新的模型结构来说，某些非融合算子如果不得到相应的优化，也会带来比较大的性能问题，例如DeepSeekMoE中的小专家计算。

除了以上琐碎的工程问题，对于第三项指标的优化，包括通信隐藏[6]、bubble填充等，则给追求极致MFU分数的从业者们带来更多的系统挑战和更大的研究空间。
