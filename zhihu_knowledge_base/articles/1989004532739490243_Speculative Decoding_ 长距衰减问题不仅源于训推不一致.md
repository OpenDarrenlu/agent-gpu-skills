# Speculative Decoding: 长距衰减问题不仅源于训推不一致

**原文链接**: https://zhuanlan.zhihu.com/p/1989004532739490243

---

对于 Speculative Decoding 这个 topic，EAGLE-3 已经成为了事实标准，即通过一个 单层 transformer layer, 复用 target model 的 hidden states，自回归的预测 draft tokens. 然而，我一直模糊的感觉，复用 KV Cache 要优于复用 Hidden States，但一直没有时间进行一次仔细的分析。之前和 cunxiao 哥@真实哥聊天也聊到，复用 kv cache 的效果优于复用 hidden states，这次就来分析一下。

主流方案 EAGLE-3, HASS, CORAL 认为，EAGLE 的长距预测能力 (指预测多个 draft tokens) 显著下降源于 训推不一致 问题，即训练时只用 target hidden states 预测 1 个 token，而推理时要用 target hidden states 预测多个 token。诚然，这确实是一个很直观且易解决的问题，三篇 paper 也各自尝试解决这个问题。而本文想从一个更高的视角来说明，复用 hidden states 也会导致长距衰减问题。一个可行的解决方案是复用 KV Cache.

3. 复用 KV Cache 的优势：无损的动态重聚焦

相比之下，KV Cache 存储的是未被加权的原始特征集合：

S_{KV} = \{ (k_1, v_1), (k_2, v_2), \dots, (k_t, v_t) \}

这是一个全集，没有发生任何维度的坍缩或信息的丢失。

当 Draft Model 复用 KV Cache 时，它并不是在“解压”历史，而是在重新检索历史。它拥有独立的 Query q' ，可以直接计算新的注意力分布：

h'_{future} = \sum_{i=1}^t \text{Softmax}(q'_{future} \cdot k_i^T) \cdot v_i

即使 Target Model 在上一轮忽略了 v_k, Draft Model 依然可以通过生成指向 k_k的 Query，将 v_k完整地提取出来。

结论：复用 KV Cache 赋予了 Draft Model Re-Attention (重注意力) 的能力，使其能够绕过 Target Model 的历史偏见，直接访问高质量的 Ground Truth 特征。

二、问题建模

为了进一步探讨“复用 hidden”和“复用 KV”对于 draft model 预测能力的影响，我们将 draft model 建模为单层 transformer layer。主要思路是，给定输入 token x_{\le t} 以及其对应的 hidden states h_{\le t} 和 KV cache k_{\le t}, v_{\le t} , 想让 draft model 成功预测 x_{t+i} 时，draft model 需要成功拟合的参数量。包含以下假设：

只复用最后一层的 hidden states / kv cache；
暂不考虑 input embedding 和 hidden states / kv cache 的拼接；
target model 冻结，不改变权重，只训练 draft model；
移除 output MLP / LN / 残差的影响，因为复用 hidden states 和复用 kv cache 都需要训练这些部分；
只考虑标准 attention (MHA, MQA, GQA), MLA 暂未分析.

以上就是我们的全部假设，接下来从标准 attention 公式出发，进行对比：(不带 ~ 的元素认为来自于 target model, 带 ~ 的元素认为来自于 draft model).

h_{t+i} = \text{Softmax}\left(\frac{q_{t+i} K_{\le t+i}^T}{\sqrt{d_k}}\right) V_{\le t+i} \\

我们暂不考虑常数项 \frac{1}{\sqrt{d_k}} 的影响，并记 attention weight 为 \alpha_j^{t+i} = \frac{\exp(q_{t+i}k_j^T)}{\sum_j \exp(q_{t+i}k_j^T)} , 表示第 t+i 个 token 对第 j 个 token 的注意力系数。

三、复用 Hidden States 的预测难度

我们首先把 h_{t+i} 的计算拆分为两部分：

h_{t+i}=\underbrace{\alpha_{\le t}^{t+i}V_{\le t}}_{对\text{prefix}的系数} + \underbrace{\alpha_{t:t+i}^{t+i}V_{t:t+i}}_{对\text{draft tokens}的系数} \\

当我们复用 hidden states 时， h_{\le t} 可视为第 t 个 token 对前 t 个 token 的 attention 系数和 Values 的加权和：

h_{t}=\alpha^t_{\le t}V_{\le t} \\ 复用 hidden states 时, q_{t}, K_{\le t}, V_{\le t} 由 h_{\le t} 变换而来：

\widetilde{q}_{t+i} = W_q \widetilde{h}_{t+i} \\ \widetilde{K}_{\le t+i} = W_k \widetilde{h}_{\le t+i}= (W_k h_{\le t}, W_k \widetilde{h}_{t:t+i})\\ \widetilde{V}_{\le t+i} = W_v \widetilde{h}_{\le t+i} = (W_v h_{\le t}, W_v \widetilde{h}_{t:t+i})\\ \widetilde{h}_{t+i+1}=\sum_{j=1}^t \underbrace{\text{SoftMax}(W_q(\widetilde{h}_{t+i} h_j^T)W_k)}_{\widetilde{\alpha}_{j}^{t+i}， 重新估计}\underbrace{W_v h_j}_{\widetilde{v}_j, 重新估计} \\ + \sum_{j=t+1}^{t+i} \underbrace{\text{SoftMax}(W_q(\widetilde{h}_{t+i} \widetilde{h}_j^T)W_k)}_{\widetilde{\alpha}_{j}^{t+i}， 重新估计}\underbrace{W_v \widetilde{h}_j}_{\widetilde{v}_j, 重新估计} \\ 可以看到，每一部分都需要重新估计注意力系数和 attention values. 由 attention sink 可知，LLM 通常更关注最相近的几个 token 和初始的几个 token。当 i 较小时，预测的 draft tokens 的长度较小，此时 attention weights 变化较小（最近的几个 token 的 overlap 很相近），因此 h_t 可以作为对 \widetilde{h}_{t+i} 的良好估计；而当 i 增大时，注意力系数也开始产生偏移, h_t 和 \widetilde{h}_{t+i} 的 gap 显著增大.

注意：EAGLE-1 的训练里是有特征对齐这一步的，即希望 \widetilde{h}_{t+i} 尽可能对齐 h_{t+i} . 然而 EAGLE-1 的训练只涉及 i=1 , 因此对齐 \widetilde{h}_{t+i} \sim h_{t+i} 相当于过拟合 attention weights 几乎保持不变这一假设，反而会导致 i 增大时，上述 4 项估计量发生更明显的偏移。值得一提的是，即使在训练时拟合 i>1 的 \widetilde{h}_{t+i} , 其仍然没法保证 \widetilde{h}_{\le t} 和 h_{\le t} 部分的约束。4 项估计量（对 prefix 的 attn weights 估计和 values 估计，对 draft tokens 的 attn weights 估计和 values 估计）导致的是一个跷跷板式约束，优化一边会导致另外一边估计偏差增大。

四、复用 KV Cache 的预测难度

当复用的 feature 层从 hidden states 转移到 KV cache 时， {K}_{\le t}, {V}_{\le t} 两个部分就不需要重新估计（已经得到了）。只需要重新计算 \widetilde{q}_{t+i}, \widetilde{K}_{t:t+i}, \widetilde{V}_{t:t+i} 。分为两部分分析：

对 prefix 部分的 attn weights 估计：由于已有 {K}_{\le t} , 因此只需要 估计准确 \widetilde{q}_{t+i} 即可得到正确的 attn weights. 难度低于复用 hidden states 的同时估计 \widetilde{q}_{t+i}, \widetilde{W}_{\le t} .
对 prefix 部分的 values 估计：无需估计，可直接复用。
对 draft tokens 部分的 attn weights 以及 values 估计：都没有有效估计，需要大量训练。
五、复用 KV Cache 的通信量

传输 KV Cache 的通信量会比传输 Hidden States 的通信量更小！

KV Cache 大小： 2\times B \times L \times N_{kv} \times H_{head}
Hidden States 大小： B \times L \times H

在 GQA 时代，有 H=G\times N_{kv}\times H_{head}

因此当 group 数 G 大于等于 2 时，KV 的通信量会小于等于 Hidden States 的通信量。

六、结论

本文尝试从理论方面来分析复用 hidden states 导致的长距衰减，根因是 hidden states 包含 target model 对 prefix tokens 注意力系数的历史偏见，从而导致预测长度较大时，预测能力产生偏移和下降。一个可能的解决方案是复用 kv cache，允许 draft model 自主更新对 prefix tokens 的注意力，从而提高 draft model 的预测能力。后续我也会尝试一些复用 kv cache 的实验，看看是否真的会比 hidden states 更有优势.
