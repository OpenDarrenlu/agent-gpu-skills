# TVM集成第三方库

**作者**: 工藤福尔摩MLSys

**原文链接**: https://zhuanlan.zhihu.com/p/389400552

---

出于某些原因，我们为了优化矩阵乘，将尝试集成第三方库，我这里先尝试的OpenBLAS（虽说性能较差），类似的方式可以用于集成其他库。

平台：ARM64 on an Android phone with RPC (在x86平台上集成是很简单的，tvm的cmake里打开openblas就行)

Motivation

我们以MobileNetV2作为benchmark network，该网络包含大量的1x1卷积，这种卷积计算上等价于矩阵乘。我们内部有个调优框架，在规模较大的时候，性能与MNN有较大差距。

比如说该网络后半部分有个卷积：输入规模是1 * 1280 * 7 * 7，kernel是320 * 1280 * 1 * 1。这个地方，MNN延迟是0.876ms，我们是0.88ms。但是当输入规模是1 * 960 * 7 * 7，kernel是320 * 960 * 1 * 1时，MNN是0.67ms，我们居然只做到了0.83ms，虽然目前还不能排除是搜索的问题。

之前考虑的主要原因：MNN对矩阵乘使用了复杂度较低的Strassen算法，当然我们也可以在TVM中实现这个算法。但是TVM的一大问题就是他的runtime只能一个算子一个算子运行，不支持inter-op parallism，参考http://graph_runtime.cc。Strassen是一个分治算法，MNN的多个分支（算子）是可以并行的，TVM即使使用Strassen算法目前也难以充分发掘该算法的性能。

一些考虑
想支持inter-op parallism，但修改runtime工作量比较大。
集成MNN的Convlution1x1Strassen部分，就是把他的代码提出来，单独编译成一个shared library，然后链接一下。嗯。。。但是他那个封装非常复杂：它在真正执行之前会调用onResize()，来分配部分内存（主要是一些local array，它有自己的内存管理），这些都让我觉得很麻烦。因为在不修改TVM Runtime的情况下，不太好搞定这个onResize()。
咱先试试别的库吧！最终选定OpenBLAS。
集成OpenBLAS

我们要做的大概有：编译OpenBLAS，把编译好的openblas.so与tvm runtime链接一下。

说起来很简单，但由于我是安卓小白，并不知道android这个要怎么搞，还是花了很多时间。

OpenBLAS交叉编译

下载OpenBLAS之后，首先要做的就是交叉编译。我自己的机器是linux-x86_64，要编译的target是arm64-v8a。OpenBLAS的编译脚本如下：

# set your own NDK path
export NDK=~/Android/android-ndk-r21e
export TOOLCHAIN=$NDK/toolchains/llvm/prebuilt/linux-x86_64
make clean
make \
    TARGET=ARMV8 \
    ONLY_CBLAS=1 \
    CC=$TOOLCHAIN/bin/aarch64-linux-android21-clang \
    AR=$TOOLCHAIN/bin/aarch64-linux-android-ar \
    HOSTCC=gcc \
    -j4
sudo make install

安装之后，在/opt/OpenBLAS/lib/目录下面有很多static and shared libraries。我们需要用的就是libopenblas.so，这是一个软链接，指向的是openblas_armv8_xxx.so。

TVM Android RPC集成
打开tvm/apps/android_rpc项目

2. 声明openblas module: 我们需要将openblas作为一个native module引入。项目的app/src/jni目录下新建openblas.mk，文件内容在下方。这个东西类似于Makefile，应该是NDK做的customize？第一次用。可以看到我们声明了一个名为"openblas"的module，然后声明了该module的shared lib地址，然后它暴露的接口函数在include里的cblas.h的头文件里，我们直接引入整个OpenBLAS/include/目录好了。

LOCAL_PATH := $(call my-dir)
include $(CLEAR_VARS)
LOCAL_MODULE := openblas
LOCAL_SRC_FILES := /opt/OpenBLAS/lib/libopenblas.so
LOCAL_EXPORT_C_INCLUDES := /opt/OpenBLAS/include
include $(PREBUILT_SHARED_LIBRARY)

3. 链接openblas module: 同一个目录下还有个Android.mk文件，该文件声明了一个"tvm4j_runtime_packed" module，这是android rpc应用依赖的主要module。我们在Android.mk开头的LOCAL PATH下面引入刚才的openblas.mk：include openblas.mk。这一行后面有include $(CLEAR_VARS)，所以我们不用自己再加clear vars之类的东西了。引入openblas.mk之后，我们在Android.mk的后面添加"LOCAL_SHARED_LIBRARIES := openblas"，这样之后构建项目时调用的ndk-build就会把openblas module链接起来了。

4. 注册openblas接口为tvm函数：jni目录下还有个"tvm_runtime.h"文件，显然是个声明运行时各种函数的文件，这个文件被jni下面的http://xxxapi.cc所include，它本身则include了许多tvm/src/runtime下的文件。然而，真正将openblas里的gemm作为一个外部函数来使用，发生在tvm/src/runtime/contrib/cblas/http://cblas.cc文件里。该文件include了前面提到的cblas.h，并且做了个wrapper，并注册为一个名为"tvm.contrib.cblas.matmul"的全局function。我们交叉编译的tvm module，就是通过全局函数注册表拿到这个func来用的。说了这么多，我们只需要在tvm_runtime.h里include ../src/runtime/contrib/cblas/http://cblas.cc即可，不过对于其他库，我们自己可以写个这样的注册机制，然后include。

测试

我们编译整个android_rpc app并安装到手机上，然后走正常的编译tvm module路子，就能进行RPC测试了，简单的测试代码：

import tvm
from tvm import te
from tvm.contrib import cblas, ndk
from tvm import rpc
​
def matmul(m, k, n):
  A = te.placeholder((m, k), name="A")
  B = te.placeholder((k, n), name="B")
  C = cblas.matmul(A, B, False, False)
  s = te.create_schedule(C.op)
  import numpy as np
  target = "android"
  if target == "android":
    build_tar_ = "llvm -device=arm_cpu -model=kirin990 -mattr=+neon -mtriple=arm64-linux-android"
  else:
    build_tar_ = "llvm"
​
  f = tvm.build(s, [A, B, C], build_tar_)
​
  if target == "android":
    f_name = "cblas_android.so"
    f.export_library(f_name, ndk.create_shared)
    key = "huawei"
    tracker_host = "192.168.1.100"
    tracker_port = 9195
    tracker = rpc.connect_tracker(tracker_host, tracker_port)
    remote = tracker.request(key, priority=0, session_timeout=60)
    print("conn established")
    remote.upload(f_name)
    mod = remote.load_module(f_name)
    dev = remote.cpu()
  else:
    dev = tvm.cpu()
    mod = f
  a = tvm.nd.array(np.random.uniform(size=(m, k)).astype(A.dtype), dev)
  b = tvm.nd.array(np.random.uniform(size=(k, n)).astype(B.dtype), dev)
  c = tvm.nd.array(np.random.uniform(size=(m, n)).astype(C.dtype), dev)
​
  mod(a, b, c)
  evaluator = mod.time_evaluator(mod.entry_name, dev, number=100, repeat=3, min_repeat_ms=300, f_preproc="cache_flush_cpu_non_first_arg")
  rslts = evaluator(*[a, b, c]).results
  perf = np.mean(rslts) * 1000
  print(c.asnumpy().mean(), "openblas perf: {:.4f}ms".format(perf))
