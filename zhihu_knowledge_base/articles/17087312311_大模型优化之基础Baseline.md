# 大模型优化之基础Baseline

**作者**: 菽陌松囿

**原文链接**: https://zhuanlan.zhihu.com/p/17087312311

---

前言:

此文目的, 对于性能优化工程师, 熟记各种基本操作性能上限, 有利于观测系统瓶颈, 以及预测系统向上的空间

平台:

GPU: A800

GPU: cpu 512+ core, 2T memory

OS: ubuntu

各种操作性能总结

GEMM: MFU 在78-86%之间不等, 要看具体shape

FA(开源sota 是flash-attn)比较拉垮fwd是57-68%, bwd是40-50%, headdim 最好是128, 64性能较差;

访存密集型, mul/norm各种访存op带宽利用率50-70%, 1.1T/0.7T(HBM2e理论带宽1.6T)左右;

Offload相关, h2d/d2h pageable-memory 8GB/s(pin memory 20GB/s, d2h其实更快), pcie5 理论带宽64GB，如果8卡均分就是2GB/s(串连pcie switch), offload本身原理很简单但是做到无损需要很强的系统功底;

关于通信, 400Gb ib网卡, 20+GB/s 40%传输效率;

欢迎有价值的base, 以上数据随手记, 可能有偏差

本文意义:

有了这些基础baseline数据之后, 通过观测模型推理那些算子/op实际运行MFU/MBU, 以及能否实现较好的overlap, 就可以有的放矢.

譬如deepseek v3为什么把moe expert 分4个node, 2个expert, 他的这些策略是通过IB/卡间p2p 理论数据推断, 但实际应该根据观测实际数据定更为合理
