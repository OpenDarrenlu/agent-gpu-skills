原始问题极其解决方案

问题描述：输入：(left, right, height)代表建筑物左边界x 坐标、右边界 x 坐标、高度，输出是城市的 skyline。(原题描述 leetcode.cn/problems/the-skyline-problem/ )

基本思路：维护一个优先队列 q，从左到右依次扫描每一条竖线，如果这条竖线是左边界线，把对应高度加入 q，否则把对应高度从 q 中删除。如果 q 中最大元素的值出现变化，则记录 q 中新的最大元素，作为对应 skyline 的纵坐标。（横坐标是当前竖线的 x 值）

细节注意：

边界条件判定：如果两条竖线横坐标相同，怎么处理。这里其实应该分类讨论。结论是都为左边界，优先考虑高的；都为右边界，优先考虑低的；一左一右优先考虑左边界。分析过程很简单，把图画出来，看看不同情况下我们期望的答案是什么样子即可。但是最为重要的就是在大体思路清晰之后，需要去考虑这种边界情况。

本题思路来自：https://www.youtube.com/watch?v=GSBLe8cKu0s

扩展问题1：2维→1维

问题描述：有一个画家，每天需要去负责[start, end]区间内的油画，但对于已经画过的区域不可以再画一遍。问每天需要画多长的区域。（原题描述：https://leetcode.cn/problems/amount-of-new-area-painted-each-day/）

基本思路：依旧是维护一个集合 s。这里建模的时候考虑竖线→(day, index)。按照 index 大小依次把各个点加入。如果 index 对应 start 点的坐标，把 index 加入 s；如果 index 对应 end 点的左边，把 index 从 s 中删除。每次执行删除/添加操作之前，检查 s 是否为空，如果不为空，那么这个 index 到之前那个 index 这个区域就应该是 s 中最小的元素那天画的。

细节注意：依旧需要去思考边界条件：如果 index 相同怎么办？思考之后发现这里顺序无所谓，只要你知道小于这个 index 的最大 index 是哪个就行了。不过为了代码简洁，如果都是 start 就先加 day 比较大的，如果都是 end 就先删day 比较小的；如果一个 start，一个end，先 end 再 start。

思考：其实碰到这个问题我最先想到的办法是线段树。这是一个可以在 O(log n)的时间复杂度内，维护当前有那些线段，每个线段的引用数是多少。但是截止目前我还没有真正碰到过非线段树不可的题。(当然我也不会实现线段树哈哈这玩意就是背模板，我只能说我曾经会过)。因此碰到类似的问题，首先需要想到的就是 skyline 这种解法。

另外比如，如果之前那个 skyline 问题问的是，在一个地点 x 最多有几栋楼。那么只需要每次去检查 q 的大小就行了。

扩展问题2：使用线段树替代优先队列

问题描述：之前 skyline 问题里，每个竖线起点的纵坐标都是0。如果取消这个限制会怎么样呢？例如：求矩形面积。给定几个矩形的横纵坐标。求矩形并集的面积。

基本思路：使用线段树替代优先队列。从左到右遍历竖线，如果是左边界就向线段树加入竖线，右边界则删除竖线。就每次我们就可以知道，目前被覆盖的线段总长是多少。这个长度乘上宽度就是当前 submatrix 的面积大小。关于这道题详情可以参考https://zhuanlan.zhihu.com/p/103616664
