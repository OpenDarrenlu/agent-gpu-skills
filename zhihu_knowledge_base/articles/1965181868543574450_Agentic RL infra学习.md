什么是 Agent (智能体 LLM)？

简单来说，Agent = LLM + 使用工具的能力 。

它不再只是一个聊天模型，而是一个能自主行动、解决问题的“行动者”。

文章 中举了一个计算数学积分的例子：

分析：Agent 知道自己不擅长计算，决定使用 Python 工具 。

行动（工具调用）：它生成一段 Python 代码（例如使用 sympy 库）来解决问题 。

执行：系统在“代码执行器”中运行这段代码，并得到准确结果（如 log(2)+7/3） 。

总结：Agent 拿到这个结果，再用自然语言回复你最终答案 。

什么是 RL (强化学习)？

RL (Reinforcement Learning) 是一种训练方法，核心是“通过试错来学习” 。

它不像传统SFT（监督微调）那样直接喂给模型“标准答案”，而是：

让模型自己去“探索”不同的做法。

如果做法带来了好结果（比如题做对了），就给它正向奖励 (Reward)。

如果做法带来了坏结果（比如代码报错），就给它负向奖励。

通过这种方式，模型会自己学会一套能获得最多奖励的“策略”（Policy）。

为什么要用 RL 训练 Agent？ (Agentic RL)

文章指出 ，如果只用 SFT（喂标准答案）的方式来训练 Agent，有两个大问题：

路径依赖：Agent 学不会自主决定“何时”该用工具。它只会死板地依赖提示词（prompt）来告诉它该怎么做 。

数据稀疏：用来训练的“标准答案”数据可能太少或太单一，导致模型要么学不会，要么只会死记硬背 。

Agentic RL 就是为了解决这些问题：

让 Agent 自主决策：在 RL 训练中，Agent 可以自主选择（比如是自己蒙一个答案，还是调用代码工具），环境会根据它的最终结果来给奖励 。这样它就学会了“何时”使用工具是最高效的。

鼓励探索：RL 鼓励 Agent 去尝试不同的解题路径，弥补了 SFT 数据不足的问题 。
