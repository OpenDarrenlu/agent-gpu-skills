# 投机采样：Llama-2 70B提速100%的秘诀！

**作者**: Uranus​清华大学 计算机系博士在读

**原文链接**: https://zhuanlan.zhihu.com/p/662950353

---

Llama-2 70B 是目前最大，也是质量最高的开源大模型之一。在很多对准确度要求较高的场景下，Llama-2 70B 是无可争议的首选。

然而 Llama-2 70B 实在是太慢了，fp16 精度下，直接用 transformers 进行推理只能达到不到 5 tokens/s 的效率。于是我们在 xinference 中引入了投机采样（speculative decoding）技术，以很小的代价，让 Llama-2 70B 的推理效率提升了 100%！

投机采样的技术在前几个月 GPT-4 泄漏的信息中便有提到。

这个算法简单来说是：

让小模型生成若干个 token。
将这些 token 一次性送入大模型进行一轮推理。
如果大模型赞同小模型生成的 token，那么我们一次性便采样了多个 token。
如果大模型模型不赞同小模型生成的某个 token，那么这个 token 和后续所有 token 全部舍弃。

是不是很像文章封面那样，两只羊驼相互交互？

这个算法依赖两个重要条件：

小模型和大模型的推理速度必须有显著差异，差异越大，效果越好。
小模型和大模型要“脾气”一致，即生成的 token 分布相似。

想要复现这个效果非常非常简单，我们首先启动 xinference：

$ xinference -p 9997

而后在另一个 terminal 中，执行下面的代码加载模型：

from xinference.client import Client

client = Client("http://localhost:9997")

model_uid = client.launch_speculative_llm(
    model_name="llama-2-chat",
    model_size_in_billions=70,
    quantization="none",
    draft_model_name="llama-2-chat",
    draft_model_size_in_billions=7,
    draft_quantization="none",
    n_gpu=2
)

speculative 模型加载后，你便可以像使用一个普通模型一样使用它，比如使用 xinference client 编程使用：

model = client.get_model(model_uid)

chat_history = []
prompt = "What is the largest animal?"
model.chat(
    prompt,
    chat_history,
    generate_config={"max_tokens": 1024}
)
