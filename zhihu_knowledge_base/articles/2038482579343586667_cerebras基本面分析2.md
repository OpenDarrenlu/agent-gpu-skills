一、装不下

WSE-3 自带 44GB 片上 SRAM，21 PB/s 的片上带宽，125 PFLOPS（FP16）的峰值算力。这套规格在不同工作负载下能力差异极大，决定了 Cerebras 真正具备结构性优势的场景范围。

以 Llama 3.1 70B 为例。FP16 精度下仅模型权重就需要 140GB，单颗 WSE-3 无法承载。INT4 量化后压缩到 35GB，可以放入 SRAM，剩余不到 10GB 空间用于 KV cache。在 KV cache 同样使用 FP16 精度的情况下，10GB 大约可承载 3 万 token 的上下文；KV cache 进一步量化到 INT8 或 INT4，可以扩展到 6 万至 12 万 token。

当模型规模或上下文超过片上 SRAM 容量时，Cerebras 通过外挂内存方案 MemoryX 承载权重。CS-3 单系统最高支持 1.2PB 的 MemoryX 配置。权重一旦下沉到外挂内存，访存路径变长，延迟优势随之收窄。在长上下文 benchmark 中，Cerebras 相对 GPU 的延迟优势从短上下文场景下的较大倍数显著收窄到几倍范围。

由此可以划定 Cerebras 具备结构性优势的场景：模型规模在 70B 量级以内、上下文不超过 32K 至 64K token、对延迟极度敏感的应用。客服对话、语音助手、短代码补全、Voice Agent 都落在这个区间。

Agent 类工作负载则位于这个区间之外。Claude 编码场景的平均上下文在 50K 至 200K token，Devin 类完整代码库 Agent 经常突破 500K，GPT-5 多轮 Agent 上下文可达百万级。这些负载下仅 KV cache 就需要几十到几百 GB 内存，Cerebras 没有结构性优势。低延迟推理需求增长很快，但在整个 AI 工作负载结构中并不是主体，Agent 类长上下文、长链路、工具调用的负载才是大头。

二、贵

我觉得这和买房去生活很像。NVIDIA H100 一张 2.5 万美元，B200 一张 3 万到 4 万，单卡功耗 700W 到 1000W。这种采购量级相当于买一套公寓，月供和水电费都好估算。Cerebras CS-3 一台 200 到 300 万美元，单价比 GPU 高一个数量级，是别墅级别的支出。

别墅的水电也跟着上去。CS-3 峰值功耗 23kW，DGX B200 整机 14.3kW，绝对值高出 60%。但 CS-3 跑 125 PFLOPS（FP16），DGX B200 整机只有 36 PFLOPS，单位算力的功耗 Cerebras 反而低一半。真正麻烦的不是电费本身，是电力密度和散热。数据中心要为 23kW 单机柜重做供电和冷却管路，这笔改造成本绕不开。

WSE 的热密度高到没有风冷选项，CS-3 强制液冷。NVIDIA Blackwell 也在转液冷，但 B200 至少还有风冷版本。Cerebras 客户没得选，机房改造是硬开支。

折旧这块要单独讲。GPU 折旧近一年已经是云厂商最大的会计争议。Michael Burry 直接点名 Microsoft、Meta、Oracle、Amazon 和 CoreWeave，说它们把 GPU 折旧年限从 3 到 4 年延长到 6 年是粉饰报表，目的是压低折旧、抬高利润，按他估算 2024 年这一项就让头部云厂商少计提了 180 亿美元的折旧。CoreWeave 因此被反复做空，股价从 2025 年 6 月的高点跌了一半多。这场争议到现在没有定论。

但 GPU 至少还有退路。H100 合同到期释放出来，二级市场能接近原价重新租掉；A100 退役后转推理负载继续创收。"训练→推理→批处理"的复用阶梯，是 GPU 行业撑住 6 年折旧的真正底气。CS-3 没有这套阶梯。专用设备没有二手市场，没有降级通道，客户买回去就是 5 年内必须靠自己跑满产能摊销掉的固定资产。需求一旦不达预期，账面价值就只能减值。

把这些加在一起，Cerebras 的总持有成本（采购、电力、冷却、折旧）都明显高于 GPU。这部分溢价必须在终端产品定价上挣回来。挣不回来，就直接挂在大模型公司的资产负债表上。

三、商业模式

Cerebras 在硬件成本端处于高端定位，但 LLM 推理市场目前没有对应的高端定价结构。

OpenAI 的 ChatGPT Plus 售价 20 美元/月，Pro 200 美元/月，Enterprise 走定制。API 按 token 数收费，与生成速度无关。用户使用 ChatGPT 时每秒获得 30 token 还是 100 token，账单价格相同。

OpenAI 和 Anthropic 的 API 都已有 Priority Tier 或 Service Tier 概念，承诺更低延迟和更稳定的吞吐，价格高于标准 API。但当前的溢价幅度（约标准价格的 1.5 至 2 倍）远不足以覆盖 Cerebras 的成本结构，也没有承诺端到端 100 毫秒级的延迟 SLA。

供给端是高端成本结构，需求端是接近标准化的定价结构，中间没有商业出口。大模型公司每多部署一台 CS-3，多出的成本无法通过现有产品结构转嫁给终端用户，只能记入资本开支。这种结构在大规模部署阶段不可持续。

要让 Cerebras 这类高单价算力实现商业闭环，模型厂商需要在产品端创造新的分层。以下是本人的猜测，可能有三类。

用户分层路径。 在 ChatGPT Pro（200 美元/月）之上再开一档，例如 2000 美元/月，主打”任何回答 50 毫秒内开始流式输出、token 生成速度为标准版的 5 倍”，对应投行、律所、咨询、医疗等对响应速度高度敏感的高净值用户。

API 优先通道路径。 在 API 层面扩展现有 Priority Tier 的能力，承诺端到端延迟低于 100 毫秒，价格设定为标准 token 的 3 至 5 倍。AWS S3 已经有 Standard 和 Express One Zone 多档定价，”快”与”贵”按场景挂钩，相同逻辑可以平移到 LLM 推理。

用例分层路径。 将 Cerebras 算力独立分配给reasoning agent语音助手等实时性强且有难度的任务产品线，普通文本生成仍走 GPU。按用例划分，而非按用户划分。（虽然还是会被上下文长度卡住。。）
