# 当我们在谈论AGI时，我们在谈论什么

**作者**: Lin Zhang​香港科技大学 计算机科学技术博士

**原文链接**: https://zhuanlan.zhihu.com/p/687353235

---

当我们在谈论AGI（通用人工智能）时，网络上总是充满各种各样的声音。

很不幸，很多声音在转述的过程中，往往会脱离原本的背景，然后脱离原本的含义。

在这篇文章中，笔者希望尽可能严谨地记录下这些观点，然后帮助大家更好地讨论。

首先，让我们从OpenAI开始说起。

压缩即智能

当我们谈到OpenAI的路线，压缩即智能是其中最广为流传的观点。

其中，压缩是一个被滥用的词语。常见的例子包括：大模型是将训练数据里的知识“压缩”到模型参数，Auto-encoder模型是将输入“压缩”到潜在空间，RNN模型是将序列信息“压缩”到状态空间。

尽管以上表达符合“压缩”的字面含义，这和OpenAI所讲的压缩并不是同一件事。

在OpenAI的语境下，压缩即智能指的是一种无损压缩。简单来说，就是压缩后的数据我们可以完整地恢复，例如大家常用的gzip压缩软件。那么，无损压缩和大模型有什么关系呢？

无损压缩与大模型训练

在OpenAI员工Jack Rae的采访视频中[1] ，提到我们可以通过大模型的训练来压缩数据。当然，由于大模型压缩数据的开销巨大，这并不是一种实际可行的算法，而是提供一种视角来理解大模型的训练。

其中，条件概率的优化问题，就是我们常说的next token prediction。在无损压缩的视角下，大模型预测下一个单词的训练过程，就是一种压缩。

在压缩过程中，我们初始化一个大模型，使用当前参数预测下一个单词的概率，进行算数编码，计算梯度更新当前参数。然后，我们用更新后的模型参数预测再下一个单词的概率，进行算数编码，用梯度更新参数。反复以上过程，当我们遍历整个数据集，就可以得到所有数据的编码（压缩后的数据），同时包括一个训练后的模型。

如果要完整地恢复整个数据集，我们只需要记录初始化模型所需要的随机数种子，训练模型所需要的代码，和数据压缩后的编码。在解压缩的过程中，我们以同样的方式训练一个大模型，预测下一个单词的概率，进行解码[2]。

值得注意的是，在解压过程中，我们并不需要用到之前训练得到的模型。也就是说，大模型的训练过程是一种无损压缩，而训练得到的大模型其实是压缩后的副产品。

在无损压缩的视角下，提高大模型的训练效果，就等于提高数据的压缩率。直观来说，当我们画出整个模型的训练loss曲线，曲线下方的面积越小，就意味着数据的压缩率越高。数据的压缩率越高，就意味着模型越智能。

压缩率高为什么意味着智能，一个形象的例子，是压缩圆周率小数点后100亿位的数字。

如果模型掌握了该数据生成的规律（足够智能），理解圆周率的数学含义，模型只需要编写一个数值算法来生成这些数字，而代码的数据量远远小于这些数字本身的规模。

从Llama的训练曲线来看[3]，大模型训练带来的压缩率，远超市面上最强的无损压缩软件，这需要模型从数据中学习到相应的结构。

Scaling Law

在无损压缩的视角下，我们发现压缩率和大模型本身的参数量没有什么关系。当数据充足的时候，扩大模型规模往往能得到更好的loss曲线，也就是得到更加智能的模型。

如果要量化这个认识，我们需要构建一个经验公式，也就是所谓的scaling law，来预测扩大模型规模和数据规模（计算成本[4]）能取得的效果。例如，在GPT4的论文中，经验公式为:

L(C) = a C^b + c,

其中，C为计算成本，L(C)为最终的loss值。也就是说，L(C)随着计算成本的增加而下降，最后收敛到c（无法避免的预测误差）。其中，下降的速度（a和b的大小）取决于模型能力和数据质量。

目前而言，transformer decoder模型在文本数据上的扩展已经得到充分的验证，而其他模型和模态的扩展效率还需要更多、更大规模的实验支持。尽管scaling law预言了大模型扩展的上限（受限于c的大小），目前的模型距离极限应该还有不少空间，没人知道在这个过程中模型还能涌现出多少智能。

相关的讨论

回到无损压缩的视角，让我们继续展开更多相关的讨论。

第一，每一个预测器都是压缩器，每一个压缩器都是预测器（Every predictor is a compressor. Every compressor is a predictor）。

我们已经解释了基于自回归的预测器可以作为一个压缩器，那么，还有其他的预测器吗？答案自然是有的，预测器的含义在于建模数据的概率分布。除了自回归模型，我们还可以使用VAE模型[5]或者Flow模型[6] 来充当无损压缩器。只是对于文本数据来说，因为离散化和序列性的特性，天然适合使用自回归模型建模。

另一方面，如果说每一个压缩器都是一个预测器，我们是否可以使用gzip压缩算法预测下一个单词？理论上来说，是的，我们可以贪心地选择能够取得最大压缩率的下一个单词作为预测的结果。当然，使用gzip压缩算法构建的语言模型实际上无法输出连贯的语句[7]。

第二，生成即理解（Generation is understanding）。

简单来说，生成即理解指的是，当模型能够很好地生成一段数据，它也能够很好地理解这段数据。

这个观点争议相对比较大，以视频生成模型Sora为例，如果模型可以生成比较连贯的视频，是否意味着这个模型已经理解了其中的物理规律，还是说仅仅是对物理世界的模拟？对这个问题，我暂时没有找到比较严谨的解释。

让我们先考虑其中一个子问题：如果我们训练模型学会生成一段数据，是否有助于模型理解这段数据。也就是说，让模型学习P(x)，是否有助于模型学习P(y|x)[8]。以iGPT为例，让模型学习生成图像（无监督学习），是否有助于图像分类任务（有监督学习）。

对于这个问题，Ilya Sutskever在一次讲座中[9]尝试从压缩的角度给出一个肯定的解释。简单来说，我们联合压缩数据集(X, Y)的表现，要好于单独压缩数据集X和数据集Y，这是因为压缩X过程中学习到的结构有助于我们压缩Y，反之亦然。当然，无损压缩的视角无法解释为什么其他自监督的方法有效，例如对比学习。

第三，所有的预测问题都可以看成是自回归（Any learning problem can be phrased as sequence prediction）。

将所有的预测问题建模成自回归问题，意味着我们可以用统一的架构来压缩所有的数据。

在Jan Leike的博士论文[10]中，解释了为什么所有的预测问题都可以看成是自回归。简单来说，计算机上的所有数据都是字节串，对任意数据的预测，都可以通过对该字节串序列进行概率建模来解决。

当然，都可以建模成自回归问题，不等于这么做就是最优的方案。例如对于图像数据来说，基于像素点的自回归代价过大，而采用视觉分词器的做法必然会损失信息。

反对的看法

尽管OpenAI的大模型路线得到了很多人认同，也有其他人提出了反对的看法，其中最激烈的恐怕是Yann LeCun。在Yann对大模型路线的众多批评中，值得注意的有两点。

第一，自回归模型可能不适合视觉建模。

我们前面提到过，使用自回归的方式生成图像，会带来极高的成本（基于像素），或者信息的损失（基于分词）。另外，图像本身并没有明确的序列性，任何一种序列预测的顺序（例如从左到右、从上到下）都会引入偏见。

与之相对的，diffusion模型将图像看成是连续变量，并且不依赖于任何一种序列顺序（non-causal），在图像生成上取得sota的表现，以至于OpenAI自己的图像生成/视频生成的产品中也选择了diffusion模型。

第二，现实世界的预测具有很大的不确定性。

对于自回归概率模型而言，我们需要根据过去的信息预测未来的信息（next token prediction）。

然而，在真实世界中，对未来状态的预测相当困难。例如在视频数据中，有太多可能的常规事件，还有一些不太可能的突发事件，导致预测准确十分困难。Yann本人更是建议使用EBM，放弃预测P(y|x)的概率，转而学习两者的匹配度。

总而言之，上述批评的关键，在于自回归能否将其在语言模型上取得的成功迁移到其他模态和其他任务，例如视觉和机器人。然而，要实现这些并不容易，毕竟压缩视角本身，并没有提供方法论上的指导。
